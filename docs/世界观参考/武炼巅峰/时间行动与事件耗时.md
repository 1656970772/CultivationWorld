## 游戏化抽象（我的判断）

| 规则 | 依据 | 建议实现 | 边界 |
|------|------|----------|------|
| 行动需要机会成本 | 采药、闭关、探查、航行都会占用时间并让外界继续变化。 | 所有高价值行动写 `duration` 和 `worldTickDuring=true`。 | 不硬造原文没有的精确天数。 |
| 时间尺度随地图升级 | 早期以天/月，星域以月/年，圣灵契约以千年。 | 按 `travelStage` 调整默认时间单位。 | 游戏中可压缩表现，但文档保留原文尺度。 |
| 窗口事件会过期 | 拍卖会三个时辰开场，探查一月回报，战场半日距离。 | `announceDay/startDay/endDay` 或 `startHour`。 | 原文没说截止日时写相对窗口。 |
| 长期任务会改变技能 | 炼器三年/十年带来技艺提升。 | 长项目应持续产出 skill exp 和关系变化。 | 不把主角成长速率当普通 NPC 标准。 |
| 秘境内部可隐藏真实时间 | 异地无日月、世界果内外时间差，角色感知和世界时钟可分离。 | 建 `localTime` 与 `worldTime` 两套字段。 | 仅对有证据的特殊空间启用。 |
| 传送网络仍有维护和身体负担 | 乾坤殿传送压缩路程，但修为不足需休息，法阵需维护。 | 传送后加 `fatigue` 和 `arrayWear`。 | 具体数值待机制侧设计。 |
| 战争事件需要“准备期/交战期/清扫期” | 魔族大战十几天、碧落关清扫两三月、军团造册三日体现不同阶段。 | 战役拆 `mobilize/fight/cleanup/rebuild`。 | 战役时间线需专题核验。 |
