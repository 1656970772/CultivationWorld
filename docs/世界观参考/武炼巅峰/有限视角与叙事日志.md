## 文案模式（我的判断）

| 事件 | 确认文本 | 消息文本 | 传闻文本 | 边界 |
|------|----------|----------|----------|------|
| 秘境开启 | “你亲眼见到传承洞天入口开启。” | “同门回报：黑风林有洞天现世。” | “坊间传闻黑风林有强者遗留宝藏。” | 不暴露谁获得最终传承，除非目击或被公开。 |

## 游戏化抽象（我的判断）

| 规则 | 依据 | 建议实现 | 边界 |
|------|------|----------|------|
| 日志按来源分级 | 原文存在亲眼、打探、请柬、担保、推测等多层信息。 | `confirmed/message/rumor/inference` 四级日志。 | 不用全知旁白替代角色视角。 |
| 日志必须带来源 | 骨族消息是否可信取决于圣地背书；拍卖会取决于请柬。 | 每条日志存 `sourceType` 与 `sourceName`。 | 无来源信息仅能写“未见可靠证据”。 |
| 可见性受距离和能力影响 | 神识、眼力、斥候和传令决定可知范围。 | 地图迷雾 + 感知半径 + 消息传播。 | UI 不应显示远方背包/真实身份。 |
| 日志应允许“事实未明”状态 | 控魂虫、墨徒、圣灵传闻都存在只见表象的阶段。 | 日志状态加 `unverified/partial/confirmed/false`。 | 不把读者全知写给角色。 |
| 路线与地图要逐步解锁 | 乾坤图、星图、烙印决定可见路径和可用传送。 | 地图节点按 `known/visited/attuned` 三层显示。 | 不知道坐标时不显示精确目的地。 |
| 高影响低可信消息仍应驱动行为 | 药王谷/圣灵传闻未必全真，但角色会因此避讳或准备。 | AI 决策同时看 `reliability` 和 `impact`。 | 谣言机制需允许误判。 |
| 敌方信息也有视角限制 | 墨巢通信难定向，墨族也可能不知道杨开是否在不回关外。 | 敌方 NPC 也使用有限视角，不给完美追踪。 | 需要机制侧支持双向迷雾。 |
