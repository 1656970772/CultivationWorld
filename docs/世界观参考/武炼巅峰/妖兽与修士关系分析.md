## 游戏化抽象（我的判断）

| 生态规则 | 依据 | 建议实现 | 边界 |
|----------|------|----------|------|
| 普通妖兽和妖族分层 | 黑风山妖兽与通玄妖族社会差异明显。 | `monster` 与 `faction_npc` 分开建模。 | 不把所有妖族都当野怪。 |
| 守护资源是常见遭遇 | 三阶妖兽守草药、秘境妖兽守宝。 | 资源点可绑定 `guardianMonsterId`。 | 掉落需逐案核验。 |
| 化形是事件不是掉落 | 化生池涉及妖族根基、阵法和生命风险。 | `DynamicEvent.BeastTransformation`，成功生成妖族 NPC。 | 不把化生池做成普通消耗品。 |
| 圣灵契约带期限 | 诸犍三千年效忠后自由。 | 契约系统需支持期限、誓约、自由恢复。 | 其他圣灵契约未必同样期限。 |
| 妖兽材料要分部位 | 赤尾紫甲蝎等证据显示高阶妖兽不是一个统一掉落包。 | 掉落表按部位、用途、参战资格拆分。 | 具体概率未见可靠证据。 |
| 妖族可做势力而非怪物 | 帝辰星赤月领有领主、行宫、比赛和人族政策。 | 妖族领地进入 `Faction` 系统，野生妖兽进入 `Monster` 系统。 | 各领主人族政策需逐个核验。 |
| 圣灵资源要绑定报复风险 | 圣灵幼体/后裔会引来成熟圣灵，市场避讳公开交易。 | 圣灵资源增加 `retaliationThreat` 和 `marketTaboo`。 | 哪些资源可安全交易待核验。 |
| 墨族单独建生态系统 | 墨虫、墨烟、墨巢、墨徒、孕育和疗伤形成闭环。 | 建 `InkEcology`，不要复用妖兽繁殖或野怪刷新。 | 墨巢等级、资源消耗和产出速度需专题化。 |
