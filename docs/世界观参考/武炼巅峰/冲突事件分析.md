## 游戏化抽象（我的判断）

| 冲突规则 | 依据 | 建议实现 | 边界 |
|----------|------|----------|------|
| 小冲突可升级 | 血战帮/风雨楼事件显示个人冲突可牵动宗门。 | 记录 `offender`、`victimFaction`、`patron`，满足条件后升级为追责或复仇链。 | 不能每次个人冲突都自动宗门战，需要声望、伤亡、亲缘/师门关系等条件。 |
| 制度冲突可日常化 | 凌霄阁挑战把打斗、贡献点和记录机制绑定。 | 宗门可拥有 `challengePolicy`，让日常冲突成为资源转移和关系变化来源。 | 不同宗门制度不同，不能全作品泛化为固定规则。 |
| 机会点吸引多方 | 传承洞天、五色宝塔、拍卖、太墟境都让多方围绕同一资源行动。 | 动态事件按价值、公开度、准入条件吸引 NPC，并生成竞争/合作/追踪。 | 主角隐秘机缘可设置低公开度或私人触发。 |
| 权力差距会压低冲突烈度 | 海克家族受影月殿压制，弱方难正面反抗。 | `powerGap` 高时弱方倾向忍让、私下求援、投靠外强。 | 具体阈值和反抗概率属于项目数值，不是原作事实。 |
| 地缘和虚弱窗口会诱发战争 | 飞烟殿趁七巧地元气大伤介入。 | 势力 AI 应关注邻域虚弱、旧怨和可夺资源。 | 不要让所有邻接势力自动开战，需利益和风险评估。 |
| 潜伏敌人控制信息暴露 | 墨徒分批行动避免人数骤减引起怀疑。 | 潜伏事件应有 `exposureRisk`、`batchSize` 和清查强度。 | 墨族规则需要另行专题化，不能混同普通邪修。 |
| 妖兽可从野怪变为阵营兵力 | 幽冥山异地中妖兽被控魂虫奴役后改变冲突形态。 | 妖兽控制机制应改变 encounter 阵营，而非只给驯兽标签。 | 控魂虫限制和反制规则仍待补证。 |
| 公开机缘会影响本地市场 | 五色宝塔开启前疗伤丹涨价。 | 事件公告后临时调整相关丹药、秘宝、情报和护具供需。 | 涨价幅度不能直接泛化，需平衡设计。 |
| 高阶禁忌资源要带报复条 | 圣灵资源价值越高，成熟圣灵报复风险越大。 | 给圣灵幼体/后裔/本源资源设置 `retaliation_threat`。 | 龙凤、祖地、太墟圣灵分支需拆分核验。 |
| 危机场景会重置秩序 | 太墟境使星市秩序崩坏、开天受限、临时势力生成。 | 危机地图可临时覆盖执法、战力上限、交易和组队规则。 | 不应长期改变全世界规则，只在特定事件/地图阶段生效。 |
| 敌方生产节点比普通据点更关键 | 墨巢可生产、晋升、疗伤，毁巢收益高但会引发反扑。 | 战争地图中区分 `outpost` 与 `production_node`，并给生产节点更高战略权重。 | 墨巢等级和域主权限待专题核验。 |
