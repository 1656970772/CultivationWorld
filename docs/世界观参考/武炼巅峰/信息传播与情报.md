## 游戏化抽象（我的判断）

| 规则 | 依据 | 建议实现 | 边界 |
|------|------|----------|------|
| NPC 不应全知 | 杨开因不结交而不知道黑风贸市。 | `knownNews` 应受关系、地点、阵营、职业影响。 | 主角旁白信息不等于角色已知。 |
| 消息有可信度等级 | 偷听、请柬、圣地担保、亲身神识确认强度不同。 | `reliability=rumor/observed/official/verified`。 | 原文无概率，数值由项目设计决定。 |
| 信息会触发行动 | 秘图消息触发跟踪，骨族消息触发联盟，拍卖请柬触发赴会。 | `news -> utility goal -> preparation job`。 | 低可信传闻应允许误判。 |
| 潜伏信息需延迟暴露 | 墨徒控制自陨节奏避免暴露。 | 加 `exposureRisk`、`suspicion`、`investigationProgress`。 | 墨族规则需专题化。 |
| 同一消息可分层可见 | 五色宝塔公开公告人人可知，分散进入和开启内幕只在小圈层流动。 | `visibility=public/private/insider`。 | 不同 NPC 应按关系网获得不同版本。 |
| 地图信息应决定行动半径 | 乾坤图、星图、乾坤殿烙印直接影响能否到达或逃离。 | `routeKnowledge` 与 `transitAccess` 分开记录。 | 地图信息过期问题待设计。 |
| 反潜伏调查要有风险 | 查小乾坤最直接，但可能惊动墨徒背后的墨族。 | 调查行为产生 `exposureToEnemy` 与 `panicRisk`。 | 具体概率无原文证据。 |
| 敌方通信也可被利用或误判 | 墨巢通信不定向，杨开考虑沟通但因风险作罢。 | 敌方网络可设 `interceptRisk` 和 `identityExposure`。 | 墨巢网络规则需专题化。 |
