## 设定摘要

角色 AI 可围绕“资源、风险、关系、阵营、污染”五个维度建模。凌霄阁早期提供挑战与贡献点循环；夺嫡战提供阵营下注与追随者选择；散修线说明高阶 NPC 也会因资源和安全改变策略；墨族线则需要处理被侵蚀后的强制阵营转换和救治行为。

## 行为链卡片

### 低阶挑战/避战

| 项 | 内容 |
|---|---|
| 代表角色/事件 | 凌霄阁弟子挑战规则 |
| 输入信息 | 自身境界、对手境界、伤势、贡献点缺口、挑战冷却 |
| Utility 目标 | `gain_contribution`、`avoid_injury`、`gain_reputation` |
| GOAP 目标状态 | 贡献点增加且伤势可接受；或避战成功。 |
| Job/Toil 步骤 | 选择对手 -> 发起挑战 -> 战斗 -> 领取贡献变化 -> 疗伤/兑换 |
| Reaction 打断 | 伤势过高、境界差过大、连续挑战冷却未结束。 |
| 成功结果 | 贡献点、名声、突破资源增加。 |
| 失败结果 | 受伤、贡献点降低、产生仇怨或羞耻记忆。 |
| 可映射文件 | `dynamic-goals.json`、`npc-job-actions.json`、`jobs/challenge` |
| 证据 | W08-AI-01、02 |

### 资源任务循环

| 项 | 内容 |
|---|---|
| 代表角色/事件 | 普通宗门弟子通过任务、寻宝和上缴获取贡献点。 |
| 输入信息 | 贡献点余额、突破资源缺口、任务风险、可上缴物品价值。 |
| Utility 目标 | `gain_contribution`、`obtain_resource`、`prepare_breakthrough` |

| Reaction 打断 | 伤势过高、任务窗口关闭、被抢夺、贡献点不足以兑换目标资源。 |
| 成功结果 | 获得修炼资源并推进突破准备。 |
| 失败结果 | 损失时间、伤势增加、资源被夺或形成债务。 |
| 可映射文件 | `jobs/resource-task`、`toils/turn-in-loot`、`npc-job-actions.json` |
| 证据 | W08-AI-02 |

## 待确认问题

- 贡献点收益、挑战冷却、受伤恢复的具体数值需从游戏系统另行设计，原文只提供规则方向。
- 墨化救治成功率、失败代价和净化资源消耗需要后续章节补证。
- 专家型 NPC 的“违背委托但提升实效”需要设计信任/解释机制，否则容易表现为随机改需求。
- 宗门招收门槛的资质、贡献、推荐关系权重仍缺原文硬数值。
- AI 性格标签与行为模块的权重关系尚未数值化。

## 二次扩展行为链卡片

### 空间逃生/合作

| 项 | 内容 |
|---|---|
| 代表角色/事件 | 杨开、鬼祖；封闭空间/虚空甬道探索 |
| 输入信息 | 是否有空间法则能力、强者胁迫、逃生目标、探查风险 |
| Utility 目标 | `escape_trap`、`survive_coercion`、`exploit_unique_skill` |
| GOAP 目标状态 | 找到出口并降低被强者控制的风险。 |
| Job/Toil 步骤 | 暴露/隐藏能力 -> 谈判合作 -> 用分神/探针探索 -> 评估甬道 -> 选择逃离时机 |
| Reaction 打断 | 分神濒临崩溃、甬道不稳定、强者翻脸。 |
| 成功结果 | 逃离封闭空间，或获得强者短期保护。 |
| 失败结果 | 分神损伤、被强迫继续探索、死亡/迷失虚空。 |
| 可映射文件 | `special-abilities.json`、`escape-jobs.json`、`risk-probes.json` |
| 证据 | W08-AI-B01 |
