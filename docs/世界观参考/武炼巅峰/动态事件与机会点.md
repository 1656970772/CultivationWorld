## 我的判断

| 规则 | 依据 | 建议实现 | 边界 |
|------|------|----------|------|
| 动态事件需要生命周期 | 多数事件不是瞬时奖励，而有消息、准备、参与、结算和余波。 | 统一使用 `scheduled/announced/active/resolved/expired`。 | 不给原文未写的事件硬造精确日期。 |
| 事件价值决定扩散强度 | 传承洞天、拍卖会、骨族危机、战功秘法扩散方式不同。 | `value`、`secrecy`、`sourceAuthority` 控制传播。 | 主角秘密机缘应允许低公开度。 |
| 战争改变机会结构 | 战功兑换和战场清扫都来自战争压力。 | 战争期动态目标替换和平期宗门任务。 | 战功兑换细则待核验。 |
| 地图级事件可覆盖日常规则 | 太墟境封开天、吞星市、禁止主动离开，直接改写地图逻辑。 | 动态事件可挂载 `ruleOverride`。 | 覆盖项必须有证据，不泛化。 |
| 服务型机会也应事件化 | 药王谷、乾坤殿不是战斗事件，但能触发求丹、避难、交易和禁地潜入。 | `service_event` 与 `combat_event` 分开。 | 收费和失败率待核验。 |
| 顶级机会要带世界余波 | 乾坤炉、世界果、圣灵契约都会改变高阶格局。 | 高价值事件结算后写入 `worldMemory`。 | 余波规模由证据强度决定。 |
| 敌方生产节点可反复驱动任务 | 墨巢不只是一次战斗点，还负责产兵、疗伤、通信。 | 清剿任务应削弱区域敌军刷新/恢复。 | 具体产出速度待核验。 |
