# 冲突事件分析模板

> 最后更新：2026-06-06
> 状态：模板
> 类型：个人、势力、资源、秘境、族群冲突调研
> 目标详细程度：L3

## 适用范围

用于整理作品中的杀人夺宝、宗门战争、秘境争夺、正邪冲突、复仇、内部斗争、种族冲突、通缉追杀等事件。采集时应尽可能覆盖各修为阶段、各势力层级、各资源类型、各冲突规模、各胜负后果和明暗战斗形态出现过的冲突样本，避免只记录主线大战。它服务战斗风险、怀璧其罪、关系仇怨、动态事件、势力 AI、信息传播和死亡后果。

## 必须写什么

| 模块 | 必写内容 | 项目用途 |
|------|----------|----------|
| 冲突类型 | 个人/势力/族群/资源/秘境/内部 | 事件分类 |
| 触发原因 | 资源暴露、仇怨、领土、机会窗口、道统冲突 | 机会点、动态目标 |
| 参与者 | 角色、势力、第三方、旁观者 | 事件参与、信息传播 |
| 冲突过程 | 侦查、试探、偷袭、围杀、逃亡、谈判 | 行为链 |
| 后果 | 死亡、抢夺、结仇、势力变化、消息扩散 | 关系、记忆、WorldNews |

## 尽量写全的内容清单

- 个人冲突：杀人夺宝、截杀、复仇、羞辱、争道侣、争师承、争功法、争法宝、抢秘境名额。
- 势力冲突：宗门战争、正邪大战、王朝征伐、商会竞争、附庸叛乱、继任内斗、派系清洗。
- 资源冲突：灵矿、灵脉、药园、妖兽尸骸、拍卖品、古宝、传承、秘境核心资源。
- 秘境冲突：入口争夺、组队背叛、限时抢夺、禁制触发、出口截杀、收获暴露后的追杀。
- 种族/生态冲突：人妖领地、兽潮、异族入侵、化形妖修建势力、灵宠争夺。
- 信息相关冲突：传闻引发争夺、宝物暴露、通缉、悬赏、情报泄露、误判导致冲突。
- 冲突阶段：预兆、消息传播、集结、试探、爆发、升级、调停/撤退、结算、长期后果。
- 项目后果：死亡、掉落、抢夺、关系边、仇怨记忆、机会点、动态事件、势力稳定度变化。

## 不要写什么

- 不只写“双方大战一场”。
- 不把主角胜利方式当成普通战斗规则。
- 不忽略冲突前的信息传播和冲突后的关系后果。
- 不把正邪标签当作冲突原因；要写资源、利益、制度或生存压力。

## 详细程度

- 冲突类型总览中每一条已采信或有待核验价值的冲突，都应尽量写成完整事件卡；不要只挑少数“代表事件”。
- 事件卡必须有触发、参与者、信息来源、行动链、结果、关系/资源变化、消息传播和证据。
- `可映射系统` 不是筛选条件；暂时没有现成系统时，写“待定 / 剧情参考 / 世界状态参考 / 任务参考 / 日志参考”。
- 如果只能写简版卡片，也要列出已知事实和待补字段，不能只停留在总览表。
- 对可转化事件可继续写清是 WorldNews、Opportunity、DynamicEvent、Quest、Relationship、Memory 还是其他后续参考。
