# 时间行动与事件耗时模板

> 最后更新：2026-06-06
> 状态：模板
> 类型：行动耗时、闭关时长、旅行时间、事件窗口与世界同步调研
> 目标详细程度：L3

## 适用范围

用于整理作品中行动需要多长时间、时间消耗如何带来机会成本。采集时应尽可能覆盖各修为阶段、各行动类型、各距离尺度、各闭关/炼制/任务周期、各事件窗口、各延误后果和各时间压缩/加速手段。它服务行动点、世界 Tick、闭关、游历、任务、动态事件窗口和“玩家忙这边，那边已经变天了”的体验。

## 必须写什么

| 模块 | 必写内容 | 项目用途 |
|------|----------|----------|
| 行动耗时 | 移动、闭关、炼丹、炼器、任务、战斗、疗伤 | time-action |
| 时间窗口 | 秘境开启、拍卖、大比、通缉、追杀、飞升 | dynamic event |
| 并行变化 | 角色闭关时外界发生什么 | world tick |
| 机会成本 | 错过事件、资源耗尽、敌人变强、关系变化 | AI/玩家体验 |
| 时间尺度 | 日、月、年、百年、纪元、相对阶段 | 叙事节奏 |

## 尽量写全的内容清单

- 行动：赶路、传送、飞行、闭关、突破、疗伤、炼丹、炼器、炼化法宝、温养本命法宝、阵法布置、采集、猎妖、交易、打探。
- 长期行为：宗门培养弟子、势力战争、秘境探索、恢复伤势、闭死关、等待天劫、等待拍卖/秘境。
- 时间窗口：公告日、准备期、开启期、结束期、过期、追杀期限、任务期限、悬赏有效期。
- 外界变化：势力更替、亲友死亡、资源价格变化、消息扩散、机会点过期、敌人突破、宗门覆灭。
- 时间单位：天、月、年、甲子、百年、千年、纪元、章节阶段、地图篇章。
- 项目映射：行动点、daysPerYear、durationDays、announceDay/startDay/endDay、Job wait toil、world tick。

## 不要写什么

- 不把所有行动都写成瞬时。
- 不只写“闭关多年”，要说明闭关前后状态变化。
- 不硬凑精确天数；原文没有时可写相对时长。
- 不忽略世界同步演化。
