# 法宝分析模板

> 最后更新：2026-06-06
> 状态：模板
> 类型：法宝、秘宝、灵宝、神器等机制资产调研
> 目标详细程度：L3

## 适用范围

用于分析单部作品中的法宝、秘宝、灵宝、神器、道器、仙器、传承宝物、本命法宝等非普通物品。重点不是挑几个代表案例，而是按作品原文尽可能多收集各品阶、各修为阶段、各用途类型出现过的法宝条目，并记录它们的获取、祭炼、绑定、成长、消耗、风险和交易属性。

如果某件物品主要作为“武器形制和战斗招式”出现，优先使用 `武器分析模板.md`；如果它已经具备独立法宝效果、祭炼关系、器灵、禁制、成长或秘境钥匙属性，可在本模板中收录，并在武器文档中交叉引用。

## 必须写什么

| 模块 | 必写内容 | 项目用途 |
|------|----------|----------|
| 法宝品阶体系 | 作品内各品阶、阶段范围、已收集条目、特点和边界 | `items/artifact.json`、战斗能力分层 |
| 法宝条目清单 | 尽可能多收集各阶段法宝，写清名称、品阶、类型、获取方式、用途和限制 | 物品、Ability、剧情资产 |
| 祭炼与绑定 | 滴血、神魂、精血、本命温养、认主、器灵、禁制、传承条件 | 绑定规则、角色成长 |
| 法宝效果 | 攻击、防御、困敌、飞行、储物、隐匿、传送、破禁、保命、召唤、增幅功法 | GameplayAbility、GameplayEffect |
| 炼制与升级 | 材料、炼制手法、修复、重铸、吞噬材料、温养成长、失败风险 | 生产、材料消耗、任务 |
| 归属与交易 | 能否转移、能否被抢、是否绑定、是否引发怀璧其罪、是否只以物易物 | 经济、争夺事件 |

## 尽量写全的内容清单

- 品阶/类型：法器、灵器、法宝、古宝、灵宝、通天灵宝、玄天之宝、仙器、神器、道器、帝兵、秘宝、禁器、本命法宝、传承宝物、封印物、套装法宝。
- 形态：鼎、塔、钟、印、镜、旗、幡、葫芦、珠、瓶、图、卷、书、舟、阵盘、阵旗、傀儡、灵虫袋、储物袋/戒。
- 效果：攻杀、防御、困敌、镇压、封印、飞行、储物、隐匿、传送、破禁、保命、替死、逃遁、召唤、炼丹/炼器加成、秘境钥匙。
- 关系：认主、本命温养、神魂绑定、血祭、器灵、禁制、传承资格、反噬、损毁、修复、升级、被夺取。
- 来源：炼制、遗迹、传承、拍卖、宗门赐予、夺宝、妖兽材料炼成、天地孕育。
- 项目映射：`items/artifact.json`、`items/material.json`、`items/talisman.json`、`abilities/*.json`、`effects/*.json`。

## 不要写什么

- 不只写少数知名法宝；目标是尽可能多收集并覆盖各阶段。
- 不把别的作品的法宝品阶套进本作品事实。
- 不把普通兵器、普通护具全部塞进法宝表；无独立法宝机制的武器应放入 `武器分析模板.md`。
- 不为每件法宝创建专用 Effect；项目规则要求优先复用通用 GE/GA，由来源提供数值。
- 不把游戏数值写成原作事实；没有原文依据时必须标注“我的判断”。

## 详细程度

- 不设固定上限；按作品原文尽可能多收集，覆盖低阶、中阶、高阶、顶阶、特殊/传说级等阶段。
- 每个品阶体系必须写“阶段范围、该阶段已收集条目、特点、可游戏化方向”。
- 每个入表法宝必须写“品阶/阶段、核心用途、获取方式、限制”和“来源说明”。
- 如果某阶段材料不足，明确写“材料不足”，并列出已检索关键词、已查路径和后续检索方向。
