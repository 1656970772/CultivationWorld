# 物品设定模板

> 最后更新：2026-06-06
> 状态：模板
> 类型：可持有物品、功法秘籍、货币、材料、法宝、丹药、符箓调研
> 目标详细程度：L3

## 适用范围

用于整理作品中的可持有、可交易、可消耗、可装备或可授予能力的物品。采集时应尽可能覆盖各修为阶段、各品阶稀有度、各物品类型、各获取来源、各使用/交易/损毁场景、各绑定关系和各正负效果出现过的物品条目。它服务 `apps/game/data/items/`、`effects/`、`abilities/`、`tags/`、怀璧其罪、奖励系统和 NPC 背包。

## 必须写什么

| 模块 | 必写内容 | 项目用途 |
|------|----------|----------|
| 分类 | 货币、材料、丹药、法宝、符箓、功法、特殊物 | `category/subCategory` |
| 获取方式 | 掉落、采集、任务、交易、炼制、传承、秘境 | 任务、奖励 |
| 使用效果 | 加真气、回血、突破、保命、攻击、防御、移动 | GE/GA |
| 交易属性 | 是否可转移、价值、稀缺性、是否会引发抢夺 | covet/economy |
| 限制 | 绑定、境界门槛、次数、耐药、副作用、主角专属 | 数据约束 |

## 尽量写全的内容清单

- 货币与等价物：低/中/高/极品灵石、灵晶、贡献点、功勋、宗门令牌、拍卖保证金、特殊兑换物、以物易物凭证。
- 丹药：修炼丹、突破丹、疗伤丹、回气丹、解毒丹、延寿丹、洗髓丹、驻颜丹、补魂丹、临时爆发丹、毒丹、禁丹、丹方。
- 灵药/材料：灵草、灵果、灵花、灵木、矿石、精金、寒铁、火晶、灵液、真水、异火、妖丹、妖骨、妖皮、鳞甲、精血、魂魄、宝骨。
- 法宝/装备：飞剑、刀枪、钟鼎塔印、镜、旗幡、葫芦、珠、盾、甲胄、飞行法器、储物袋/戒、灵兽袋、阵盘、阵旗、傀儡、本命法宝、古宝、灵宝、仙器。
- 符箓/消耗品：遁地符、传送符、护身符、攻击符、防御符、隐匿符、破禁符、传音符、定位符、爆裂符、封印符。
- 功法与知识载体：功法秘籍、玉简、残篇、丹方、器方、阵图、地图、情报玉简、传承令、秘境钥匙。
- 特殊道具：主角外挂、封印物、器灵载体、魂牌、命灯、契约书、血誓媒介、傀儡核心、空间宝物、时间/因果类宝物。
- 灵宠/活体资产：灵兽蛋、灵虫卵、坐骑、契约兽、炼尸、傀儡、可交易妖兽幼体。
- 获取方式：采集、猎妖、任务奖励、交易、拍卖、炼制、继承、秘境、夺宝、宗门兑换、黑市。
- 使用效果：加真气、加修炼进度、回血、解毒、突破辅助、战力加成、保命、逃遁、储物、移动、信息传递、开门/破禁。
- 限制属性：绑定、境界门槛、血脉门槛、消耗次数、冷却、耐药、丹毒、副作用、不可转移、死亡掉落、暴露风险。
- 项目字段：`category`、`subCategory`、`grade`、`value`、`transferable`、`combatBonus`、`effects`、`grantsAbilities`、`drops`。

## 不要写什么

- 不为同一机制创建一物一 Effect；Effect 应通用，数值来自物品 spec。
- 不把“珍贵”当作 value，必须说明为什么珍贵。
- 不只列名词，不写获取、效果、消耗和限制。
- 不把不可交易的传承或血脉写成普通物品。

## 详细程度

- 可进入数据的物品至少写：id 建议、category、subCategory、grade、value 方向、transferable、effects/abilities。
- 物品效果必须优先映射到现有通用 GE/GA；没有合适机制再写“新增候选”。
- 主角专属物品必须标注“不可普适化”。
