# 势力设定模板

> 最后更新：2026-06-06
> 状态：模板
> 类型：势力、组织、宗门、家族、王朝、商会调研
> 目标详细程度：L3

## 适用范围

用于整理作品中的宗门、家族、王朝、商会、异族、邪道组织、散修联盟等势力。采集时应尽可能覆盖各修为阶段成员、各组织规模、各阵营立场、各职位层级、各资源来源、各治理制度、各外交/战争/继任场景出现过的势力样本。它服务 `factions.json`、势力 AI、外交关系、资源库存、职位制度、攻伐/结盟/贸易和继任规则。

## 必须写什么

| 模块 | 必写内容 | 项目用途 |
|------|----------|----------|
| 势力类型 | 正道、邪道、中立、妖族、王朝、商会等 | `faction.type/subtype` |
| 组织结构 | 首领、长老、弟子、执事、继承人、分支 | role、roleQuota、晋升制度 |
| 资源与产业 | 灵脉、矿脉、药园、坊市、战利品、贡赋 | 势力资源、经济循环 |
| 行为倾向 | 扩张、防御、贸易、暗杀、收徒、探索 | 势力 AI 策略 |
| 外交关系 | 敌对、盟友、附庸、交易、世仇 | 初始关系与事件触发 |
| 兴衰事件 | 创建、大战、覆灭、分裂、吞并 | 动态事件、历史、继任风险 |

## 尽量写全的内容清单

- 势力类型：正道宗门、魔道/邪道宗门、中立宗门、修仙家族、凡人王朝、商会、拍卖行、坊市、镖行、悬赏组织、情报组织、杀手组织、散修联盟、妖族族群、异族部落、学院、寺观、军团。
- 组织结构：掌门、宗主、族长、皇帝、长老、太上长老、继承人、真传/核心/内门/外门弟子、执事、护法、供奉、客卿、暗线、附庸。
- 制度：收徒、晋升、考核、贡献、月俸、任务、奖惩、禁令、叛逃处理、继任、联姻、盟约、附属供奉。
- 资源：灵脉、矿脉、药园、功法传承、丹药库存、法宝库、妖兽材料、凡俗人口、情报网、商路、军备。
- 行为：扩张、守成、贸易、结盟、攻伐、镇压、招募、培养弟子、探索秘境、举办大比、发布悬赏、暗杀渗透。
- 内部矛盾：派系、继任争夺、资源分配、长老掣肘、弟子竞争、附庸不稳、邪修渗透。
- 外部关系：敌对、同盟、附庸、贸易、世仇、互相忌惮、共同抗敌、暗中合作。
- 兴衰节点：创建、鼎盛、分裂、迁徙、覆灭、重建、吞并、被主角改变格局。

## 不要写什么

- 不按“门派百科”罗列无项目用途的小人物。
- 不把主角短暂停留过的地点误写成长期势力归属。
- 不把势力强弱只写成“很强/很弱”，必须拆成资源、战力、组织、外交。
- 不直接照搬作品中的完整规模到游戏数据；规模转化必须标注“我的判断”。

## 详细程度

- 势力总览中每个已采信或有参考价值的势力，原则上都要有势力卡；小势力也可写简版卡片。
- 每个势力卡必须有证据、资源、行为倾向、关系/冲突和参考用途。
- `可映射字段` 不是筛选条件；没有现成数据表落点时，写“剧情参考 / 世界状态参考 / 任务参考 / 待定”。
- 若用于 `factions.json` 候选，必须写出建议 `type/subtype/traits/resources/relations`，但数值用“建议区间/方向”，不要伪装成原作事实。
