# 有限视角与叙事日志模板

> 最后更新：2026-06-06
> 状态：模板
> 类型：玩家有限视角、观察信息、日志文本和叙事呈现调研
> 目标详细程度：L2；涉及 UI/日志规则时提高到 L3

## 适用范围

用于整理作品如何让读者或角色通过有限信息理解世界：传闻、目击、日志、战报、榜单、宗门公告、道听途说和误判。采集时应尽可能覆盖各修为阶段、各观察距离、各信息来源、各可信度层级、各日志/公告/传闻形式、各误判后果和各玩家可见性边界。它服务玩家日志、事件描述、确认/消息/传闻分级、UI 信息密度和叙事反馈。

## 必须写什么

| 模块 | 必写内容 | 项目用途 |
|------|----------|----------|
| 信息呈现 | 亲眼所见、听闻、公告、榜单、玉简、战报 | LogPanel |
| 可信度文本 | 确认、消息、传闻、谣言、误导 | 日志分级 |
| 叙事颗粒度 | 玩家/NPC 应知道多少，不知道什么 | 有限视角 |
| 反馈文本 | 战斗、突破、死亡、秘境、拍卖、抢夺如何写 | UI 文案 |
| 遮蔽机制 | 迷雾、隐匿身份、假名、隐藏修为、保密 | 交互边界 |

## 尽量写全的内容清单

- 信息文本来源：宗门公告、悬赏榜、拍卖预告、坊市传闻、战报、命灯、魂牌、玉简、弟子汇报、商会消息。
- 可信度表达：亲眼所见、确认、有人传来消息、听说、传闻、疑似、谣传、被证实、被否定。
- 可见信息：地点、势力、NPC、境界、物品、资源、事件阶段、关系、伤势、死亡、财富暴露。
- 不可见信息：隐藏修为、真实身份、背包详情、秘境内部、远方战争、敌方意图、假消息。
- 日志事件：突破、死亡、抢夺、复仇、师徒互动、宗门大比、拍卖、秘境开启、妖兽尸骸、世界状态变化。
- UI 启示：日志短句、信息分级、按天归档、传闻过期、可点击定位、来源显示、可信度颜色。
- 项目映射：LogPanel、WorldNews content、INFO_RECEIVED、事件弹窗、地图迷雾、玩家神识。

## 不要写什么

- 不把全知旁白当玩家已知信息。
- 不写大段文学化文案而没有信息结构。
- 不让日志暴露不该知道的隐藏状态。
- 不忽略消息来源和可信度。
