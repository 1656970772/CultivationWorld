# 建筑设施与场所功能模板

> 最后更新：2026-06-06
> 状态：模板
> 类型：宗门、坊市、洞府、功能建筑和场所规则调研
> 目标详细程度：L3

## 适用范围

用于整理作品中场所不只是地图点，而是提供功能、消耗资源、触发行为的设施。采集时应尽可能覆盖各修为阶段可进入的场所、各势力/组织设施、各生产交易建筑、各修炼战斗地点、各秘境/禁地功能和各开放条件。它服务建筑布局、任务地点、修炼室、交易、宗门设施、动态事件定位和 UI 展示。

## 必须写什么

| 模块 | 必写内容 | 项目用途 |
|------|----------|----------|
| 设施类型 | 洞府、任务殿、藏经阁、丹阁、器阁、坊市等 | 地图/建筑 |
| 功能 | 修炼、兑换、交易、发布任务、传送、疗伤 | Action/Job |
| 使用条件 | 身份、贡献、灵石、权限、境界、时间 | precondition |
| 产出/消耗 | 灵气、丹药、功法、情报、资源、稳定度 | economy |
| 风险 | 被袭击、抢夺、禁制、维护成本、资格竞争 | risk/event |

## 尽量写全的内容清单

- 宗门设施：山门、主殿、任务殿、执法堂、藏经阁、丹房、器房、阵法堂、灵兽园、药园、矿脉、修炼室、洞府、护宗大阵。
- 市场设施：坊市、拍卖行、商会、黑市、客栈、情报楼、悬赏阁、传送阵。
- 野外设施：古修洞府、秘境入口、传承碑、试炼塔、禁制阵眼、灵脉节点、妖兽巢穴。
- 功能：修炼加速、兑换物品、领取任务、购买情报、传送、疗伤、炼丹炼器、收徒考核、职位晋升。
- 条件：贡献、灵石、职位、境界、令牌、宗门许可、时间窗口、关系。
- 维护：阵法消耗、灵石维护、守卫、稳定度、资源库存、被攻破后果。
- 项目映射：地图地点、dynamic event pos、quest location、building function、Action targetResolver。

## 不要写什么

- 不只写建筑名称，要写它能让角色做什么。
- 不把设施当纯装饰。
- 不硬定地图坐标，除非项目数据需要。
- 不忽略进入权限和维护成本。
