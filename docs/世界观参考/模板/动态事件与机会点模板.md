# 动态事件与机会点模板

> 最后更新：2026-06-06
> 状态：模板
> 类型：秘境、拍卖、宝物出世、强者陨落等事件窗口调研
> 目标详细程度：L3

## 适用范围

用于整理会在世界中产生时间窗口、消息传播、NPC 准备和参与行为的事件。采集时应尽可能覆盖各修为阶段、各区域尺度、各势力参与方式、各机会类型、各消息传播速度、各准备周期和各失败/错过后果的事件样本。它服务 `dynamic-events.json`、`dynamic-goals.json`、`opportunities.json`、Job/Toil、信息传播和奖励风险。

## 必须写什么

| 模块 | 必写内容 | 项目用途 |
|------|----------|----------|
| 事件类型 | 秘境、拍卖、大比、宝物、陨落、战争、兽潮 | event.type |
| 生命周期 | 预告、准备、开启、参与、结算、过期 | phase |
| 信息传播 | 谁知道、何时知道、可信度 | awareness |
| 参与条件 | 境界、令牌、队伍、距离、关系、资源 | goal/action |
| 收益风险 | 奖励、死亡、抢夺、关系变化、后续事件 | reward/risk |

## 尽量写全的内容清单

- 事件类型：秘境开启、宗门大比、拍卖会、宝物出世、强者洞府、妖王陨落、高手遗泽、兽潮、宗门战争、通缉悬赏、飞升通道。
- 生命周期：scheduled、announced、active、resolved、expired，对应原作中的预兆、传闻、开启、争夺、结算、余波。
- 参与者：散修、宗门弟子、长老、商会、邪修、妖兽、主角、旁观势力。
- 准备：购买丹药/法宝/符箓、打探情报、组队、取得令牌、赶路、隐藏身份。
- 机会点：尸骸、宝物、洞府、拍卖品、灵药成熟、弱者暴露财富、宗门大战战利品。
- 收益：物品、功法、材料、境界突破、关系、人情、声望、职位、情报。
- 风险：死亡、重伤、被抢、被追杀、队友背叛、事件过期、资源不足、误信传闻。
- 项目映射：`eventType`、`announceDay/startDay/endDay`、`scope`、`value`、`riskKey`、`dynamic goal`、Job/Toil。

## 不要写什么

- 不把一次剧情写成无时间窗口的静态设定。
- 不只写奖励，不写参与门槛和失败后果。
- 不忽略消息传播；没人知道就不会参与。
- 不把主角独享机缘默认开放给所有 NPC。
