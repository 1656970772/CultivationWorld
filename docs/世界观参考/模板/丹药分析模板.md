# 丹药分析模板

> 最后更新：2026-06-06
> 状态：模板
> 类型：丹药、药剂、毒丹、禁丹与炼丹体系调研
> 目标详细程度：L3

## 适用范围

用于分析单部作品中的丹药、药剂、灵液、药散、毒丹、禁丹、延寿药、突破丹、疗伤药和炼丹体系。重点不是挑关键丹药，而是按作品原文尽可能多收集各修为阶段、各功效类型、各炼制体系出现过的丹药条目，并记录材料、丹方、炼制条件、适用阶段、副作用、耐药/丹毒、交易和游戏化效果。

如果某种资源主要是药材、灵草、妖兽材料或经济资源，可同时放入 `物资产出与消耗模板.md` 或 `物品设定模板.md`；本模板侧重“服用/使用后产生效果”的丹药机制。

## 必须写什么

| 模块 | 必写内容 | 项目用途 |
|------|----------|----------|
| 丹药品阶体系 | 作品内丹药层级、适用修为、该阶段已收集条目和特点 | `items/pill.json`、Effect 分层 |
| 丹药条目清单 | 尽可能多收集各修为阶段、各功效类型丹药 | 药品库、修炼、战斗、经济 |
| 功效分类 | 修炼、突破、疗伤、回气、解毒、延寿、洗髓、补魂、爆发、伪装、毒丹 | 通用 GameplayEffect |
| 材料与丹方 | 灵药、妖丹、精血、矿材、药引、火种、炉鼎、丹方来源 | `items/material.json`、生产任务 |
| 炼制条件 | 炼丹师、丹炉、火候、失败风险、炸炉、成丹率、品相 | 生产系统、职业行为 |
| 副作用与限制 | 丹毒、耐药、透支潜力、境界限制、服用间隔、禁忌体质 | 平衡约束、风险事件 |
| 交易属性 | 稀缺性、宗门供应、拍卖、以物易物、被抢风险、低阶普及度 | 经济、任务、市场 |

## 尽量写全的内容清单

- 功效：修炼增益、突破、疗伤、回气、解毒、延寿、驻颜、洗髓、补魂、强化肉身、短期爆发、伪装/易容、毒丹、禁丹、兽丹、灵兽饲养丹。
- 阶段：凡俗/入门、低阶、中阶、高阶、顶阶、仙界/神界、道丹/法则丹、特殊传说级。
- 材料：灵药、灵草、灵果、妖丹、妖核、精血、魂魄、矿材、药引、水源、火种、丹炉、丹方。
- 过程：采药、催熟、配伍、炼制、温养、开炉、失败、炸炉、成丹、品相、杂质、丹毒。
- 使用：直接服用、外敷、喂养灵兽、临战爆发、闭关辅助、突破辅助、救命、交易。
- 项目映射：`items/pill.json`、`items/material.json`、`effects/*.json`、`abilities/*.json`、经济和任务配置。

## 不要写什么

- 不只写突破丹或主角常用丹；低阶补给、疗伤、解毒、毒丹、饲兽丹也要尽量收集。
- 不把丹药效果写成专用 GE；项目规则要求通用 GE + 来源数值。
- 不把成功率、增益数值、持续时间写成原作事实；没有原文依据时标注“我的判断”。
- 不把其他作品的丹药品阶、材料和丹毒规则套入本作品事实。

## 详细程度

- 不设固定上限；目标是覆盖低阶、中阶、高阶、顶阶、特殊/传说级等阶段和各类功效。
- 每个阶段必须尽量收集条目，并说明该阶段丹药的适用修为、材料门槛和经济稀缺度。
- 每个入表丹药必须写“品阶/阶段、功效类型、功效、主要材料、限制/副作用”和“来源说明”。
- 如果某阶段材料不足，明确写“材料不足”，并列出已检索关键词、已查路径和后续检索方向。
