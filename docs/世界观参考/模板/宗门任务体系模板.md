# 宗门任务体系模板

> 最后更新：2026-06-06
> 状态：模板
> 类型：任务、贡献、奖励、晋升与组织运行调研
> 目标详细程度：L3

## 适用范围

用于整理宗门或组织如何发布任务、分配资源、考核弟子、奖励贡献、组织试炼和维持秩序。采集时应尽可能覆盖各修为阶段、各弟子/职位层级、各任务类型、各贡献奖励、各失败惩罚、各组织规模和各临时/长期任务链。它服务 `quest-templates.json`、势力经济、晋升制度、宗门大比、动态事件和 NPC 任务行为。

## 必须写什么

| 模块 | 必写内容 | 项目用途 |
|------|----------|----------|
| 任务类型 | 采集、猎妖、巡逻、护送、探索、战争、炼丹炼器 | 任务模板 |
| 接取条件 | 境界、职位、贡献、队伍、时间窗口 | GOAP 前置、Job |
| 奖励体系 | 灵石、贡献、丹药、功法、职位、关系 | 经济、晋升 |
| 失败后果 | 处罚、死亡、降级、关系变化 | 记忆、稳定度 |
| 组织闭环 | 任务如何反哺资源、声望和成员成长 | 势力 AI |

## 尽量写全的内容清单

- 日常任务：采药、采矿、巡逻、看守山门、打理药园、照料灵兽、跑腿送信、维护阵法、杂役。
- 战斗任务：猎妖、清剿邪修、追杀叛徒、护送、守矿、边境防御、宗门战争、通缉悬赏。
- 探索任务：秘境试炼、古修洞府、遗迹发掘、地图绘制、禁地侦查、传承考核。
- 生产任务：炼丹、炼器、制符、布阵、修复法宝、上交材料、培育灵药、饲养灵兽。
- 社交/组织任务：收徒、传功、考核、宗门大比、联姻、外交、情报收集、卧底渗透。
- 条件：境界、职位、贡献、声望、队伍人数、时间窗口、前置任务、物品门票、宗门许可。
- 奖励：灵石、贡献、丹药、功法、法宝、洞府、职位、指导、人情、声望、秘境资格。
- 失败后果：罚款、扣贡献、降级、受伤、死亡、关系变差、宗门稳定下降、被追责。

## 不要写什么

- 不写大型网游式任务面板，除非原作有类似制度证据。
- 不直接套用 F/S/A 等等级，除非明确标注为项目化抽象。
- 不只列奖励，不写资源从哪里来、如何消耗。
- 不把宗门任务和散修悬赏混为一类。

## 详细程度

- 每类任务至少写：发布者、目标、条件、奖励、风险、证据。
- 可转化任务必须能映射到 Action、Job、Toil 或 Quest 模板。
- 奖励数值只写相对梯度，除非原文有明确数值。
