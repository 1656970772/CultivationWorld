# 世界观参考调研模板目录

> 最后更新：2026-06-06
> 状态：已启用
> 类型：模板导航
> 适用范围：`docs/世界观参考/<作品名>/` 下的小说调研 Markdown，以及跨作品综合分析文档。

## 定位

本目录存放世界观参考资料的写作模板。新增或重写某部小说的调研文档时，先在本目录选择对应模板，再复制结构到 `docs/世界观参考/<作品名>/`。

这些模板的目标不是写小说百科，而是为当前项目的整体设计提供可追溯参考：NPC AI、势力制度、关系网、物品/GAS、动态事件、信息传播、经济循环、妖兽生态、平衡验证和 Wiki 设定确认。

通用采集原则：每个模板都应在 `适用范围` 前段明确“尽可能覆盖各修为阶段、各身份/势力/职业/场景等维度”的要求；具体维度按模板主题调整，避免只摘录主角线、名场面或少数高阶案例。

## 通用写作规则

所有调研文档都必须区分三类内容：

| 分类 | 含义 | 写法 |
|------|------|------|
| 原作事实 | 能被原文或 StoryGraph 依据支持的内容 | 写入“原作事实”或事实表，附证据 |
| 我的判断 | 基于原作事实抽象出的游戏化设计启示 | 明确标注“我的判断”，说明依据 |
| 待核验 | 暂时没有足够证据、可能串书或需要复查的内容 | 在对应正文、表格或卡片中标注“待核验”，不得当成结论 |

每个主题文档至少保留以下元信息：

```md
# 《作品名》主题名

> 最后更新：YYYY-MM-DD
> 状态：草案 / 原文已复核 / 待复核 / 废弃
> 资料来源：`docs/世界观参考/作品名/作品名.txt`、`.storygraph/storygraph.db`
> 对应模板：`docs/世界观参考/模板/xxx模板.md`
> 适用范围：说明本文覆盖什么、不覆盖什么
```

## 写全规则

模板中的“尽量写全的内容清单”是写作时的检索范围。写正文时应尽量把作品中出现过、且对项目设计有参考价值的内容列出来：

- 能找到证据的内容，写入正文或表格，并附来源说明。
- 只找到间接线索的内容，写入对应正文、表格或卡片并标注“待核验”，不要当成结论。
- 作品中明显没有的类型，可写“未见可靠证据”，避免空白让后续读者误以为漏查。
- 数值、概率、权重、可数据化字段若不是原文明示，必须标注为“我的判断”。
- 优先写全“类型、触发、条件、资源、行为链、后果、项目映射”，少写纯设定辞藻。

## 总览与卡片规则

总览表只是索引，不是正文替代。凡是已经进入总览表、并且有证据或明确待核验价值的条目，都应尽量继续展开成对应的完整卡片：

- 不以文档长度限制卡片数量；写得长没有问题，遗漏有价值条目才是问题。
- 不以“当前有没有可映射系统”作为筛选条件；没有现成系统时，写“待定 / 剧情参考 / 世界状态参考 / 任务参考 / 日志参考”等。
- 总览表中的每一行原则上都要有对应卡片；如果信息不足，也应写“简版卡片”，说明已知事实、缺失字段和待核验证据，而不是只停留在总览表。
- 模板里出现“至少”“代表”“核心”等字样时，表示最低要求，不表示只写这些；一旦调研已经找到更多条目，就继续补卡片。
- 卡片可以服务游戏机制、剧情、任务、日志、世界状态、势力关系、后续 Wiki 或 ADR，不必只服务运行时数据表。
- 如果某条总览内容最后判断不值得展开，应在对应卡片备注或“弃用/不采信说明”里写明原因，例如证据不足、疑似串书、只是一句无后续影响的背景名词。

不适合逐条写普通卡片的模板，也不能让表格成为唯一正文：

- `README.md` 的模板清单必须落到具体模板文件和使用规则，不承载作品事实正文。

## 详细程度分级

| 等级 | 用途 | 要求 |
|------|------|------|
| L1 摘要 | 快速了解该作品有没有相关素材 | 3-8 条要点，说明主要资料来源 |
| L2 可参考 | 可用于系统设计讨论 | 有分类表、机制链路，结论分事实/判断 |
| L3 可转化 | 可进入 Wiki 草案、ADR、数据设计、剧情/任务/日志设计 | 总览条目尽量全量展开成卡片；每张卡有触发条件、对象、过程、结果、边界、参考用途和证据 |

默认目标是 L2；涉及当前项目已实现或即将实现的系统时，目标提高到 L3。

## 禁止事项

- 不写没有来源的“印象流设定”。
- 不把主角光环、独有外挂、一次性奇遇直接抽象为普适规则。
- 不把不同作品的术语混在同一作品事实里。
- 不粘贴大段原文；只写路径、章节/行号、短事实摘要。
- 不把游戏化数值、权重、概率伪装成原作事实。
- 不把世界观参考文档直接当作项目已敲定 Wiki；进入 Wiki 前仍需用户确认。
- 不用“指纹”“golden”一类摘要一致性话术证明行为合理。

## 与项目文档的关系

- 可确认为项目设定的结论进入 `docs/worldbuilding/wiki/`。
- 影响系统边界的结论进入 `docs/decisions/` ADR 或 `docs/systems/`。
- 影响 JSON 配置的结论对齐 `apps/game/data/` 与 `docs/data/data-config-rules.md`。
- 仅作为素材的内容保留在 `docs/世界观参考/<作品名>/`。
