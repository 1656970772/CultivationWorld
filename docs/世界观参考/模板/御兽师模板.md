# 御兽师模板

> 最后更新：2026-06-06
> 状态：模板
> 类型：御兽、驯兽、灵宠、虫群与妖兽契约调研
> 目标详细程度：L3

## 适用范围

用于整理作品中的御兽师、驯兽师、灵宠、坐骑、战兽、灵虫、兽群控制、契约、血脉培养和妖兽资源。采集时应尽可能覆盖各修为阶段、各妖兽等级、各契约/驯养方式、各培养资源、各战斗/运输/生产用途、各反噬风险和各职业势力样本。它服务妖兽数据、NPC 职业、宠物/坐骑、战斗召唤、资源产出和任务设计。

## 必须写什么

| 模块 | 必写内容 | 项目用途 |
|------|----------|----------|
| 职业类型 | 御兽师、驯兽师、灵虫师、兽栏管理者 | NPC 副业 |
| 兽类对象 | 妖兽、灵兽、坐骑、灵虫、战兽、守山兽 | creature 数据 |
| 获取方式 | 捕捉、孵化、契约、购买、宗门赐予、血脉培养 | 任务与经济 |
| 控制机制 | 契约、禁制、喂养、血脉压制、神识控制 | AI 关系 |
| 产出用途 | 战斗、巡逻、移动、材料、侦查、守护、繁育 | 系统映射 |
| 失败后果 | 反噬、失控、逃逸、噬主、族群报复、污染 | risk、事件 |

## 尽量写全的内容清单

- 御兽对象：灵宠、坐骑、守山兽、战兽、灵虫、妖禽、毒虫、兽群、特殊血脉。
- 控制方式：契约、血契、禁制、驯化、喂养、血脉压制、驭虫术、兽牌、笼舍。
- 成长机制：幼体、血脉、进阶、饲料、灵药、战斗磨炼、繁育、变异。
- 使用场景：战斗、护山、运输、侦查、采集、追踪、献祭、材料产出。
- 社会关系：御兽宗门、兽场、灵宠市场、妖兽猎捕、禁忌驯化。
- 项目映射：妖兽族群、NPC 关系边、宠物系统、坐骑移动、任务、资源掉落、事件风险。

## 不要写什么

- 不把所有妖兽都写成可驯化对象。
- 不忽略妖兽自主性、反噬和族群关系。
- 不把主角专属灵宠奇遇当作通用御兽机制。
- 不只写战斗用途，也要记录移动、侦查、生产和社会价值。
