# 人物关系与事件分析模板

> 最后更新：2026-06-06
> 状态：模板
> 类型：关系网、事件因果与关系驱动行为调研
> 目标详细程度：L3

## 适用范围

用于整理角色之间的师徒、道侣、同门、血亲、恩义、仇怨、利益同盟、主仆、灵宠、人妖关系，以及这些关系如何通过事件产生、强化、背叛或消亡。采集时应尽可能覆盖各修为阶段、各身份差距、各势力阵营、各亲疏关系、各利益冲突、各恩怨来源和各关系结局。它服务 `RelationshipSystem`、记忆、复仇、护短、报恩、师徒互动和动态事件。

## 必须写什么

| 模块 | 必写内容 | 项目用途 |
|------|----------|----------|
| 关系类型 | 关系名、方向、强度、是否对称 | relationship edge types |
| 建边事件 | 关系因何产生 | eventBindings、记忆 |
| 行为后果 | 护短、复仇、报恩、背叛、继承遗志 | 关系驱动目标 |
| 关系衰减 | 是否稳定、是否会随利益改变 | decay、strength |
| 事件因果 | 一件事如何引出下一件事 | 动态事件、时间线 |

## 尽量写全的内容清单

- 人际关系：师徒、道侣、血亲、同门、好友、盟友、竞争者、宿敌、恩人、仇人、主仆、上下级、债主/欠债者。
- 势力关系：同盟、附庸、贸易、世仇、联姻、共同抗敌、暗线渗透、背叛、吞并、归顺。
- 人妖/异族关系：灵宠、坐骑、契约兽、妖兽仇敌、领地威胁、族群报复、化形合作、半驯化共生。
- 建边事件：拜师、救命、赠宝、背叛、羞辱、击杀亲友、同门初始化、联姻、交易、共同探索、战争。
- 行为后果：护短、传功、报恩、复仇、追杀、背叛、求援、继承遗志、争夺继承、放过或加害。
- 关系属性：方向、是否对称、强度、好感、衰减、是否公开、是否秘密、是否受势力约束。
- 项目映射：RelationType、eventBindings、memory、obsession、dynamic goal、relationship_target。

## 不要写什么

- 不只写“某某与某某关系很好/很差”。
- 不把读者情感判断当作关系强度。
- 不遗漏关系方向；恩人/仇人/师徒往往不是完全对称。
- 不把没有行为后果的社交关系过度设计。

## 详细程度

- 关系类型总览中的每类关系、关键关系边中的每条重要关系，原则上都应继续展开成关系/事件卡片。
- 每条有参考价值的关系都要写：from、to、type、触发事件、后续行为、公开/秘密状态和证据。
- 没有现成关系系统落点时，写“剧情参考 / 任务参考 / 叙事日志参考 / 待定”，不能只停留在总览表。
- 复杂关系要标明冲突事实和待核验点。
