# 统一经济交易底座

> 最后更新：2026-06-07
> 关联规格：`docs/superpowers/specs/2026-06-07-统一经济交易底座设计.md`

统一经济交易底座负责价值校验、估价、托管、原子交割、账本、债务和经济信号。交易动机仍由 AI、任务、动态事件、机会点和玩家行为产生，经济底座只在外部系统提交交易意图后负责检查、结算和留下证据。

## 设计边界

| 边界 | 说明 |
|------|------|
| 负责 | 资产读写、估价、托管、原子交割、账本记录、债务状态、经济信号输出 |
| 不负责 | 主动生成买卖目标、判断岗位授权、处理关系后果、实现区域动态供需、鉴宝真假货 |
| 接入来源 | AI、任务契约、动态事件、机会点、正式店铺、贡献兑换、材料上交、拍卖 |
| 输出对象 | 交易结果、托管记录、债务记录、分层账本、AI 可消费经济信号 |

## 模块

| 模块 | 职责 |
|------|------|
| `EconomicSystem` | 统一门面，组合估价、托管、交割、账本、债务、信号和抽象拍卖 |
| `TransactionEngine` | 接收交易请求，编排资产校验、估价、交割、账本和债务创建 |
| `AssetAdapter` | 读写 NPC inventory、势力 state 资源和组织点数 |
| `PriceService` | 按物品 `value`、场景倍率和私人交易修正估价 |
| `SettlementService` | 执行原子交割，失败时按 delta 回滚已发生的资产变化 |
| `EscrowService` | 托管、释放、返还和罚没资产 |
| `TransactionLedger` | 保存分层可见的交易证据 |
| `DebtService` | 维护长期债务、偿还、宽免、争议与逾期状态 |
| `EconomicSignalProvider` | 输出 AI、任务、关系、Job/Toil 可消费的经济信号 |
| `AuctionService` | 在玩家不实际到场时抽象结算拍卖 |

## 交易流程

```text
交易意图来源
  -> EconomicSystem / TransactionEngine
  -> AssetAdapter 校验资产
  -> PriceService 估价
  -> EscrowService 锁定或释放托管资产
  -> SettlementService 原子交割
  -> TransactionLedger 写入分层账本
  -> DebtService 创建或推进债务
  -> EconomicSignalProvider 输出信号
```

正式交易默认走托管与原子交割。私人交易可以不用托管，也可以通过押金、见证人或第三方托管降低风险；违约时由底座生成债务或争议信号，但不替 NPC 做追债、报复或审判决策。

## 资产规则

| 资产 | 结算规则 |
|------|----------|
| 灵石、材料、丹药、法宝、功法秘籍 | 可交易、可抵押、可托管、可赔偿，读写实体 inventory |
| 势力 `low_spirit_stone`、`food`、`disciples` | 遵守 ADR-015，以势力 `state` 为单一真相源 |
| 贡献、战功、宗门信用 | 组织点数，只能由所属组织发放、扣除、冻结和兑换 |

组织点数不可私下转让、借贷或抵押。经济底座在结算时应拒绝把组织点数当作普通私人交易资产。

## 正式交易与私人交易

| 类型 | 默认规则 | 典型场景 |
|------|----------|----------|
| 正式交易 | 托管安全、资产校验、原子交割、机构账本可见 | 拍卖行、坊市正规店铺、宗门兑换、任务堂备案任务、悬赏阁 |
| 私人交易 | 可出现估价偏差、违约和债务，账本默认个人可见 | 私人委托、拍后私下交易、荒野交易、黑活、人情交换 |

正式交易的稳定性来自托管和交割规则，不来自交易动机本身。私人交易的风险会被记录成账本、债务或经济信号，后续由关系、任务、声望、岗位/职权、调查/审判系统消费。

## 托管与债务

托管资产必须真实从原持有者资产容器移出，进入交易实例或托管方账户。成功时划转给目标方，失败或流拍时返还，违约时按场景规则罚没或赔付。

债务只记录事实和状态，不直接生成行为。到期未还时，`DebtService` 将债务标记为逾期并输出信号，后续追债、封禁、担保牵连、通缉或追杀由外部系统决定。

## 拍卖

拍卖作为一等交易场景接入统一底座。玩家角色实际到场参与拍卖会时，后续可展开完整拍卖流程；玩家不在场时，`AuctionService` 执行抽象竞价结算，只记录参与者、赢家、成交价、流拍、财富曝光、账本和后续机会信号。

重要 NPC 到场但玩家不在场时仍走抽象结算。系统不逐轮叫价，但成交和交割必须经过同一套托管、估价、账本和债务规则。

## 验证原则

经济底座验证必须观察真实资产变化、账本记录、债务状态和模拟行为，不使用指纹或哈希作为正确性证据。重点观察：

- NPC inventory 与势力 state 资源不会重新出现双轨覆盖。
- 正式交易能校验资产、托管、交割并写入账本。
- 失败交割能回滚已发生的资产变化。
- 私人交易、任务短款和违约能生成债务或争议信号。
- 抽象拍卖能产生成交、流拍、财富曝光和账本。
