# 系统设计：稳定度系统

> 最后更新：2026-05-23

## 概述

稳定度是防止"雪球效应"的核心机制——大势力不会无限扩张。

## 稳定度计算

每天 Tick 时重算每个势力的稳定度：

```
稳定度变化 = 基础恢复 - 领地负担 - 战争消耗 + 资源补益 + 世界状态修正
```

### 因素

| 因素 | 影响 |
|------|------|
| 领地面积过大 | 稳定度下降（维持成本高） |
| 正在打仗 | 稳定度下降 |
| 刚占领新领地 | 稳定度下降（新领地不稳） |
| 资源充足 | 稳定度缓慢恢复 |
| 掌门忠诚度高 | 稳定度恢复加成 |
| 世界状态"瘟疫" | 稳定度全面下降 |
| 世界状态"大旱" | 稳定度下降（粮食危机） |

## 稳定度后果

| 稳定度范围 | 状态 | 效果 |
|-----------|------|------|
| 70-100 | 稳定 | 正常运作 |
| 40-69 | 动荡 | 发展速度下降，结盟困难 |
| 30-39 | 危险 | 可能触发叛乱事件 |
| 10-29 | 崩溃边缘 | 高概率分裂、弟子叛逃 |
| 0-9 | 瓦解 | 势力可能直接消亡 |

## 设计目的

- 大势力扩张越多 → 稳定度越难维持 → 最终分裂或收缩
- 小势力虽然弱 → 但稳定度高 → 有生存空间
- 产生历史感：帝国兴起 → 过度扩张 → 内乱 → 衰落 → 新势力崛起
