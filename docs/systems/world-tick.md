# 系统设计：世界演化 Tick

> 最后更新：2026-05-27

## 概述

每消耗 5 行动点（1 天），WorldEngine 执行一次 Tick，按固定顺序推进世界状态。

## Tick 执行流程（模板方法模式）

```
第1步：updateWorldModifiers()
  │  - 世界状态的产生（按概率）
  │  - 强度变化、到期消退
  │
第2步：runFactionDecisions()
  │  - 每个势力的 NPC 领袖根据性格 + 状态做决策
  │  - 输出行为意图（扩张/防御/结盟/攻伐/发展/无动作）
  │
第3步：triggerEvents()
  │  - 规则引擎匹配当前条件
  │  - 预设事件检查触发条件
  │  - 生成事件列表
  │
第4步：executeEvents()
  │  - 攻伐结算（双方资源/地形/弟子对比）
  │  - 领地变更
  │  - 资源变动
  │  - NPC 伤亡
  │
第5步：updateStability()
  │  - 根据领地面积、资源、战争状态重算稳定度
  │  - 触发稳定度过低的后果（叛乱/分裂）
  │
第6步：updateRelations()
  │  - 根据当天事件调整势力间好感度
  │  - 战争 → 好感度暴跌
  │  - 共同敌人 → 好感度上升
  │
第7步：propagateInfo()
  │  - 更新所有信息事件的传播半径
  │  - 势力获知新信息
  │  - 玩家收到传播到位的信息
  │
第8步：updateNPCs()
     - 推进具名核心 NPC 年龄
     - 按寿元上限与自然死亡曲线处理 NPC 死亡
     - 掌门继任
     - 无继任候选时势力覆灭，领地转为无主地
```

## 执行保证

- 每步完成后才执行下一步，保证数据一致性
- 整个 Tick 在 Web Worker 中执行，不阻塞 UI
- Tick 完成后将结果（世界快照 diff）发送回主线程
