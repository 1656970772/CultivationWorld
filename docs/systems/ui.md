# 系统设计：UI 面板设计

> 最后更新：2026-05-23

## 概述

UI 使用 DOM 元素覆盖在 Canvas 之上，通过 EventBus 与其他模块通信。

## UI 布局

```
┌─────────────────────────────────────────────┐
│ 状态栏：天数 | 行动点 (3/5) | 坐标 (50,50) │
├─────────────────────────────┬───────────────┤
│                             │               │
│                             │   日志面板     │
│        Canvas 地图           │   ┌─────────┐ │
│                             │   │ [确认]   │ │
│                             │   │ 青云宗...│ │
│                             │   │ [传闻]   │ │
│                             │   │ 听说...  │ │
│                             │   └─────────┘ │
│                             │               │
├──────────┬──────────────────┴───────────────┤
│ 小地图   │ 操作栏：[打坐] [存档] [设置]      │
└──────────┴──────────────────────────────────┘
```

## 各面板职责

### 顶部状态栏（StatusBar）
- 当前天数
- 行动点（剩余 / 总量）
- 玩家坐标
- 当前世界状态图标（如魔气上涨时显示图标）

### 右侧日志面板（LogPanel）
- 滚动显示已知信息
- 传闻用灰色斜体，确认信息用正常样式
- 按天分组
- 可折叠/展开

### 事件弹窗（EventDialog）
- 居中显示
- 事件描述文本
- 选项按钮列表
- 每个选项标注行动点消耗
- 弹出时暂停地图交互

### 存档面板（SavePanel）
- 可收起的侧边栏
- 存档列表（名称 + 天数 + 时间戳）
- 新建存档 / 读取存档 / 删除存档
- 导出 / 导入 JSON

### 底部小地图（Minimap）
- 缩略版全局地图
- 标注玩家当前位置
- 标注已知势力范围
- 点击可快速定位

### 底部操作栏
- 打坐按钮（弹出选择消耗行动点数的输入）
- 存档按钮（打开存档面板）
- 设置按钮（后续扩展）
