# 系统设计：时间与行动系统

> 最后更新：2026-05-23

## 核心机制

- **1 行动点 = 移动 1 格**（沼泽消耗 2 点）
- **5 行动点 = 1 天**
- **事件处理消耗 1-N 天**（取决于事件规模）
- **打坐：** 玩家主动选择消耗 N 个行动点快进时间（如 10 点 = 2 天）

## 流程

```
玩家行动（移动/选择/打坐）
    │
    ▼
消耗行动点
    │
    ▼
行动点耗尽？──否──▶ 等待下次行动
    │
    是
    ▼
触发世界 Tick（1 天）
    │
    ▼
重置行动点为 5
    │
    ▼
如果事件消耗了多天（N 天），连续触发 N 次 Tick
```

## 多天事件处理

当玩家参与一个消耗 3 天行动点的事件时：
1. 扣除 15 个行动点（3 × 5）
2. 世界连续演化 3 天
3. 在这 3 天中，世界照常运转（其他势力在行动、信息在传播）
4. 产生"你在忙这边，那边已经变天了"的效果
