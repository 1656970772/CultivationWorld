# ADR-033：世界模拟自迭代优化流程（模拟 → 诊断 → 调参 → 验证 闭环）

最后更新：2026-06-01

状态：已采用（作为平衡调优与"让世界运行起来"的标准工作流程；v3/v4 已按此流程产出）

## 背景

项目目标是「让整个修仙世界自动运行起来」——世界按规则自动演化（宗门兴衰、势力攻伐、修士成长、恩怨叙事），玩家选择性介入。但"活起来"长期是一个**感性目标**，缺乏可证伪的判据：

- 体检报告 `历史体检结论（已并入当前文档）`（P2 节）已点名"高层晋升/复仇/情绪涌现偏弱"，但没有量化门槛。
- 历史 tuning v1/v2（`历史调参记录（已清理，结论见 ADR-033~037）`）证明：默认默认关闭不改变既有行为态下 3000 天会出现"叙事冷启动"（0 攻伐 / 0 突破 / 0 复仇 / 0 势力覆灭），但每轮调参的目标、归因和验收都散落在各文档里，缺乏统一方法论。
- 5000 天全激活态基线进一步暴露：人口从 151 单调衰减到 38，世界趋向"空城化"——这是比"叙事弱"更根本的问题（死掉的 NPC 不会复仇/夺权）。

需要一套**可复用、可证伪、低风险**的工作流程，把"让世界运行起来"从感性目标变成工程化迭代。

## 决策

确立标准迭代循环：**跑基线 → 采集 → 诊断 → 归因 → 备份 → 最小调参 → 重跑验证 → 记录 → 回归保护**，并配套一组量化 KPI 与归因决策树。流程全文见 `docs/balance/simulation-iteration-process.md`。

### 一、"世界活起来"的量化定义（KPI）

把目标拆成四个维度、可证伪的判据（5000 天全激活态）：

| 维度 | 关键指标 | 健康判据 | 失败信号 |
|------|----------|----------|----------|
| 人口可持续 | 末态存活 / 初始；出生 vs 死亡；性别比 | 存活 ≥ 60%；出生 ≥ 死亡×0.5；0.5≤女/男≤2 | 单调衰减、单性别枯竭 |
| 权力流动 | 势力覆灭；高层席位补位/挑战 | ≥ 1 | 全员 100% 存活 |
| 叙事涌现 | 复仇 PvP 击杀；执念分布种类；峰值心魔/愤怒 | 击杀 ≥1；流派 ≥4；峰值 >0 | 链路通但执念不行动 |
| 健康度 | GOAP 回退率；任务/猎妖失败率 | 回退 <50%；失败 <35% | 规划层失效、自杀式接单 |

原则：**先定义指标，再调参**——没有指标的调参是盲调。

### 二、迭代纪律

1. **一次只撬一类杠杆**：同时改多个参数无法归因。
2. **先上游后下游**：人口枯竭时先修"为什么不补人"，而非先调复仇。
3. **结构问题不要用参数硬调**（见决策三）。
4. **每轮必留回滚点**：改前备份到 `历史备份（已清理）：pre-tuning-vN-*`。

### 三、参数问题 vs 结构问题（归因决策树）

```
某 KPI 长期为 0 或单调恶化
   ├─ 改 1 个参数后线性变化？ → 参数问题，继续撬数据杠杆
   └─ 改参数无响应 / 链路根本不存在？ → 结构问题（改代码/数据契约，而非数值）
         ├─ 行为/实体根本没被创建？（如招募不生成个体）
         ├─ precondition 死锁？（如 hasAdjacentEnemy 永不满足，见 v1）
         └─ 链路断在某一环？（如复仇执念有了但不行动，见 v2/v4）
```

历史教训固化为证据：
- v1 撬数据杠杆（weight/阈值）→ 攻伐 0→775。
- v2 修结构链路（攻战真正杀 NPC + 写记忆）→ 攻伐 775→10166、复仇执念 0→2。
- v4 P1 调阈值无响应 → 证伪"调阈值就能覆灭"，逼出"抽象资源 vs 真实个体不同步"的真病根（ADR-034 §未解问题）。

## 数据与接口

- 离线模拟入口：`apps/game/tools/simulate-analysis.mjs [天数]`，产物 `tools/report-data.js`（被 `report.html`/`report-3000.html` 读取）。
- 激活态开关（仅内存覆盖、不写回 JSON，便于对照实验）：
  - `UTILITY_ACTIVE=1`：utility/reward/obsession.goalMult。
  - `INFO_ACTIVE=1`：news/opportunities/reward/covet。
  - `RELATIONSHIP_GOALS_ACTIVE=1/0`：关系驱动 Goal/妖群/领地（强制开/关）。
- 推荐基线命令（全激活态 5000 天）：

```powershell
cd apps/game
$env:UTILITY_ACTIVE="1"; $env:INFO_ACTIVE="1"; $env:RELATIONSHIP_GOALS_ACTIVE="1"
node tools/simulate-analysis.mjs 5000
```

- 快速指标提取（无需打开 HTML）：`node -e` 读取 `report-data.js` 的 `summary`/`deaths`/`timeline`/`goap`/`worldEvents` 字段聚合死因与人口曲线（命令模板见流程文档 §2）。
- 回归保护：每轮调参后跑 `tools/test-*.mjs`，关键看 GOAP 旧摘要回归 `5740e12a` 是否漂移；纯数据/选择层调参应保持摘要不变。

## 后果

- 把"让世界运行起来"变成可证伪、可追踪的工程目标，任何后续 Agent 都能按同一流程接力。
- 每轮迭代都建立在上一轮的发现上，避免重复盲调；"失败"的迭代（如 v4 P1）也有价值——它证伪假设、逼出真病根。
- 配套 KPI 表使"叙事涌现"等抽象诉求有了验收门槛，降低过度调参与回归风险。
- 流程本身数据驱动 + 可回滚，符合项目"数据驱动 / 易扩展 / 低耦合"的设计原则。

## 验证

- v3（ADR-034 §出生率）按本流程产出：出生 35→93，末态人口 38→44，方法论首次端到端走通。
- v4（ADR-034 §接单自检/覆灭阈值）按本流程产出：任务死亡显著下降、末态人口 44→54；P1 按归因树正确识别为结构问题并顺延 v5。
- 两轮 GOAP 旧摘要回归均保持 `5740e12a`，证明流程在"撬数据杠杆/选择层"时能守住默认关闭不改变既有行为红线。
