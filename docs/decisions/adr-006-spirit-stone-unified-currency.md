# ADR-006: 灵石多品阶统一计价方案

> 日期：2026-05-28
> 状态：已采纳

## 背景

灵石作为修仙世界核心货币，按设定分为四个品阶：

| 品阶 | ID | 兑换比（以低级为基准） |
|------|-----|----------------------|
| 低级灵石 | `low_spirit_stone` | 1 |
| 中级灵石 | `mid_spirit_stone` | 100 |
| 高级灵石 | `high_spirit_stone` | 10,000 |
| 极品灵石 | `top_spirit_stone` | 1,000,000 |

在模拟运行中，所有经济活动（势力产出/消耗、贸易、掠夺、NPC 任务奖励）都涉及灵石加减运算。如果每处逻辑都要处理四种灵石的兑换、找零、优先扣除，会导致：
- GOAP 规划需要做多资源背包问题，复杂度过高
- 每个 ActionExecutor 都要实现货币换算逻辑
- 数值平衡变得困难

## 决策

采用 **"内部统一计价，展示自动换算"** 方案：

1. **内部存储**：所有灵石以低级灵石数量作为统一单位存储在 `inventory` 和 `state` 中
2. **运算逻辑**：经济活动全部用单一数字的加减法，不区分品阶
3. **展示换算**：通过 `formatSpiritStones(amount)` 工具函数，将数字自动换算为最高面额优先的可读格式（如 `152340` → `"1高级 52中级 40低级"`）
4. **品阶门槛**（未来扩展）：当高阶活动需要特定品阶时，只需检查总值是否达到该品阶对应的 exchangeRate 即可

## 实现位置

- 工具函数：`apps/game/js/core/constants.js` → `formatSpiritStones()`
- 品阶定义：`apps/game/data/definitions/macro-resources.json 或 data/items/`（currency 类型的 `exchangeRate` 字段）
- UI 接入：`apps/game/js/simulation-main.js`（势力列表 + 势力详情面板）

## 后果

- **正面**：现有经济逻辑零改动；GOAP 规划保持简单；大数字自动变可读
- **负面**：无法模拟"特定品阶灵石稀缺"的场景（如高级灵石紧缺但低级灵石充裕）
- **未来扩展**：如需品阶稀缺性，可在 inventory 中增加独立的高阶灵石槽位，仅用于特殊活动
