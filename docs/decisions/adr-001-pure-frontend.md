# ADR-001：采用纯前端单体架构

> 日期：2026-05-23
> 状态：已决定

## 背景

项目需要选择技术架构：纯前端 vs 前后端分离。

## 决策

采用**纯前端单体架构**，所有逻辑在浏览器中运行。

## 考虑的方案

### 方案一：纯前端（选定）

- 技术栈：原生 JS + Canvas 2D + IndexedDB
- 部署：静态文件
- 世界演化在 Web Worker 中计算

### 方案二：前后端分离（放弃）

- 前端渲染 + Node.js 后端运算
- 服务器存档

## 选择理由

1. 第一版核心目标是验证"世界演化 + 玩家干预"的玩法，纯前端最快到达
2. 固定格子地图的规则演算可放在 Web Worker 中执行；当前已扩展为 300×300。
3. IndexedDB 满足多存档需求
4. 零部署成本
5. 核心引擎是纯 JS，后续需要后端时可整体迁移

## 风险

- 存档只在本地，换设备丢失（通过导出/导入 JSON 缓解）
- 超大规模地图可能有性能瓶颈；当前 300×300 需依赖视口裁剪、寻路优化和模拟工具观察性能。

## 后果

- 不需要服务器
- 存档管理在客户端
- 后续如需多人模式，需要引入后端
