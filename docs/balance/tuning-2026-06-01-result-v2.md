## 4. 显著改进与未解问题

### ✅ v2 解决了

1. **攻战真正杀 NPC**：链路通了
2. **attacked 记忆被触发**：16+ NPC 写记忆
3. **复仇执念首次出现**：2 个 NPC 持有复仇执念
4. **"加强防御"占比 60% → 0.04%**：v1 改 weight + v2 加前置条件 = 彻底解决
5. **修真大爆发**：突破 0 → 33（灵根分布微调 + 突破概率微调）
6. **统计 bug 修复**：历史最大心魔从 0（瞬时值）→ 45.5（真实值）
7. **修代码 bug**：`simulate-analysis.mjs` 之前没从 `tick.deaths` 收集死亡，现在收集了

### ❌ v2 未解决（v3 计划）

1. **复仇 PvP 击杀 0**：
   - 链路通了（slain 17 + 复仇执念 2）但执念没去追仇人
   - 根因：`act_npc_hunt_enemy` 的 precondition 太严 / GOAP 选目标时复仇执念被其他需求压制
   - v3 计划：让复仇执念的 goalState 优先级在 `decorateGoalConsiderations` 中放大（timeValue、emotion.anger 调制），让 NPC 真的去追仇人

2. **NPC 死亡过多（219 死亡 vs 29 出生）**：
   - 净减 157 → 151 → -6 (62 存活)
   - 死亡主因：monster 138 + slain 17 + quest 40 + quest_hunt_failed 22
   - v3 计划：让 `act_npc_explore` 在境界低时更频繁触发，但**不接超过境界的高风险任务**（已经实现 50%）；需要进一步修 NPC 任务自检战力

3. **势力覆灭仍 0**：
   - 攻战杀 NPC 但弟子数仍 > winDefMin=5
   - v3 计划：让 winDefMin 跟随"实际活 NPC 数"动态计算

## 8. 结论

**v2 是一次成功的结构性调优**——通过让 `attackEnemy` 真正杀 NPC、写 `attacked` 记忆、修复分析脚本的死亡收集，**首次打通了"攻战→致死→记忆→复仇执念"的完整链路**。

但链路到复仇执念就断了：**执念没去追仇人**。这是 v3 任务。

同时 v2 引入的副作用是死亡过多（219 vs 64）。需要 v3 平衡出生率。
