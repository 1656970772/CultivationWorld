## 1. 背景

v1 调优后攻伐 0→775、贸易 0→481、执念 30→42、新死因 power_struggle 涌现——但**复仇 PvP 仍 0**。3000 天只见到 1 例 power_struggle 致死，**但 cause 写 "power_struggle" 而非 "slain"**，所以复仇链路最后一公里没通。

## 2. v1 后根因分析

经过代码层调研：

1. **`attackEnemy` 走的是"势力 vs 势力"宏观博弈**：扣 `disciples` 数字（winDefLoss=0.08），但**从来不杀 NPC**。3000 天 775 次攻战 → 0 个 NPC 写 `_deathInfo.cause='slain'`。
2. **`attacked` 记忆类型已定义**（memory.json: `intensity: 45, decay: 0.4, grudgeGain: 25`），但**全代码库 0 处调用** `recordMemory('attacked', ...)`。
3. **`acquired.revenge` 规则只覆盖 `sect_destroyed / betrayed / humiliated`**：全代码库搜索 `recordMemory('attacked', ...)` 0 命中 → 即使加了 `memoryType: attacked` 规则也永远触发不了。
4. **`act_defend` 无 precondition**：v1 改了 weight 1+stability +2，但仍是 fallback 主力（60% 占比）。

**死锁**：
- 攻战不杀 NPC → 没人写 _deathInfo
- 没人写 _deathInfo → `_collectDeaths` 收不到死亡
- 没有死亡 → 没人写 attacked 记忆 → 没人触发复仇执念 → 复仇 PvP 链路 0

## 3. v2 调优方案

### 3.1 引擎代码改动

#### `tick-manager.js` `attackEnemy` 胜利时（Line 761 附近）

新增：
1. **实际杀 NPC**：从战败方活 NPC 池里随机选 `npcKillCount` 个，按顺序：
   - 写 `state.set('alive', false)`, `entity.alive = false`
   - 写 `_deathInfo = { cause: 'slain', killerId: null, killerName: '势力战争', killerFactionId: attackerId, npcId, npcName, factionId, ageYears, maxAgeYears, rankName, day, x, y, locationName }`
2. **给战败方所有活 NPC 写 `attacked` 记忆**：`recordMemory('attacked', { factionId: attackerId, tick, location })`
3. **战败方 stability 进一步下调**（-15 改成 -20），让"被攻击"的实质影响更明显

#### `faction-actions.js` `FactionAttackExecutor` 不变（delegation）

#### `faction-actions.js` `FactionDefendExecutor` 加 stability 修正

不动 JSON effects，但 executor 加 `stability` 自校验（防止 stability > 70 时还选 defend）。

### 3.2 数据改动

#### `combat.json`：加战斗致死参数

```diff
+ "winNpcKillRatio": 0.005,    // 战败方弟子 × 此比例 = 杀 NPC 数
+ "winNpcKillMax": 2,          // 单次攻战最多杀 2 个 NPC（防人口雪崩）
+ "winNpcKillMinDefender": 50, // 战败方弟子 < 50 不杀（小势力保护）
+ "winNpcKillStabilityFloor": 30, // 战败方 stability < 30 时必杀 1 个
```

#### `faction-actions.json` `act_defend`

加 `preconditions: { stability: { op: 'lt', value: 70 } }`——和平时期不防御。

#### `obsession.json` `acquired.revenge`

加规则：
```json
{
  "type": "revenge",
  "name": "复仇",
  "memoryType": "attacked",
  "minMemoryIntensity": 30,
  "intensity": 75,
  "goalState": { "enemyKilled": { "op": "eq", "value": true } }
}
```

#### `cultivation.json` 灵根分布

```diff
"heaven":  { "weight": 1 } → 3
"dual":    { "weight": 9 } → 15
```

让天才比例从 1% → 3%，**150 NPC 中约 4-5 个天才**可能在 3000 天内破境。

#### `simulate-analysis.mjs` GOBT 心智统计

跟踪**历史最大心魔**（而非瞬时值），修复 3000 天衰减归 0 的统计 bug。

```diff
- let maxAnger = 0, maxInnerDemon = 0;
+ let maxAnger = 0, maxInnerDemon = 0;
+ let maxAngerHistory = 0, maxInnerDemonHistory = 0;
// tick 内
+ for (const ent of allNpcs) {
+   const e = ent.emotions;
+   if (e) {
+     maxAngerHistory = Math.max(maxAngerHistory, e.snapshot().values.anger || 0);
+     maxInnerDemonHistory = Math.max(maxInnerDemonHistory, e.snapshot().values.inner_demon || 0);
+   }
+ }
```

## 4. 回归保证

- 黄金指纹可能漂移（因为攻击致死路径变了，action plans 后续步骤会受影响）
- 端到端稳定性：
  - 18 势力 100% 存活（**必须**）
  - NPC 净增长 ≥ -10（**必须**）
  - 复仇 PvP 击杀 > 0（**期望**）

## 5. 风险评估

| 风险 | 概率 | 缓解 |
|------|------|------|
| 攻战杀 NPC 让世界人口雪崩 | 中 | `winNpcKillMax=2` + `winNpcKillMinDefender=50` 双重护栏 |
| 修心魔记忆触发复仇过频繁 | 低 | `minMemoryIntensity: 30` + `attacked` decay 0.4 会自然衰减 |
| act_defend 加前置让势力无法防御 | 低 | stability < 70 仍然覆盖大多数场景（攻战胜利 -5，失败 -10） |
