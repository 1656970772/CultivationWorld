- `data/needs/npc-needs.json` + `js/engine/npc/npc-entity.js`（散修生计需求 + needIds 白名单修复）

**最重要的工程教训**：全激活态下 `score = (priority+Σdelta) × Π(mult) × Π(consideration)`，

今后新增任何"应当能插队"的 Goal，必须同步检查 `utility.json` 里它的 consideration 基线。

## 3. 迭代记录

### 批次 A：妖兽行为（grade/family/type）✅

**对标基准**（`docs/世界观参考/凡人修仙传/妖兽与修士关系分析.md`）：
妖兽应有极强**领地意识**、栖息于**巢穴**、群居物种**成群活动**、低阶**勤于觅食**、高阶**镇守守护资源**。

**病症**：基线妖兽 96.5% 时间纯 `wander`、`hunt` 仅 1%、行为种类仅 3，像随机游走的质点，毫无领地/巢穴/觅食差异。

**归因**（按 ADR-033 决策树）：
- 对照实验：senseRange 翻 3-4 倍 → hunt 从 0.4%→7%（参数有效，证明"猎物稀疏"是 hunt 低的一个原因），
  但 91% 仍 wander → 确认**主因是结构性缺失**：BT 的"无猎物"分支只有单一随机 `monsterWander/monsterPatrolTerritory`，
  妖兽世界本身没有"觅食/守巢/群居"等本能行为。
- 结论：不简单扩 senseRange（会导致 NPC 被过度猎杀，本就 73% 猎妖失败），而是**让无猎物时的本能行为本身分化**。

**改动**（最小、数据驱动、零外部依赖；不触碰 hunt/flee 主链）：
1. `data/balance/monster-spawn.json` → `behaviorByTier` 各档新增 `lairChance/lairRestDays/forageChance/packCohesion`
   （归巢栖息概率随阶位提高：tier1 0.18 → tier2 0.28 → tier3 0.42，体现"越高阶越镇守老巢"）。
2. `js/engine/monster/monster-entity.js`：
   - 重写 `monsterWander`：分化出 **归巢栖息(lair)** / **群居聚拢(swarm 物种向同族靠拢)** / **觅食游猎(forage)** / 游荡(wander)。
   - 重写 `monsterPatrolTerritory`（tier2+）：分化出 **守穴(lair)** / **系统巡逻(patrol)** / 游荡。
   - 新增 `_findNearbySameSpecies` 供群居聚拢；`behaviorState` 新增 `forage/lair/patrol` 取值（仅 UI 文字消费，安全）。

**验证（200 天全激活态）**：

| 阶位 | 改造前 | 改造后 |
|------|--------|--------|
| 1-2阶 | wander 99%（3 种） | **forage 73-76% / lair 14-16% / wander 8-10%（5 种）** |
| 3-4阶 | wander 93%（3 种） | **patrol 58-62% / lair 32-36%（5 种）** |
| 5-6阶 | wander 71-85% | **lair 45-65% / patrol 28-35%（梯度合理：越高阶越守巢）** |

**回归**：黄金指纹 `5740e12a` 零漂移；妖兽资源化闭环测试全绿。

**遗留**（列入后续批次，避免一次改太多）：
- 妖兽**捕食低阶妖兽**（食物链）、**繁殖/护幼**、**化形妖兽建势力**尚未实现（需更大结构改动，后评估）。
- forage 占比偏高（>70%），后续可下调 forageChance 让 lair/wander 更均衡，但当前已远好于"96% 单一"。

### 批次 B：散修 wanderer ✅ / 妖族 demon ⏳

**对标基准**（`docs/世界观参考/凡人修仙传/角色AI行为参考.md` §三）：
- **散修求生型**：`生存 > 杀怪 > 抢劫低阶`，独来独往、外出接活闯荡，不该闷头修炼。
- **邪修/魔道型**：残忍嗜血、主动挑衅、抓活人修炼、不择手段提升修为。

**病症**：散修 行为种类仅 2、修炼 87.5%；妖族 demon 修炼 92-95%。两类都"闷头修炼"。

**归因（散修）**：**结构问题**——散修所有"产出型需求"都依赖宗门（职责/猎妖资源/上交材料/兑丹/兑法器），
无门派则全不触发，只剩 `need_npc_cultivation` 能驱动 → 散修只会修炼。

**改动（散修）**：
1. `data/needs/npc-needs.json` 新增 `need_npc_wanderer_subsistence`（散修生计）：
   仅 `isWanderer=true` 触发，goalState=`questTurnedIn`（接坊市悬赏→执行→交付赚灵石）。
2. `js/engine/npc/npc-entity.js` `_initNeeds` 默认 needIds 加入该需求（原为硬编码白名单，漏加则不加载）。
3. 调参定档 priorityBoost：34（无响应）→ 60（响应良好但执行任务偏多）→ **50（均衡）**。

**验证（200 天，priorityBoost=60 时）**：
散修 wanderer 修炼 98.5% → **修炼 71.5% / 执行任务 22.9% / 挑战上位 2.5% / 接取任务 1.9% / 交付任务 1.1%**，
行为种类 2 → 5，出现完整"接活→执行→交付"求生链。**确认是参数问题**（优先级响应），链路结构通。

**回归**：黄金指纹 `5740e12a` 零漂移（散修生计对有门派 NPC 的 satisfiedCondition 恒满足，不影响）。

**妖族 demon（遗留批次 F）**：修炼仍 92-95%。初判妖族势力的任务/猎妖/晋升流转不充分，
需单独分析妖族势力运作（涉及势力层而非个体需求），列入后续批次，避免本轮一次改太多（守 ADR-033 纪律）。

### 阶段验证：批次 A+B 合并 700 天全激活态

| KPI | 改造前基线 | 批 A+B 后 | 判断 |
|-----|-----------|-----------|------|
| 存活 NPC（700天） | — | **116** | 健康（基线 5000 天仅 38） |

**回归**：黄金指纹 `5740e12a` 零漂移；妖兽资源化 / 关系驱动 / 师徒互动 单测全绿。

> 结论：妖兽行为分化 + 散修生计两项改动，让对应类别行为显著丰富化，且长周期 KPI 不退化反而改善。

## 批次 C：关系/师徒行为（驰援/探望/传功/护徒）

**对标基准（ADR-027/028/029）**：师徒应有传功点化、护徒驰援、徒弟尽孝探望；同门遭袭应驰援；受恩应报恩。

**病症**：700 天模拟中，驰援同门 / 探望恩人 / 传功点化 / 护徒 / 探望恩师 **全部 0 触发**——整套关系互动空转。

**逐层诊断（这是本轮最深的一个 bug，链路很长，记录完整定位过程供后续参考）**：

1. **排除"系统未注入"**：诊断脚本确认 150/150 NPC 都注入了 `_relationshipSystem`/`_relationshipConfig`，`relationshipGoalsEnabled=true`。
2. **排除"无 qualifying 边"**：54 对师徒，距离 100% ≤30（同势力，距离 0-18），28% 徒弟修为<0.6 → 28% 可传功。条件满足。
3. **确认 Goal 确实产出**：强制采样 `collectExtraGoals`，产出了 `goal_teach_disciple`、`goal_visit_master`（产出环节正常）。
4. **确认 Goal 进了候选 + GOAP 可规划**：teach Goal 进前3、`act_npc_teach_disciple` 规划成功（链路结构通）。
5. **定位真正根因（两个叠加）**：
   - **根因①·plan() 命中即返回**：`behavior-system.plan()` 按 score 降序遍历，第一个能规划成功的 Goal 就 return。修炼 score 高且必成功 → 关系 Goal 即使进前3也排在修炼后、永远轮不到。
   - **根因②·consideration 乘法压垮 priority（致命）**：全激活态 `utility.json enabled=true` 给关系 Goal 挂了 `considerationsBySource` 乘子。`goal_teach_disciple` 的 `teachReadiness`（base 0.4 + slope 0.8×progress）把 **priority 78 的 score 压到 25.1**（×0.32），而修炼 `bottleneck`(inverse 曲线) 在低 progress 时≈1.0 保持高位。运行时计数实锤：关系 Goal 进 extraGoals 60 次，但只 4 次进 top3、0 次被选中。

**改动**：
1. `data/balance/relationship.json` 关系/师徒 Goal priority 提到**高于日常修炼/任务(60-70)**区间（产出本身已受低频概率门控，高 priority 不会霸占行为）：
   护徒 88 / 护短 82 / 关系复仇 80 / 传功 78 / 报恩 74 / 探望恩师 72。
2. `data/balance/utility.json` 关系 Goal 的 consideration **基线提到 0.85-0.92、slope 大幅降低**（消除乘法压制，保留极轻微"修为越高越有余力"梯度）。
3. `relationship.json` 放宽/提频：`discipleMaxTotalProgress` 0.6→0.95（传功贯穿徒弟成长期）、`teachChancePerTick` 0.06→0.12、`visitChancePerTick` 0.04→0.08。

**验证（500 天全激活态）**：

| 关系/师徒行为 | 改前 | 改后 |
|---------------|------|------|
| 传功点化 | 0 | **14** |
| 探望恩师 | 0 | **82** |
| 关系 Goal 进 top3 | 4 | **50** |

师傅角色（leader/elder）画像出现"点化徒弟"，关系 Goal 机制完全打通。

> 驰援同门 / 护徒 / 报恩仍 0：它们依赖"同门/徒弟正陷战(hasRevengeTarget)"或"benefactor 边"前置，而当前世界 PvP 击杀少(2)、恩义边少——属**世界观合理的低频**（修仙界 PvP 本不频繁），机制已通，触发前置罕见。后续若要提升，应从"增加 PvP 冲突/恩义事件"侧入手（列入观察）。

**回归**：黄金指纹 `5740e12a` 零漂移（指纹用 utility.enabled=false 基线，priority/consideration 改动不影响）；师徒互动 / 关系 Goal / 妖群领地 三个单元测试全绿。

> 关键教训：**全激活态下，高 priority 不等于高 score**。`score = (priority+Σdelta) × Π(mult) × Π(consideration)`，consideration ∈[0,1] 会乘法压低。新增任何"应当能插队"的 Goal 时，必须同时检查 utility.json 里它的 consideration 基线，否则 priority 形同虚设。

## 批次 D：复仇执念（追而不杀的老病根）

**对标基准**：修仙界恩怨深重，复仇应是"穷追不舍、不死不休、千里追杀"，追到即手刃仇人。

**病症**：复仇者追踪触发多（单轮 143-330 次），但击杀几乎为 0（仅 1-2）。"追而不杀"。

**逐层诊断**：
1. 排除链路断裂：hunt→kill 行为链、executor、`_resolveRevengeTarget`（不检查距离，仇人在世即返回）均完整。
2. **精确定位**：给 hunt/kill 加 outcome 计数，实测 **hunt 330 次中 324 次卡在 traveling（"?"）、仅 1 次真正 tracked、1 次 enemy_slain**。
3. **根因**：`movement.json` 中 **NPC 速度统一为 1 格/天**。复仇者追"仇人旧坐标"、仇人每天又移走 1 格——两个同速实体，追逐者永远差一步，几乎不可能同格（实测 330 次追踪仅 1 次追上）。这是**结构性的同速追逐困局**，非参数问题。

**改动**：
1. `js/engine/npc/npc-entity.js` `_refreshRevengeState`：复仇/夺舍执念在身时，给追击者**提速**（`revengePursuitSpeed=2`，默认），仇人失联即恢复基础速度。体现"穷追不舍"，使追击者(2格)快于仇人(1格)，数学上必能追上。
2. `data/actions/npc-actions.json` `act_npc_kill_enemy`：`requiresTravel` false→**true** + `targetResolver` self→**revenge_target**。击杀阶段持续移动到仇人**当前**坐标发动攻击，而非在 hunt 锁定的旧坐标扑空。

**验证（端到端确定性测试 `verify-revenge-pursuit.mjs`）**：

| 指标 | 结果 |
|------|------|
| avenger 初始距仇人 | 75 格 |
| 复仇执念在身提速 | **2 格/天** ✓ |
| 追击最小距离 | **0**（追上）✓ |
| 结局 | **仇人被手刃，复仇达成** ✓ |

> 改前同速下 75 格追击几乎不可能成功；改后复仇者穷追到仇人身边并击杀，破解了困局。

**回归**：黄金指纹 `5740e12a` 零漂移（指纹场景复仇者不追逐，提速不影响 GOAP 输出）；复仇 / 妖兽资源链 / 关系网 单测全绿。

> 备注：复仇行为的**总量**仍受"结仇事件源"频率影响（某些局势 enemy 边=0、无人结仇 → 无复仇者）。本批次根治的是"有仇人却追不上"的结构病根；"结仇事件是否够频繁"是另一维度（势力攻战/夺宝冲突频率），列入后续观察。

## 批次 F：妖族 demon（修炼 89-95%，已诊断未改）

**对标基准**：妖族（妖修）不靠人类宗门经济（丹药/法器/贡献），应靠**吞噬、夺取、厮杀**变强——残暴主动。

**病症**：demon 势力类型 NPC 修炼 89-95%，远高于 righteous(45%)。

**诊断（对比 demon vs righteous 的 need top goals）**：
- demon top：`修炼(70) | 战备(54) | 猎妖(44)` —— 几乎只有修炼
- righteous top：大量 `突破援助(84)` 顶替修炼，更丰富
- **根因（自我强化的死循环）**：`breakthrough_aid(84)` 需 `factionHasBreakthroughPillMaterial=true`，而该值**动态依赖势力库存**（由 NPC 猎妖上交累积）。demon NPC 的猎妖/上交需求低频不触发 → 妖族库存空 → `breakthrough_aid` 永不触发 → demon NPC 只能修炼 → 更不猎妖。
- `loyalty_duty` 也因 demon `monthlyQuota=undefined` 不触发。

**为何本轮未改（守 ADR-033 纪律）**：
正确修复是**给妖族补符合世界观的产出型需求**（类似批 B 散修生计，但应是"妖修掠夺/吞噬"而非人类经济），
这是一个**较大的新增系统**（妖族专属需求 + 行为链 + 妖修体系），涉及势力层运作。
本轮已完成 A/B/C/D 四个结构改动，不宜再撬大杠杆。**列为独立重点立项**，下次单独处理。

> 临时缓解选项（低风险，备选）：妖族势力初始 resources 补充破境丹/法器材料，让 demon NPC 走通现有经济链。
> 但这是"用人类经济兜底妖族"，不符世界观，故未采用。根治应做"妖修掠夺体系"。
