> 关联报告：`docs/reports/2026-05-30-项目分析与下一步建议.md` + 3000 天报告 `apps/game/tools/report-3000.html`

## 1. 背景与目标

体检报告（5-30）已点名 4 个 P2 风险，**3000 天模拟全部坐实**：

| 指标 | 默认态 3000 天 | 设计意图 |
|------|----------------|----------|
| 突破成功 | 0 | 正常（设计慢节奏） |
| 攻伐 | 0 | ❌ 应有 |
| 势力覆灭 | 0 | ❌ 应有 |
| 复仇 PvP | 0 | ❌ 应有 |
| 峰值心魔 | 0 | ❌ 应有 |
| 猎妖失败率 | 65.7% | ⚠️ 偏高 |
| 势力行为"加强防御" | 60.3% | ❌ 行为单一 |
| NPC GOAP 贪心回退 | 97.6% | ⚠️ 路径规划薄弱 |

激活态（`INFO_ACTIVE=1`+`UTILITY_ACTIVE=1`，1000 天）显示：
- 心魔峰值 0 → **12**（10x）
- 执念分布新增 `retire:3` `power:3`（4 流派真涌现）
- 妖兽高阶掉落 g4/g5 出现

但**复仇 PvP / 攻伐 / 势力覆灭**仍为 0 → **结构问题**，不是参数问题。

## 2. 根因分析

### 2.1 攻伐死锁

`data/actions/faction-actions.json` 的 `act_attack`：
- precondition: `stability>=40`, `disciples>=50`, **`hasAdjacentEnemy: true`**

`hasAdjacentEnemy` 要求相邻格子存在敌对势力。18 势力在 300×300 地图上分布稀疏（每势力领地约 50-100 格），**很多势力根本不相邻**。所以 `act_attack` 几乎永远进不了候选集，AI 退回到无前置的 `act_defend`。

### 2.2 复仇链路死锁

`data/balance/obsession.json` 的 `acquired.revenge` 触发条件是记忆事件 `sect_destroyed | betrayed | humiliated`。
- 攻伐 0 → 致死 0 → 记忆 0 → 复仇执念 0 → 复仇行为 0

**死锁环**。

### 2.3 行为分布失衡

`act_defend` precondition: `{}`（无门槛），stability +5（强收益），weight 2。
- 任何时候、任何势力都能选它
- 收益 = +5 稳定度 + 降 borderThreat
- 永远是 fallback

## 3. v1 调优方案（最小变更）

### 3.1 改 `faction-actions.json`

| 行为 | 改前 | 改后 | 理由 |
|------|------|------|------|
| `act_defend` | weight=2, stability+5, borderThreat-1 | weight=1, stability+2, borderThreat-2 | 降低"加强防御"的相对吸引力，分流到 attack/recruit |
| `act_attack` | precondition: `stability>=40 AND disciples>=50 AND hasAdjacentEnemy` | precondition: `stability>=30 AND disciples>=30` | **移除 hasAdjacentEnemy**，让任何势力都能选 attack，executor 自己选目标；下调门槛让中等势力也能发起 |
| `act_attack` | effects: `enemyCount-1, food-200` | effects: `enemyCount-1, food-150, low_spirit_stone+200` | 让 AI 看到 attack 的"掠夺"正反馈（灵石+200 由攻击结算时从战败方掠夺，**仅作 GOAP 效用提示**；真实结算由 `faction_attack` executor 按战力比动态算，单一真相源） |

### 3.2 改 `emotion.json`

| 触发事件 | 改前 | 改后 | 理由 |
|----------|------|------|------|
| `attacked` | anger=20, fear=15 | anger=40, fear=20, inner_demon=10 | 被攻击的 NPC 愤怒/心魔上升更明显，给"复仇执念"提供情绪基质 |

### 3.3 改 `obsession.json`（acquired.revenge）

| 触发条件 | 改前 | 改后 | 理由 |
|----------|------|------|------|
| `betrayed` | minMemoryIntensity=60 | 40 | 背叛记忆更容易转化为复仇执念 |
| `humiliated` | minMemoryIntensity=50 | 35 | 受辱更容易转化为复仇执念 |
| `sect_destroyed` | minMemoryIntensity=60 | 50 | 门派被灭记忆稍微放宽 |

### 3.4 改 `cultivation.json`（微调突破概率）

不改基础 speed，改 **`breakthrough.successRates`** 微调（仅在激活态下生效）：

| 境界 | 改前 | 改后 | 备注 |
|------|------|------|------|
| qi_refining→foundation_building | 0.60 | 0.65 | 略提 |
| foundation_building→golden_core | 0.40 | 0.45 | 略提 |

**设计意图保留**：3000 天仍以低境界为主，金丹/元婴级突破罕见。但炼气→筑基的成功率提升 → 增加"小人物破境"叙事。

## 4. 回归保证

- 黄金指纹 `f08c3248`（GOAP 计划内容零漂移）：v1 不改 actions 列表，仅改 precondition / effects 值；黄金指纹需重新验证。
- 单元测试 `test-goap-golden.mjs`、`test-bt.mjs`、`test-utility.mjs` 必须全过。
- 端到端指标：
  - **必须变好**：act_defend 占比 < 50%，攻伐次数 > 0，心魔峰值 ≥ 15
  - **不能崩**：势力存活率 ≥ 80%（防 attack 把世界打废），NPC 存活率 ≥ 70%

1. 跑 3000 天（默认零漂移模式 + 激活态 INFO_ACTIVE=1 + UTILITY_ACTIVE=1）两次对照
2. 提取关键 KPI 对比
3. 写报告 `docs/reports/2026-06-01-tuning-v1-result.md`
4. 如有副作用 → 回滚 → 进 v2
