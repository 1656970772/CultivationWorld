# 项目概览

> 最后更新：2026-06-05

## 一句话描述

网页版修仙世界沙盒模拟器。世界按规则自动演化，玩家可以观察、移动、干预事件；即使玩家不介入，宗门、NPC、妖兽、关系网和世界事件也会持续运行。

## 核心体验

打开网页后，看到一个会自己推进的修仙世界：宗门经营资源、NPC 修炼游历、师徒/道侣/宿敌关系演化、妖兽觅食巡逻、消息扩散成江湖热点。玩家是行走者和干预者，不是全知上帝。

## 当前实现范围

来源：`apps/game/data/` 与 `apps/game/js/` 当前代码。

- 地图：300×300 固定地图，共 90,000 格。
- 势力/组织：18 个，包含 12 个核心宗门/王朝/妖族势力和 6 个功能组织。
- 初始 NPC：152 个，另有婚配生育生成新 NPC 的运行时机制。
- 妖兽：36 条定义，按一至九阶、族群、行为档位参与模拟。
- AI：四层反应式 AI，包含 Reaction、Utility/Intent、GOAP、Execution。
- 行为：修炼、疗伤、游历、任务、上交材料、兑换/服用物品、复仇、夺宝、师徒、关系驰援、动态事件准备/参与等。
- 世界事件：世界修正器、信息传播、机会点、怀璧其罪、动态世界事件。
- 战斗机制：GAS 风格的 Tag / Attribute / Effect / Ability / 统一伤害管线。
- 编辑器：Web 数据编辑器 + Tauri 2 桌面壳，用于读取、校验、保存 JSON 数据。

## 入口与运行

| 目标 | 入口 | 运行方式 |
|------|------|----------|
| 游戏 Web 端 | `apps/game/index.html` | `apps/game` 下执行 `npm run serve` 或 `python serve.py` |
| 自动模拟面板 | `apps/game/simulation.html` | 同游戏 Web 端服务 |
| 编辑器 Web 端 | `apps/editor/data-editor.html` | `apps/editor` 下执行 `npm run editor:serve` 或 `python serve.py` |
| 编辑器桌面端 | `apps/editor/src-tauri/` | `apps/editor` 下执行 `npm run editor:dev` / `npm run editor:build` |

## 核心关键词

- **叙事沙盒**：世界自动演化，玩家选择性介入。
- **规则驱动**：运行时不依赖 AI API，主要由 JSON 配置和规则代码推进。
- **有限视角**：神识、消息传播和机会点让信息逐步扩散。
- **数据驱动**：运行时规则、机制资产和初始实体集中在 `apps/game/data/`。
- **可验证模拟**：平衡和行为改动以真实、多种子、长程模拟的行为数据为依据。

## 非目标

- 不做多人联机。
- 不把普通网页交互动画当作 Remotion 视频资产。
- 不让编辑器替代游戏运行时；编辑器只负责数据生产与校验。
