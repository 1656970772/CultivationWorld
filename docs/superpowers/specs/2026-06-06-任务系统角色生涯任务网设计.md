# 任务系统角色生涯任务网设计

> 最后更新：2026-06-06
> 状态：设计草案已确认，待用户审阅书面规格后进入实施计划
> 类型：任务系统规格
> 关联文档：`docs/systems/event-system.md`、`docs/worldbuilding/wiki/rules/sect-operation.md`、`docs/decisions/adr-009-wanderer-bounty-and-location-event-log.md`、`docs/世界观参考/宗门运行流程与制度平衡分析.md`

## 一句话定义

任务系统不是“任务板刷新器”，而是 **角色欲望与社会契约系统**：任务由具名 NPC 或组织岗位基于职责、利益、恐惧、资源缺口、仇怨、人情或派系压力发布，再由真实实体接取、准备、移动、执行、交付、争功和追责，并把结果回写到关系、声望、恶名、派系支持、公开记录和后续任务链。

## 设计目标

1. **任务必须有发布者**：原则上每个任务都能追溯到具名 NPC 或组织岗位，而不是由抽象系统凭空刷新。
2. **任务服务三层体验**：底层驱动世界运转，中层塑造 NPC 生涯，上层允许玩家旁观、亲自参与、间接干预或高阶发布任务。
3. **全程实体模拟优先**：任务接取、准备、组队、移动、执行、遭遇、交付和争议尽量发生在真实地图、真实实体、真实库存和真实关系中。
4. **任务是社会契约**：每个任务都应有契约形式、失败惩罚、违约惩罚和背叛惩罚；角色可以拒绝履约或拒绝受罚，但不遵守本身会继续生成剧情。
5. **结果不止经济奖励**：任务会改变人情、威望、恶名、派系支持、关系记忆、公开记录和后续因果链。
6. **真相不自动公开**：系统记录真实事件，但角色认知和公开记录可以不一致；真相可以调查，但不保证被揭露。

## 范围与回退原则

当前规划以 **全程实体模拟** 为目标。NPC 接任务后，应真实前往发布点或接触发布者，准备资源，选择独行或组队，移动到目标地点，执行采集、战斗、护送、调查、谈判、潜入等行为，最后返回交付或进入失败、违约、争议阶段。

如果后续发现世界规模、性能或实现复杂度不可控，再退回 **分级混合模拟**：低价值日常任务可抽象结算，玩家关注任务、重要 NPC 任务、宗门大事、秘境、战争、复仇和高风险任务仍保留关键节点或全程实体模拟。该回退是工程策略，不改变本规格的世界观目标。

## 核心模型

### 发布者

第一版采用 **分级开放发布者**。

1. **组织岗位发布者**
   - 示例：掌门、长老、执事、外务堂、执法堂、炼丹堂、悬赏阁管事、坊市管事、商会掌柜。
   - 典型任务：采集、巡守、值守、劳役、斩妖、调查、护送、追捕、矿脉镇守、宗门调令。
   - 约束：岗位必须有职权边界、奖励来源、备案能力和追责能力。

2. **具名 NPC 发布者**
   - 示例：弟子、散修、炼丹师、伤者亲友、仇家、队友、派系成员。
   - 典型任务：求丹、求药、求材料、求法器、寻人、救人、报恩、复仇、护道、秘境组队、夺宝协作、调停纠纷。
   - 约束：NPC 必须知道自己的需求，有传播渠道，有可支付报酬或可承诺人情，并愿意承担曝光、失败或违约后果。

未来预留 **所有 NPC 都可发布任务**，但需要受信息、资源、胆量、关系、身份和传播渠道限制。

### 动机来源

任务来源不是“系统刷新”，而是发布者的动机触发。

- **资源缺口**：缺丹药、缺材料、缺法器、缺护道者、缺情报。
- **职责压力**：执事堂维持巡守，外务堂处理妖兽，执法堂追捕叛徒，炼丹堂筹集材料。
- **关系压力**：亲友失踪、师徒求援、欠人情、遭背叛、需要调停。
- **利益机会**：发现秘境、洞府线索、灵脉、稀有妖兽、稀有材料。
- **危机事件**：兽潮、战争、敌袭、天灾、宗门内乱、队伍失踪。
- **派系斗争**：拉拢、打压、试探忠诚、抢功、破坏对手任务。

势力需求、世界事件、秘境开启、战争爆发等不会直接生成无主任务，而是先改变某些角色或岗位的处境，再由他们发布任务。

## 可见性与接取

任务不是所有人都能看见。第一版支持三种可见性。

1. **公开任务板**
   - 发布到宗门任务堂、悬赏阁、坊市公告、商会委托栏。
   - 适合公共事务、低隐秘度任务、常规悬赏和组织备案任务。
   - 接取条件以资格、修为、声望、信誉、贡献、职业能力和契约接受度为主。

2. **定向委托**
   - 发布者指定某个人或某类人，例如可信弟子、炼丹师、同派系成员、高战力修士、曾经帮过自己的人。
   - 适合私人求助、护道、炼丹、秘密调查、报恩、复仇。

3. **关系传播**
   - 任务通过师徒、好友、道侣、派系、流言、情报商、目击者扩散。
   - 适合高价值机缘、复仇计划、秘境线索、黑活委托和隐秘政治任务。
   - “知道任务本身”就是一种资格。

接取判断不只看修为，还应考虑：

- 关系与信任。
- 声望、恶名和信誉。
- 派系立场。
- 职业能力，例如炼丹、炼器、阵法、侦查。
- 是否拥有前置情报。
- 是否接受契约、押金、担保、道誓或保密约束。

## 执行者模式

任务按目标决定执行者结构。

- **单人任务**：求药、送信、低阶采集、私人跑腿。
- **小队任务**：护送、斩妖、救援、秘境探索。
- **多人竞争任务**：悬赏首杀、秘境名额、资源点争夺。
- **协作竞争混合任务**：共同进秘境或共同抗敌，但对宝物、功劳、传承存在分配冲突。

系统应允许同一任务中出现队长、队员、临时盟友、竞争者、旁观者、担保人、雇主和追责者。协作竞争任务要能产生同生共死、抢功、背刺、分赃不均、替人作证、队长威望等关系记忆。

## 生命周期

任务是活的，不是发布后固定不变。

1. **草拟**：发布者产生需求，但尚未公开。
2. **发布**：进入公开任务板、定向委托或关系传播。
3. **接取**：执行者接受任务，可形成单人、小队或竞争关系。
4. **准备**：购买补给、找队友、调查情报、申请外出、签契约。
5. **执行**：移动到地点，采集、战斗、护送、调查、谈判或潜入。
6. **交付**：提交物品、带回目标、报告情报、确认击杀或完成护送。
7. **结算**：发放奖励，记录功劳，更新关系、声望、恶名、派系支持。
8. **争议**：发生抢功、谎报、私吞、失败追责、违约辩解时进入争议阶段。
9. **后续**：生成复仇、救援、追债、调查、审判、报恩、派系站队等任务链。

发布者可以根据世界变化调整任务：

- **撤销**：需求消失、目标死亡、发布者死亡或资源不足。
- **过期**：时间窗口关闭，如秘境关闭、护送错过期限。
- **加价**：任务卡住、危险升级、发布者急需。
- **追加目标**：调查变清剿，护送变救援，斩妖变追踪妖王。
- **转化任务**：失败变追责，失踪变救援，违约变追债，背叛变追杀。

## 契约与追责

契约形式按任务风险、隐秘程度和发布者能力选择。

- **口头委托**：靠关系和信誉约束。
- **任务堂备案**：组织记录，影响贡献、声望、接取资格。
- **押金/担保**：冻结灵石、贡献或牵连担保人。
- **血契/道誓/魂契**：高价值、隐秘、危险任务，违约代价极重。

每个任务都应声明三类后果：

1. **失败惩罚**
   - 适用：执行者努力执行但失败。
   - 示例：受伤、押金损失、好感下降、发布者损失、任务目标逃脱。

2. **违约惩罚**
   - 适用：接取后放弃、超时、不交付、私吞目标物。
   - 示例：信誉下降、赔偿、禁接任务、派系打压、公开谴责。

3. **背叛惩罚**
   - 适用：主动坑害发布者、杀护送对象、私通敌方、夺宝灭口。
   - 示例：通缉、追杀、宗规审判、结仇、恶名上升、逐出宗门。

角色可以拒绝接受惩罚，例如拒赔、跑路、投靠敌对势力、找靠山说情、杀人灭口。系统不应因此停住，而应继续生成追债、追杀、庇护、调停、审判等任务。

## 奖励与关系回写

任务奖励分为硬收益和软收益。

### 硬收益

- 灵石、贡献点、战功。
- 丹药、材料、法器、功法线索、秘境资格。
- 宗门库存、坊市库存、发布者个人库存的资源转移。
- 修炼阅历、战斗经验、职业经验。

### 软收益

软收益是任务核心收益之一。

- **人情**：救命之恩、护道之恩、炼丹之恩、保密之恩。
- **威望**：宗门威望、区域名望、同辈声望、职业声望。
- **恶名**：狠辣、背信、杀人夺宝、邪修名声、威慑力。
- **派系支持**：某长老派、执法堂、外务堂、真传阵营、散修圈子支持度。
- **关系记忆标签**：同生共死、夺宝之仇、背弃委托、替我作证、抢我功劳。

软收益采用 **数值 + 标签并存**：

- 数值用于 AI 决策、接取门槛、UI 展示和系统判断。
- 标签用于叙事记忆、特殊任务触发、关系反应和长期命运线。

## 功劳、分配与争议

功劳和收益分配采用 **混合规则 + 允许造假**。

- 基础奖励按任务契约发放。
- 战利品、声望、功劳按真实贡献、队伍约定、发布者裁定、派系影响分配。
- 系统区分 **真实贡献** 与 **公开记功**。
- 允许私吞、谎报、抢功、压功、冒领。
- 争议可触发申诉、调查、作证、报复、派系站队、宗门审判。

真实贡献和公开记功可以不一致。谁真的出了力、谁被记功、谁被冤枉、谁成功私吞，都可以成为后续任务和关系变化的来源。

## 真相、认知与公开记录

真相分三层。

1. **真实事件层**
   - 引擎记录实际发生了什么。
   - 包含真实贡献、真实击杀、真实背叛、真实目击者、真实物品流向。

2. **角色认知层**
   - 每个角色只知道自己看见、听说、推断或被告知的内容。
   - 角色认知可以错误、不完整或被欺骗。

3. **公开记录层**
   - 任务堂、悬赏阁、坊市流言、宗门审判或历史日志承认什么。
   - 公开记录可以与真实事件不一致。

真相可以通过调查、目击者、证据、法术、情报网逐步揭露，但不保证一定揭露。错过真相、冤屈未解、私吞成功、无名死亡都允许成为世界的一部分。

## 任务链

任务链采用 **预设链 + 动态因果链并存**。

### 预设链

用于表达宗门大事、兽潮、战争、秘境、主线式区域事件。

```text
兽潮前兆 -> 边境调查 -> 斩妖清剿 -> 兽潮防线 -> 妖王追杀
```

### 动态因果链

由任务结果自然生成。

- 护送失败 -> 货主索赔 / 仇怨 / 调查。
- 斩妖未尽 -> 妖兽迁徙 / 袭村 / 加价悬赏。
- 秘境夺宝成功 -> 同行者嫉恨 / 外人追杀 / 分赃争议。
- 救人成功 -> 人情债 / 拜师线 / 派系拉拢。
- 任务违约 -> 追债 / 追杀 / 担保人受牵连。
- 抢功成功 -> 被压功者申诉 / 私下报复 / 派系站队。

## 玩家关系

任务网应能在玩家不介入时自行运行。

玩家可以：

- **旁观**：通过神识、日志、传闻、任务记录观察世界任务网。
- **亲自参与**：低阶时像普通修士一样接任务、组队、交付、承担失败和违约后果。
- **间接干预**：资助某人、加价悬赏、泄露情报、安排护道者、保人、暗中破坏。
- **发布和调度**：身份提高后以宗门、岗位、派系或个人名义发布任务，调配资源和人手。

任务系统应支持玩家身份变化：低阶时为任务奔波，中阶时组队、委托、经营人脉，高阶时发布任务和影响资源流向。

## 首批任务类型与分期

规划覆盖私人委托和组织任务，但实现分阶段推进。

### 第一阶段：资源求取与关系事件

优先验证“人发布任务”这条主线。

1. **资源求取**
   - 求丹：突破前缺丹药，委托炼丹师或悬赏材料。
   - 求药/求材料：缺灵草、妖兽材料、矿石。
   - 求法器：战斗前需要武器、防具、护身符。
   - 代购/交换：从坊市、商会、宗门库存获取资源。

2. **关系事件**
   - 救人：亲友失踪、重伤、被妖兽围困。
   - 寻人：外出未归、秘境失联、逃债跑路。
   - 报恩：发布者用资源、人情、引荐回报恩人。
   - 复仇：因杀亲、夺宝、背叛、抢功产生委托。
   - 调停：两个 NPC 或派系发生争执，第三方调停。
   - 护送亲友：护送低阶亲友、伤者、商队、重要证人。

### 第二阶段：机缘争夺

- 秘境组队。
- 洞府线索。
- 夺宝协作。
- 截杀竞争者。
- 争夺妖兽材料、灵脉节点、传承名额。
- 协作后分赃、背刺、抢功、追责。

### 第三阶段：派系政治

- 拉拢某 NPC。
- 打压竞争者。
- 试探忠诚。
- 破坏对手任务。
- 争夺职位。
- 宗门审判站队。
- 长老或真传派系发布隐秘委托。

### 组织岗位任务贯穿三阶段

- **执事堂**：采集、巡守、值守、劳役、任务备案。
- **外务堂/悬赏阁**：斩妖、护送、追捕、调查、清剿。
- **执法堂**：追捕叛徒、调查违约、审判取证。
- **炼丹堂/炼器堂**：材料收集、成品交付、职业委托。
- **掌门/长老**：秘境名额、宗门战争、派系调度、高阶调令。

## 第一版可实现规格

第一版实现时，应聚焦最小闭环：

1. 任务有发布者。
2. 任务有可见性。
3. 任务有契约、失败惩罚、违约惩罚、背叛惩罚。
4. 任务有阶段状态。
5. 任务由实体执行。
6. 任务能写入关系、声望、恶名、记忆标签。
7. 任务能因成功、失败、违约、背叛生成后续任务。

## 依赖系统

任务系统以角色生涯任务网为核心，但依赖周边系统。当前先做规划，不要求一次实现完。

1. **关系与记忆系统**
   - 好感、仇怨、人情债、恩义、背叛、同生共死、抢功。
   - 任务结算和争议必须能写入关系与记忆标签。

2. **声望/恶名系统**
   - 区域声望、宗门声望、散修圈声望、职业声望。
   - 恶名不只是负面，也可能带来威慑、邪修机会、黑市委托。

3. **派系系统**
   - 宗门内部长老派、真传派、执法堂、外务堂等支持度。
   - 任务可改变派系支持，也可由派系暗中发布。

4. **岗位/职权系统**
   - 明确谁能代表组织发布任务、备案任务、兑现奖励、追究责任。
   - 岗位资源和组织资源要区分，但可授权动用。

5. **资源与库存系统**
   - 发布者必须有报酬来源。
   - 任务物品、战利品、押金、赔偿、私吞都要真实转移。

6. **信息传播与认知系统**
   - 谁知道任务、谁知道真相、谁听到流言。
   - 任务可通过公开栏、定向委托、关系网络、情报商传播。

7. **组队与协作竞争系统**
   - 小队成员、队长、贡献记录、战利品分配、背刺和争功。
   - 同一任务支持多人协作、竞争或协作竞争混合。

8. **调查/审判/追责系统**
   - 真相与公开记录不一致时，允许调查、作证、取证、裁决。
   - 失败、违约、背叛能转化为追债、追杀、调停、审判任务。

## 验证方式

后续实施验证应使用真实模拟观察行为本身，避免用字节一致性结论自证。

建议验证点：

- 具名 NPC 或组织岗位能基于资源缺口发布任务。
- 任务通过公开、定向或关系传播被合适角色看见。
- 执行者能接取、准备、移动、执行和交付。
- 成功结算能真实转移资源，并写入关系、声望、恶名或记忆标签。
- 失败、违约、背叛能触发不同后果，并生成后续追责或关系任务。
- 真相层、角色认知层、公开记录层可出现差异。
- 玩家不介入时，任务网仍能自行演化。

## 用户已确认的设计决策

- 任务系统按世界运转、玩家体验、NPC 生涯三者打通设计。
- 玩家身份随成长切换：低阶亲自接任务，中阶组队或委托，高阶发布任务和调度势力。
- 任务来源采用三源分层：势力需求、事件机会、NPC 私人委托都存在，但任务必须由具体角色或岗位发布。
- 任务过程优先全程实体模拟；若后续不可控，再退回分级混合模拟。
- 任务失败按类型分层，失败本身会生成后续任务。
- 人情、威望、恶名、派系支持都是核心收益或代价。
- 软收益采用数值 + 标签并存。
- 任务链采用预设链 + 动态因果链并存。
- 第一版文档覆盖原则、数据结构、系统行为规则与可实现规格。
- 发布者第一版采用组织岗位 + 具名 NPC，设计预留所有 NPC 可发布。
- 私人委托完整规划资源求取、关系事件、机缘争夺、派系政治，并分阶段实现。
- 任务可见性支持公开任务板、定向委托、关系传播。
- 执行者模式支持单人、小队、多人竞争、协作竞争混合。
- 任务发布者可撤销、过期、加价、追加目标或转化任务。
- 每个任务设定失败惩罚、违约惩罚、背叛惩罚，角色可以不遵守惩罚并触发后续剧情。
- 契约形式支持口头委托、任务堂备案、押金/担保、血契/道誓/魂契。
- 功劳分配采用混合规则，并允许私吞、谎报、抢功、压功、冒领。
- 系统记录真相，角色认知不全，真相可调查但不保证揭露。
- 玩家可完全旁观，也可通过身份、资源、情报和任务发布间接干预。
