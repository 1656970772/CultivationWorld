# 十二境界运行时主链重构规格

> 最后更新：2026-06-07  
> 状态：设计确认，待实施计划  
> 适用范围：`ranks.json`、修为/真气突破门槛、寿元桶、修士战斗属性表、妖兽危险层级、默认战斗属性开关、相关文档口径  
> 关联：ADR-051、ADR-052、ADR-053、`docs/superpowers/specs/2026-06-07-修为数值化与突破重构-design.md`、`docs/superpowers/specs/2026-06-06-修士战斗属性体系-design.md`

## 背景

当前运行时 `apps/game/data/definitions/ranks.json` 只开放到 `spirit_transformation`（化神）。此前修士战斗属性体系已在三张战斗参考表中保留 12 个槽位，但这些槽位仍包含旧的“大罗天仙 / 道祖”运行时参考项，且 `combat.cultivatorAttributes.enabled=false`、`numericArmorDamage=false` 默认关闭。

本轮目标是把新的十二境界作为运行时主链一次性接入，并同步改造战斗属性参考表与妖兽危险层级。此前讨论过的大劫机制不进入本轮范围；本轮只把寿元最高不超过 4999 年作为境界配置约束。

## 参考取样

- 小说参考：`docs/世界观参考/凡人修仙传/境界提升与功法分析.md` 记录炼气、筑基、金丹、元婴、化神、炼虚、合体、大乘等阶段的寿元与突破资源阶梯；项目采纳其“寿元与瓶颈阶梯明显拉开”的方向，但整体压低到最高 4999 年。
- 游戏参考：《觅长生》公开资料中修为需求不是单纯每阶乘十，而是低阶和关键瓶颈跨度较大、高阶逐渐收敛；项目据此避免把高阶修为门槛拉成天文数字。
- 游戏参考：《鬼谷八荒》更适合参考“境界内小阶段 + 突破材料/宝物/天劫门槛”的结构；本轮暂不实现材料链和大劫，仅保留后续扩展空间。

## 目标

1. `ranks.json` 扩展为新的 12 档运行时主链，普通 NPC 可以沿主链突破到天仙。
2. 现有 `spirit_transformation / 化神` 保持原位置和语义，不迁移、不降级。
3. 新增 `void_refining / 炼虚`、`body_integration / 合体` 两个运行时境界 ID。
4. `cultivationRequired === qiRequired` 继续保持，避免修为门槛和真气门槛分叉。
5. 从炼虚开始，新增高阶门槛按上一境界 ×2 延展。
6. 寿元最高不超过 4999 年；本轮不设计 4999 年大劫机制。
7. 三张战斗属性表改成新 12 档名称和 ID，沿用原 12 个数值槽位，不重新调战斗数值。
8. 妖兽危险层级同步改为十一阶对应天仙。
9. 默认开启 `combat.cultivatorAttributes.enabled=true` 与 `numericArmorDamage=true`，让新六属性和数值护甲成为主干路径。
10. 文档中不得再把“大罗天仙 / 道祖”写成本轮运行时主链或战斗参考表档位。

## 非目标

- 不实现 4999 年大劫、世界级大劫、个人大劫或飞升退场机制。
- 不新增突破材料消耗链、天劫、心魔、境界跌落或突破失败死亡。
- 不重调战斗属性数值，只改名、改 ID、改对应关系。
- 不重做功法、法宝和丹药体系；只要求高阶配置能被后续系统引用。
- 不把大罗天仙、道祖作为本轮运行时 `rankId`。

## 运行时主链

运行时主链固定为：

```text
凡人 → 炼气 → 筑基 → 金丹 → 元婴 → 化神 → 炼虚 → 合体 → 大乘 → 渡劫 → 地仙 → 天仙
```

| order | 境界 | ID | `cultivationRequired` / `qiRequired` | 最大寿元 | `baseYears` / `varianceYears` |
|---:|---|---|---:|---:|---|
| 0 | 凡人 | `mortal` | 0 | 100 | 80 / 20 |
| 20 | 炼气 | `qi_refining` | 50 | 150 | 125 / 25 |
| 40 | 筑基 | `foundation_building` | 500 | 380 | 320 / 60 |
| 60 | 金丹 | `golden_core` | 5,000 | 900 | 750 / 150 |
| 80 | 元婴 | `nascent_soul` | 50,000 | 1,800 | 1,500 / 300 |
| 100 | 化神 | `spirit_transformation` | 500,000 | 2,500 | 2,200 / 300 |
| 120 | 炼虚 | `void_refining` | 1,000,000 | 3,200 | 2,850 / 350 |
| 140 | 合体 | `body_integration` | 2,000,000 | 3,800 | 3,400 / 400 |
| 160 | 大乘 | `mahayana` | 4,000,000 | 4,300 | 3,900 / 400 |
| 180 | 渡劫 | `tribulation` | 8,000,000 | 4,700 | 4,350 / 350 |
| 200 | 地仙 | `earth_immortal` | 16,000,000 | 4,900 | 4,650 / 250 |
| 220 | 天仙 | `heaven_immortal` | 32,000,000 | 4,999 | 4,850 / 149 |

`successionScore` 默认等于 `order`，保持掌门继任等既有排序逻辑稳定。天仙为当前运行时顶级；无下一境界时 `nextCultivationRequired=0`，成功突破到天仙后仍显式进入 `rankStage="early"`。

## 战斗属性槽位映射

三张战斗属性表使用新的 12 档 ID 和名称，但数值沿用原槽位，不重新调参。

| 新境界 | 新 ID | 使用原槽位数值 |
|---|---|---|
| 凡人 | `mortal` | 原 `mortal` |
| 炼气 | `qi_refining` | 原 `qi_refining` |
| 筑基 | `foundation_building` | 原 `foundation_building` |
| 金丹 | `golden_core` | 原 `golden_core` |
| 元婴 | `nascent_soul` | 原 `nascent_soul` |
| 化神 | `spirit_transformation` | 原 `mahayana` |
| 炼虚 | `void_refining` | 原 `tribulation` |
| 合体 | `body_integration` | 原 `spirit_transformation` |
| 大乘 | `mahayana` | 原 `earth_immortal` |
| 渡劫 | `tribulation` | 原 `heaven_immortal` |
| 地仙 | `earth_immortal` | 原 `great_luo_heaven_immortal` |
| 天仙 | `heaven_immortal` | 原 `dao_ancestor` |

`great_luo_heaven_immortal` 与 `dao_ancestor` 不再作为本轮战斗表 key 出现。若后续需要大罗天仙、道祖，应另开传说位格、称号、Boss 或扩展境界设计。

## 妖兽危险层级

妖兽最低层级为猛兽/凡兽，对应凡人；之后十一阶对应天仙。

| 妖兽层级 | 对应新境界 |
|---|---|
| 猛兽/凡兽 | 凡人 |
| 一阶 | 炼气 |
| 二阶 | 筑基 |
| 三阶 | 金丹 |
| 四阶 | 元婴 |
| 五阶 | 化神 |
| 六阶 | 炼虚 |
| 七阶 | 合体 |
| 八阶 | 大乘 |
| 九阶 | 渡劫 |
| 十阶 | 地仙 |
| 十一阶 | 天仙 |

该对应只表示危险层级和调参标尺，不代表同阶妖兽与同阶修士必须战力相等。

## 默认开关

本轮选择一次性切换主干路径，默认开启修士六属性和数值护甲：

```json
{
  "cultivatorAttributes": {
    "enabled": true,
    "numericArmorDamage": true
  }
}
```

默认开启后，NPC 初始化和突破刷新应写入 `maxHp/hp`、`maxYuan/yuan`、`attack`、`defense`、`speed`、`soul`，统一战斗入口使用数值护甲公式。

## 配置影响

实施时需要同步补齐或更新：

| 文件 | 变更 |
|---|---|
| `apps/game/data/definitions/ranks.json` | 扩展为新 12 档，补门槛、寿元、别名和继任分数 |
| `apps/game/data/balance/cultivation.json` | 补新增高阶的修炼速度、灵石消耗、真气产出、月贡、任务难度、晋升门槛等 |
| `apps/game/data/balance/combat.json` | 默认开启修士六属性和数值护甲 |
| `apps/game/data/definitions/combat-base-table.json` | 改新 12 档 key/name，数值按槽位迁移 |
| `apps/game/data/definitions/cultivator-combat.json` | 改新 12 档 key/name，数值按槽位迁移 |
| `apps/game/data/definitions/monster-combat.json` | 改新 12 档 key/name，数值按槽位迁移 |
| 初始 NPC 与测试夹具 | `rankId` 只允许新 12 档；现有化神保持 `spirit_transformation` |
| UI、报告、数据模型、Wiki、ADR/系统文档 | 统一新主链，移除大罗天仙/道祖作为本轮运行时档位的说法 |

`cultivation.json` 的高阶数值不在本规格逐项锁死；实施计划应按“到化神不破坏现有节奏，高阶平滑延展”的原则补表，并用真实模拟观察。

## 风险

- 之前修士战斗属性验证报告显示，迁移开关开启时斩妖任务击杀偏低；本轮默认开启后这是最高风险，需要作为主验证项。
- 高阶突破门槛从化神后按 ×2 延展，可能让高阶更容易被观察到；后续若高阶晋升过快，应优先用功法品阶、材料、天劫、心境和机会点门槛调节，而不是立即把修为数值拉成天文数字。
- `order` 扩展到 220 后，所有按境界差距判断压制、锁血、任务难度、猎妖可接取范围和继任评分的逻辑都需要重新验证。
- 移除大罗天仙/道祖运行时档位后，旧文档或测试若仍引用对应 ID，应一并清理。

## 验证

Focused tests：

- `ranks.json` 恰好包含新 12 个运行时境界 ID。
- 每个修仙境界的 `cultivationRequired === qiRequired`。
- 寿元桶满足 `baseYears + varianceYears <= 最大寿元`，且天仙最大值为 4999。
- 三张战斗属性表 key 与新主链对齐，不再包含 `great_luo_heaven_immortal` / `dao_ancestor`。
- 妖兽危险层级说明与表项同步为十一阶对应天仙。
- `combat.cultivatorAttributes.enabled=true` 与 `numericArmorDamage=true` 为默认值。
- 天仙顶级无下一境界，GOAP 派生字段和突破逻辑不制造假突破。
- UI、报告、快照展示新增境界和数值修为，不输出旧比例字段。

真实模拟：

- 先跑短程 100 天确认初始化、AI、战斗、猎妖、报告不崩。
- 再跑多种子长程模拟，观察突破次数、猎妖接取/完成、妖兽死亡、低阶死亡率、高阶 NPC 寿元、数值护甲伤害和默认新战斗路径是否合理。
- 不使用哈希、指纹或单一摘要值作为正确性证据；以真实行为数据判断是否达成。
