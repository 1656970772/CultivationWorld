# 数据编辑器设计

> 最后更新：2026-05-25

## 目标

为 `data/` 目录中的 JSON 配置提供一个纯前端编辑器，让设计者能看懂字段含义、修改数值、发现引用错误，并在浏览器授权后写回本地文件。

## 范围

第一版编辑以下数据集：

- `factions.json`：势力基础信息、资源、关系、特性、掌门引用。
- `npcs.json`：NPC 所属势力、性格四维、存活状态。
- `terrains.json`：地形类型、颜色、移动消耗、资源倍率。
- `modifiers.json`：世界状态、持续时间、概率、效果键值。
- `rules.json`：事件规则、触发条件、概率、冷却。
- `events.json`：事件模板、效果、玩家选项、信息传播参数。
- `map.json`：第一版提供摘要、引用校验和 JSON 高级编辑，不做 10000 格可视化地图画笔。

## 架构

编辑器作为独立入口 `data-editor.html` 存在，不修改游戏主入口 `index.html`。代码放在 `js/editor/`，样式放在 `css/data-editor.css`，避免和游戏 UI、渲染、引擎逻辑耦合。

核心模块：

- `schema-registry.js`：声明每个数据集的文件路径、主键、字段标签、字段帮助、表单类型、范围和枚举。
- `validation.js`：基于所有数据集做通用校验与跨文件引用校验。
- `data-store.js`：负责通过 `fetch` 读取默认数据，通过 File System Access API 读取/写入授权目录，并提供导出降级。
- `field-renderer.js`：把字段元信息渲染成输入控件。
- `editor-app.js`：组合导航、列表、表单、校验面板、JSON 预览和保存动作。

## 界面

视觉方向是“天机阁案牍房”：沉稳、清晰、可长期使用。布局分为左侧数据集导航、中间记录列表与表单、右侧说明/校验/JSON 预览。工具界面保持信息密度，避免营销式大图和装饰性卡片。

字段控件遵循数据语义：

- 数字范围使用滑杆加数字输入。
- 布尔值使用开关。
- 枚举使用下拉框。
- 数组标签使用标签编辑。
- 颜色使用色板。
- 嵌套对象使用分组字段。
- 复杂对象保留 JSON 子编辑器。

## 保存策略

浏览器支持 `showDirectoryPicker` 时，用户可以授权项目目录，编辑器写回 `data/*.json`。不支持或未授权时，用户可以下载当前数据集 JSON 文件。该策略保持项目纯前端、零后端依赖。

## 校验

第一版校验包括：

- JSON 数据是否为预期结构。
- 数据集内 `id` 或 `type` 是否重复。
- 数字字段是否在定义范围内。
- 势力 `leader` 是否引用存在的 NPC。
- NPC `factionId` 是否引用存在的势力。
- 势力关系是否引用存在的势力，关系值是否在 -100 到 100。
- 事件 `player_options` 是否有 id、文本、消耗与效果。
- 地图格子的 `terrain` 和 `ownerId` 是否引用存在的数据。

## 测试

使用 Node 脚本测试 `schema-registry.js` 与 `validation.js`，覆盖现有数据通过校验、重复 ID、断裂引用和数值越界。界面使用本地服务和浏览器插件做桌面与移动宽度检查。
