# 修士战斗属性体系规格

> 最后更新：2026-06-06
> 状态：设计确认，待实施计划
> 适用范围：修士基础战斗属性、境界/小层、妖兽等阶对齐、战斗基表、功法装备属性来源、后续 GAS/AttributeSet 接入

## 背景

当前运行时 NPC 已有 `hp/maxHp`、境界减伤 `baseDef` 和综合战力估值 `npcCombatPower`，但修士的完整战斗属性尚未确定。妖兽侧已经形成属性模板规划，包含气血、真元、攻击、防御、速度、神魂和习性等概念；为了让修士、妖兽、功法、装备和 GAS 机制后续能统一接入，需要先确定修士战斗属性的基础模型。

本规格只确定设计和数据方向，不直接实施代码。后续进入实施计划时，再拆分 `ranks.json` 清理、属性表配置、AttributeSet 接入、战斗公式迁移和真实模拟验证。

## 目标

1. 明确项目主境界链和小层规则。
2. 统一修士与妖兽的基础战斗属性名称。
3. 拆分修炼真气 `qi` 与战斗真元 `yuan/maxYuan`。
4. 建立一张只作参考的境界战斗基表。
5. 明确修士表和妖兽表分开维护，不强制互相推导。
6. 明确修士最终战力主要来自功法和装备，经 GAS/AttributeSet 修正基础属性。

## 非目标

- 不在本规格中实现代码。
- 不在本规格中完成所有功法、法宝、符箓和阵法数值。
- 不把修士表与妖兽表写成强绑定比例关系。
- 不在第一阶段设计完整命中、闪避、暴击、五行克制和逐回合蓝条战斗。

## 境界与小层

项目主境界链固定为：

```text
凡人 → 炼气 → 筑基 → 金丹 → 元婴 → 大乘 → 渡劫 → 化神 → 地仙 → 天仙 → 大罗天仙 → 道祖
```

凡人不拆小层；从炼气开始，每个大境界拆为：

```text
初期 → 中期 → 后期 → 圆满
```

小层倍率采用后期拉开型：

| 小层 | 倍率 |
|------|-----:|
| 初期 | 1.00 |
| 中期 | 1.15 |
| 后期 | 1.45 |
| 圆满 | 2.00 |

`rankId` 只表示修仙境界。`disciple`、`outer_disciple`、`core_disciple`、`elder`、`leader`、`general`、`officer` 等只作为 `role` 或社会身份，不参与境界属性表。

建议后续使用的境界 ID：

| 境界 | 建议 ID |
|------|---------|
| 凡人 | `mortal` |
| 炼气 | `qi_refining` |
| 筑基 | `foundation_building` |
| 金丹 | `golden_core` |
| 元婴 | `nascent_soul` |
| 大乘 | `mahayana` |
| 渡劫 | `tribulation` |
| 化神 | `spirit_transformation` |
| 地仙 | `earth_immortal` |
| 天仙 | `heaven_immortal` |
| 大罗天仙 | `great_luo_heaven_immortal` |
| 道祖 | `dao_ancestor` |

## 妖兽等阶对齐

妖兽最低层级为猛兽/凡兽，对应凡人层级。之后妖兽等阶与修士主境界平行：

| 妖兽层级 | 对应境界层级 |
|----------|--------------|
| 猛兽/凡兽 | 凡人 |
| 一阶 | 炼气 |
| 二阶 | 筑基 |
| 三阶 | 金丹 |
| 四阶 | 元婴 |
| 五阶 | 大乘 |
| 六阶 | 渡劫 |
| 七阶 | 化神 |
| 八阶 | 地仙 |
| 九阶 | 天仙 |
| 十阶 | 大罗天仙 |
| 十一阶 | 道祖 |

该对应只表示危险层级和调参参考，不代表普通同阶妖兽与普通同阶修士必须战力相等。

## 基础属性

修士与妖兽第一版统一六项基础属性：

| 属性 | 含义 |
|------|------|
| 气血 | 最大生命、肉身承伤能力 |
| 真元 | 战斗中可调动的真元上限和技能耐力 |
| 攻击 | 普攻、术法、法宝基础伤害能力的通用面板 |
| 防御 | 数值护甲，参与通用伤害公式 |
| 速度 | 战斗速度、追逃、反应、机动能力标尺 |
| 神魂 | 感知、幻术攻防、驯化难度、威压和 AI 估值 |

`qi` 与 `yuan/maxYuan` 拆分：

- `qi` 继续表示修炼、积累、突破用的真气资源。
- `yuan/maxYuan` 表示战斗中可调动的真元，用于技能、飞遁、护体、领域、持续追击等消耗。
- 凡人和猛兽也进入统一属性表，但真元默认为 0 或极低，不代表可施法。

## 防御与神魂边界

防御采用双层模型：

1. 基础属性 `防御` 是数值护甲，用于通用伤害公式。
2. 额外的减伤、抗性、护盾、免疫、反伤不混进基础六项，由功法、法宝、符箓、阵法、体质、临时状态通过 GAS/AttributeSet 提供。

第一版建议通用伤害公式采用数值护甲思路，例如：

```text
基础伤害 = 攻击 × 技能倍率 × 场景倍率
护甲系数 = 攻击 / (攻击 + 防御)
实际伤害 = max(1, 基础伤害 × 护甲系数 × 随机波动 × 额外减伤系数)
```

具体公式可在实施计划中结合现有 `combat-encounter` 迁移细化。

神魂第一阶段不进入普通物理/术法伤害公式，主要用于：

- 感知与探查。
- 幻术攻防。
- 驯化难度。
- 威压判断。
- AI 战力估值。

后续制作神魂技能、幻境、夺舍、心魔时，再让这些技能走 `神魂 vs 神魂` 或 `神魂 vs 特定抗性` 对抗。

## 境界战斗基表

基表只作为同阶参考强度，不是修士表或妖兽表的强制来源。数值按每个境界初期给出；中期、后期、圆满乘小层倍率。凡人不拆小层。

| 境界 | 气血 | 真元 | 攻击 | 防御 | 速度 | 神魂 |
|------|-----:|-----:|-----:|-----:|-----:|-----:|
| 凡人 | 100 | 0 | 20 | 8 | 10 | 5 |
| 炼气 | 250 | 120 | 55 | 25 | 25 | 25 |
| 筑基 | 800 | 450 | 170 | 85 | 70 | 80 |
| 金丹 | 2400 | 1600 | 520 | 280 | 190 | 260 |
| 元婴 | 7000 | 5500 | 1500 | 850 | 500 | 850 |
| 大乘 | 20000 | 18000 | 4200 | 2500 | 1300 | 2600 |
| 渡劫 | 60000 | 65000 | 12500 | 7600 | 3500 | 8000 |
| 化神 | 180000 | 220000 | 38000 | 23000 | 9000 | 25000 |
| 地仙 | 550000 | 750000 | 115000 | 70000 | 24000 | 80000 |
| 天仙 | 1700000 | 2500000 | 350000 | 220000 | 65000 | 260000 |
| 大罗天仙 | 5500000 | 8500000 | 1100000 | 700000 | 180000 | 900000 |
| 道祖 | 20000000 | 35000000 | 4000000 | 2800000 | 600000 | 4000000 |

设计意图：

- 大境界之间大体保持 2-5 倍跨度，保留越阶战斗空间。
- 气血、真元、攻击、防御、神魂可随境界显著提升。
- 速度是战斗/追逃/反应速度标尺，不直接等同地图移动格数；地图移动可另行换算并设置上限。

## 修士表与妖兽表

后续建议分三张配置表：

| 表 | 职责 |
|----|------|
| `combat-base-table.json` | 境界战斗基表，只作调参标尺 |
| `cultivator-combat.json` | 普通修士裸面板 |
| `monster-combat.json` | 普通妖兽/猛兽裸面板 |

三张表不做强绑定。修士表和妖兽表都参考基表，但可以独立调整。调参时只要求解释偏离原因，例如普通修士裸身、体型巨大、上古血脉、族群模板、装备不足、宗门传承强弱等。

普通修士第一版方向：

- 气血、防御低于同层级参考基表。
- 攻击略低于同层级参考基表。
- 真元、神魂高于同层级参考基表。
- 速度接近同层级参考基表。
- 体修、剑修、法修、幻修等不写进裸表，通过修正层叠加。

普通妖兽第一版方向：

- 气血、攻击、防御高于同层级参考基表。
- 速度按族类和移动模板浮动。
- 真元、神魂按灵智、血脉、战斗风格和特殊类型浮动。
- 现有妖兽五层模板继续有效，并扩展到猛兽/凡兽和一至十一阶。

## 修士属性来源

修士裸表只表示普通修士基础体魄与可调动真元，不是最终战力。最终有效属性通过 AttributeSet/GAS 叠加：

```text
有效属性 = 境界裸表 × 小层倍率
        + 功法修正
        + 装备/法宝修正
        + 体质/灵根修正
        + 丹药/符箓/阵法/临时状态修正
```

第一优先级是功法和装备：

- 功法决定长期成长方向：法修偏真元/攻击/神魂，剑修偏攻击/速度，体修偏气血/防御，幻修偏神魂，遁法偏速度。
- 装备/法宝决定实战差距：攻击法宝、护身法宝、飞行法宝、神魂法宝、阵旗等都通过 GAS Effect 增加基础属性或提供额外减伤/护盾/技能。
- 普通无装备修士不应轻松正面压制同阶妖兽；有好功法和好法宝的修士可以越阶或反杀强妖。

## 实施阶段建议

1. 清理境界数据，使 `rankId` 只保留修仙境界；职位与身份迁移到 `role`。
2. 新增境界小层字段，例如 `rankStage`，凡人无小层，炼气以后取 `early/middle/late/perfection`。
3. 新增战斗基表、修士基础属性表和妖兽基础属性表。
4. 在 NPC 初始化时生成 `maxHp/hp`、`maxYuan/yuan`、`attack`、`defense`、`speed`、`soul` 等基础属性。
5. 将 `baseDef` 比例减伤迁移为数值防御，额外减伤保留为 GAS 修正层。
6. 让功法和装备通过 GameplayEffect 或 AttributeSet modifier 修正属性。
7. 保留数据开关分阶段迁移，避免一次性改变所有战斗结果。

## 验证方式

验证必须使用真实、完整的模拟环境，不使用黄金指纹或哈希自证。

需要观察：

- 凡人/低阶修士遭遇猛兽、一阶妖兽时是否危险但不是无解释秒杀。
- 普通同阶修士裸身是否弱于同阶妖兽。
- 有功法和装备后，修士是否能合理越阶战斗。
- 高一大境界是否有明显压制，但不是完全不可逆。
- 小层倍率是否让圆满期明显强于初期，但不等于跨越完整大境界。
- `qi` 与 `yuan` 拆分后，战斗消耗不应破坏突破节奏。
- 神魂用于感知、幻术、驯化、威压和 AI 估值时是否能产生可观察差异。
