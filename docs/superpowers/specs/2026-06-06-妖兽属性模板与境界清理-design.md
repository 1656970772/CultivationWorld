# 妖兽属性模板与境界清理规格

> 最后更新：2026-06-06
> 状态：设计确认，待实施计划
> 适用范围：`apps/game/data/definitions/ranks.json`、`apps/game/data/entities/`、`apps/game/data/definitions/monsters.json`、妖兽配置校验与后续战斗平衡

## 背景

当前 NPC 的 `rankId` 同时混用了修行境界、凡人身份头衔和武道头衔，例如 `disciple`、`advisor`、`general`、`commander`、`grandmaster`、`martial_saint`。这会让血量、减伤、突破和战力公式读到不一致的数据：修仙境界在 `combat.json` 中有血量配置，凡人头衔则回退到默认值，出现 30 血、0 减伤等不合理结果。

妖兽数据也存在类似问题：当前 `monsters.json` 直接手填 `strength`、`speed`、`defense`、`sense`、`vitality`，但缺少体型、移动方式、战斗风格、属性和特殊身份的结构化约束。小型飞行妖兽、重甲玄龟、幻术狐族、虫群、族长、上古血脉等差异应由模板表达，而不是靠散乱手填。

## 目标

1. 清理运行时境界体系，使 `rankId` 只表示修仙境界。
2. 从初始运行时世界移除凡人王朝体系。
3. 建立妖兽的基础属性、技能和五层模板系统。
4. 建立妖兽配置检查规则，为后续新增妖兽和迁移现有妖兽提供标准。

## 非目标

本规格不展开修士/NPC 的完整属性体系。普通修士血量、体修路线、功法、法宝、真元、神魂等，等妖兽体系定稿后另开规格讨论。

本规格不立即要求实现完整逐回合蓝条战斗。`真元` 先作为技能消耗和持续战斗耐力的配置基础，后续可逐步接入战斗管线。

## 境界与凡人王朝清理

`rankId` 只保留修仙境界：

| rankId | 中文名 |
| --- | --- |
| `mortal` | 凡人 |
| `qi_refining` | 炼气 |
| `foundation_building` | 筑基 |
| `golden_core` | 金丹 |
| `nascent_soul` | 元婴 |
| `spirit_transformation` | 化神 |

以下内容不再作为运行时境界：

- `disciple`、`outer_disciple`、`core_disciple`、`elder`、`leader`、`general`、`officer` 等只作为 `role`。
- `advisor`、`commander`、`grandmaster`、`martial_saint` 等凡人王朝或武道头衔从运行时境界表移除。
- `大晋王朝` 等凡人王朝势力从初始世界移除，不作为可行动势力参与模拟。

如果后续仍需要“王朝衰败”“世俗动荡”等内容，只保留为世界事件、背景修饰或机会点叙事素材，不进入核心 NPC/势力模拟。

迁移原则：

- `rankId: disciple` 的 NPC 如需保留，改为 `mortal` 或 `qi_refining`，`disciple` 只留在 `role`。
- 王朝系 NPC 默认移除，不强行转化为修士，避免污染修仙沙盒。
- 血量、减伤、修炼、突破、战力公式只读修仙境界，不为凡人头衔保留兜底分支。

## 妖兽基础属性

妖兽基础属性采用直观命名：

| 属性 | 含义 |
| --- | --- |
| 气血 | 最大生命、肉身耐久 |
| 真元 | 法力值、技能消耗、持续飞遁/追击/领域耐力 |
| 攻击 | 普攻和直接伤害能力 |
| 防御 | 承伤、护甲、鳞甲、抗打能力 |
| 速度 | 地图移动、追逃、行动半径 |
| 神魂 | 感知、幻术/神魂攻防、威压、驯化难度 |
| 习性 | AI 倾向和少量战斗倾向，不是纯数值面板 |

## 妖兽技能

技能不设固定数量。低阶普通妖兽可以没有技能，高阶、头领、族长、上古血脉、秘境守护、异变污染、纯血后裔可以拥有多个技能。

技能分三类：

| 技能类型 | 示例 |
| --- | --- |
| 身法技能 | 飞遁、潜伏、瞬移、冲锋、遁地、水遁 |
| 主动技能 | 爪击、毒液、火焰、雷击、幻术、领域、吞噬 |
| 被动技能 | 鳞甲、群居、再生、毒抗、龟息、涅槃、血脉威压 |

使用真元的技能必须配置消耗或消耗规则。领域、飞遁、持续追击等持续型技能需要表达持续消耗。

## 五层模板

妖兽配置由以下五层模板组合：

1. `体型`：必填且只能一个。
2. `移动`：可多个。
3. `战斗风格`：可多个。
4. `属性`：可多个。
5. `特殊类型`：可多个，但 `普通` 与其他特殊类型互斥。

属性生成规则：

- 气血只受 `阶位基准 + 体型 + 特殊类型` 修正。
- 攻击受 `阶位基准 + 体型 + 移动 + 战斗风格 + 特殊类型` 修正。
- 防御受 `阶位基准 + 体型 + 战斗风格 + 特殊类型` 修正。
- 速度受 `阶位基准 + 体型 + 移动 + 特殊类型` 修正。
- 真元受 `阶位基准 + 战斗风格 + 特殊类型` 修正。
- 神魂受 `阶位基准 + 战斗风格 + 特殊类型` 修正。
- 习性用于 AI 倾向，不参与通用面板公式。
- 属性不直接加面板，只表达特点、技能风格、抗性、材料方向和后续结算规则。

## 体型模板

| 体型 | 气血 | 攻击 | 防御 | 速度 | 说明 |
| --- | ---: | ---: | ---: | ---: | --- |
| 微型 | ×0.25 | ×0.35 | ×0.4 | ×1.4 | 单体极脆，如小虫、小蝠 |
| 小型 | ×0.45 | ×0.6 | ×0.65 | ×1.25 | 飞禽、小狐、小蛇 |
| 中型 | ×1.0 | ×1.0 | ×1.0 | ×1.0 | 默认基准 |
| 大型 | ×1.6 | ×1.35 | ×1.25 | ×0.8 | 虎、熊、猿、大蟒 |
| 巨型 | ×2.4 | ×1.8 | ×1.6 | ×0.55 | 蛟龙、玄龟、巨兽 |
| 群体 | ×0.35 | ×0.45 | ×0.5 | ×1.1 | 单体弱，但配合群居或虫群被动 |

体型不直接修正真元和神魂。

## 移动模板

| 移动 | 攻击 | 速度 | 身法倾向 |
| --- | ---: | ---: | --- |
| 地行 | ×1.0 | ×1.0 | 无默认 |
| 飞行 | ×0.9 | ×1.5 | 飞遁、俯冲、空袭 |
| 水栖 | ×1.05 | ×0.9 | 水遁、水域加速 |
| 遁地 | ×0.95 | ×0.85 | 潜地突袭、脱战 |
| 攀爬 | ×0.95 | ×1.15 | 林间穿梭、垂直追击 |
| 潜伏 | ×1.15 | ×0.75 | 伏击、隐匿突袭 |
| 瞬移/空间 | ×1.0 | ×1.25 | 短距闪现、空间躲避 |

多个移动模板可叠加。速度总倍率超过 ×2.2 时，校验工具提示需要原因。

| 属性 | 特点 |

| 无属性 | 普通肉身、无明显灵力属性 |

| 木 | 再生、毒/药性、藤蔓、生机 |
| 水 | 水域加速、柔韧、缠绕、恢复 |
| 火 | 爆发、燃烧、范围伤害 |
| 土 | 重甲、护体、震荡、守护 |
| 风 | 速度、切割、飞遁、闪避 |
| 雷 | 爆发、麻痹、威压、破邪倾向 |
| 冰 | 减速、冻结、寒毒、控制 |
| 毒 | 持续伤害、麻痹、腐蚀 |
| 幽冥 | 神魂侵蚀、恐惧、生命流逝 |

## 特殊类型模板

特殊类型使用建议区间，不硬性禁止超出。超出建议区间时必须写 `reason`、`source` 或 `description` 说明。

| 特殊类型 | 建议整体强度 |
| --- | ---: |
| 普通 | ×1.0 |
| 精英 | ×1.2-1.5 |
| 头领 | ×1.5-2.0 |
| 族长 | ×1.8-2.5 |
| 纯血后裔 | ×1.5-2.5 |
| 上古血脉 | ×2.0-3.5 |
| 秘境守护 | ×1.5-3.0 |
| 异变污染 | ×1.3-3.0，允许属性异常和副作用 |

## 习性

`习性` 主要控制 AI 倾向，并可提供少量战斗倾向。第一版采用可读标签，例如：

- 凶暴：更主动攻击，追杀意愿更强，可略偏攻击。
- 谨慎：更容易撤退、守巢、保命，可略偏防御。
- 群居：更依赖族群加成和呼群技能。
- 守巢：更重视领地、巢穴、资源点防守。
- 伏猎：更偏潜伏、绕后、突袭。
- 灵智：更容易使用技能、记仇、选择目标。

## 妖兽配置校验规则

新增或修改妖兽时，校验工具需要检查：

- `grade` 必须存在，并能找到对应阶位基准。
- 五层模板必须合法：体型必填且只能一个；移动、战斗风格、属性、特殊类型可多个。
- `普通` 不能和其他特殊类型同时出现。
- 属性不能直接修改面板数值。
- 气血、攻击、防御、速度、真元、神魂必须按本规格的修正来源计算。
- 技能必须标明类型：身法技能、主动技能、被动技能。
- 技能数量不设硬限制，但低阶普通妖兽拥有过多高阶技能时提示风险。
- 使用真元的技能必须配置消耗或消耗规则。
- 领域、飞遁、持续追击类技能要能表达持续消耗。
- 特殊类型超过建议倍率时不报错，但必须写明原因。
- 头领、族长、上古血脉、秘境守护、异变污染、纯血后裔等高强度妖兽，必须能从描述、来源或世界观设定中看出强度来源。
- 小型飞行、虫群、幻术狐族、玄龟、蛟龙等典型模板组合应呈现合理倾向，避免“飞行小妖血厚如龟”这类配置。

## 实施阶段建议

1. 清理 `ranks.json`、初始 NPC 和凡人王朝势力。
2. 新增妖兽模板配置和校验工具，先让配置可检查。
3. 迁移现有 `monsters.json`，为每只妖兽挂上体型、移动、战斗风格、属性和特殊类型。
4. 用生成值替代当前手填属性，保留少量手工微调字段。
5. 在真实模拟中复核妖兽袭击、斩妖任务、妖兽死亡和材料产出。

## 验证方式

验证必须使用真实模拟观察行为，不使用黄金指纹或哈希自证。

需要观察：

- 低阶普通妖兽是否仍是低阶修士的日常威胁。
- 普通修士血量是否约为普通同阶妖兽的 40%-50%。
- 小型、飞行、幻术、虫群、重甲、巨型妖兽是否呈现不同战斗生态。
- 头领、族长、上古血脉等是否强得有理由，但没有无解释膨胀。
- 斩妖任务、妖兽袭击、妖兽死亡、材料产出是否仍形成闭环。

## 后续讨论项

- 修士/NPC 基础属性体系。
- 普通修士血量曲线。
- 体修路线：体质 + 炼体功法/流派叠加，可达到同阶妖兽甚至更强。
- 属性是否在未来进入克制或抗性结算。
- 妖兽真元蓝条是否进入逐战斗过程。
