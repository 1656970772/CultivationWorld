## 追杀令链观察

- `combat.kill` 公开击杀会写入受害者势力对凶手的 faction ledger，并生成 `wantedOrder` mark。
- `wantedOrder` 对同势力成员输出 `isWantedByFaction`、`canTriggerWantedHunt`、`huntWeight`、`revengeTargetBonus`。
- `lifeDebt` 会降低追杀权重，并输出 `canIntercede`；当目标只有救命恩而没有血仇时，`topWantedTargetForFaction` 会跳过该目标。
- `bloodFeud` 会增强个人追杀偏好，并与势力通缉信号通过 `RelationshipSignalProvider` 合并。
- `decorateGoalConsiderations` 已消费 `worldContext.getRelationshipSignals`，对复仇类 Goal 挂载 `relationship.huntWeight` 调制项。

## 真实模拟样本

命令：

```powershell
node apps/game/tools/simulate-analysis.mjs 120
```

本次样本：

- 引擎初始化：16 个势力，126 个 NPC。
- 模拟种子：1426773073。
- 模拟完成：120 天，用时 13.92 秒。
- 存活 NPC 执念分布：`supremacy=11`、`resurrection=5`、`revenge=3`、`power=4`、`longevity=1`。
- 复仇 PvP：追踪触发 0 次，击杀触发 0 次，PvP 致死总数 3，存活 NPC 持有复仇执念 3。
- 关系账本：`individual=2176`、`group=0`、`faction=0`、旧边存量 `legacyEdges=1734`。
- 关系标记：`wantedOrder=0`、`bloodFeud=60`、`resourceGrudge=644`、`lifeDebt=0`。
- 旧关系投影样本：`beast_grudge=1`、`territory_threat=104`、`grudge=0`。
- 师徒互动：传功点化 2 次，探望恩师 7 次。
- Job 诊断 Top：`replan:safe_hunt_target_missing=6340`、`in_progress:in_progress=3455`、`replan:no_available_quest=270`。

本次最终真实样本没有自然出现 `wantedOrder`，但 `bloodFeud/resourceGrudge` 账本和 `beast_grudge/territory_threat` 旧关系投影均非空；公开击杀到势力通缉账本的链路由 `test-relationship-wanted-chain.mjs` 覆盖到死亡收集器接入。该结论只基于真实行为统计，不使用指纹或哈希作为正确性证据。

## 结论

第一版三层关系底座已完成并通过复核：

- 关系真相源收敛到三层账本，旧边 API 通过 mark/tag 投影保持兼容。
- 关系规则通过仓储、解释器、selector、effect operator、impact engine、signal provider 分层实现，新增规则可优先走数据配置。
- 追杀令链可从公开击杀写入通缉标记，并把 wanted/bloodFeud/lifeDebt 信号传入复仇目标解析和 Utility 调制。
- 旧关系、师徒、复仇、Job/Toil、Utility 评分回归均通过。

## 剩余风险

- 120 天单种子真实样本未自然出现 `wantedOrder` 和 `lifeDebt`；后续应做多种子长程观察，确认追杀令与报恩/求情链在真实节奏中自然涌现。
- `group` 层已有 schema 与数据入口，但本轮真实样本仍为 0；后续新增宗门小团体、队伍、妖群组织规则时需要补充 group 写入样本。
- 初始化与部分旧调用点仍通过兼容门面调用 `addEdge`，迁移期可接受；后续新增业务应优先使用 `handleEvent/addMark/addTag/getSignals`。
