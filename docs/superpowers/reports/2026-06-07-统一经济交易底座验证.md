# 统一经济交易底座验证

> 最后更新：2026-06-07
> 状态：已验证

本报告记录统一经济交易底座实现完成后的真实验证结果。验证只接受资产变化、账本记录、债务状态、事件输出和多种子模拟行为作为证据；不使用指纹、哈希或单一摘要值证明功能正确。

## 验证命令

在 `apps/game` 目录运行：

```powershell
node tools/test-economic-asset-adapter.mjs
node tools/test-economic-transaction-platform.mjs
node tools/test-economic-integration.mjs
node tools/test-economic-auction-abstract.mjs
node tools/test-job-config-load.mjs
node tools/test-quest-reward-economy.mjs
node tools/test-economy-toils.mjs
node tools/test-dynamic-event-system.mjs
node tools/simulate-analysis.mjs 300 --seed=1001
node tools/simulate-analysis.mjs 300 --seed=1002
node tools/simulate-analysis.mjs 300 --seed=1003
```

## 结果记录

| 命令 | 退出码 | 关键观察 | 结论 |
|------|-------:|----------|------|
| `node tools/test-economic-asset-adapter.mjs` | 0 | NPC 背包灵石可扣除和回滚；势力 `low_spirit_stone` / `food` 读写走 state，未改普通 inventory；贡献点不可私下转让 | 通过 |
| `node tools/test-economic-transaction-platform.mjs` | 0 | `direct_purchase` 成功写 `settled` 账本；卖方资产不足时回滚买方灵石和卖方收入；托管释放后资产转入收款方；债务到期转为 `overdue` 并输出经济信号；`private_trade` 平台层拒绝私下转让组织点数并写失败账本 | 通过 |
| `node tools/test-economic-integration.mjs` | 0 | 材料上交写 `material_donation` 账本；贡献兑换写 `contribution_exchange` 账本；悬赏付款方短款时任务交付仍完成，写失败 `quest_reward` 账本并生成 active 债务；FactionAI 势力贸易写 `faction_trade` 账本且 `low_spirit_stone` / `food` 读写走 state；`FactionEntity.onPreTick` 以 state 为真相源并同步 inventory；`act_trade` 只保留 plannerEffects，实际交易由经济底座结算 | 通过 |
| `node tools/test-economic-auction-abstract.mjs` | 0 | 抽象拍卖成交，赢家获得拍品、支付灵石，账本记录 `auction_sale`，高价成交产生财富曝光事件 | 通过 |
| `node tools/test-job-config-load.mjs` | 0 | `loadGameConfigs` 加载 `economicTransactionConfig`；`WorldEngine` 初始化 `EconomicSystem`；worldContext 暴露 `settleTransaction` / `openEscrow` / `economicSignalsFor`；世界快照暴露经济账本状态 | 通过 |
| `node tools/test-quest-reward-economy.mjs` | 0 | 任务额外奖励、材料捐献、贡献兑换、聚气丹消耗、破境丹加成和法宝自动装备链路保持可用 | 通过 |
| `node tools/test-economy-toils.mjs` | 0 | `ensure item`、购买、装备、货币不足、非法购买参数、缺失 inventory add 等 Toil 边界均保持预期行为；贡献兑换 Toil 通过单笔 `contribution_exchange` 交易扣除贡献点并写入组织点账本；缺少经济端口时返回 `economic_system_missing`，不扣贡献、不扣灵石、不发放物品 | 通过 |

## 观察项

| 项 | 结果 |
|----|------|
| NPC 与组织资产读写 | 通过：NPC inventory 扣除、存入与失败回滚均被测试覆盖 |
| 势力 state 资源单一真相源 | 通过：势力 `low_spirit_stone` / `food` / `disciples` 读写走 state，测试确认不直接改普通 inventory |
| 正式交易交割与账本 | 通过：`direct_purchase` 成功结算并写入 `settled` 账本 |
| 失败交易回滚 | 通过：卖方资产不足时交易失败，买方灵石和卖方收入均回滚，并写入失败账本 |
| 材料上交与贡献兑换 | 通过：`material_donation` 与 `contribution_exchange` 账本可观察 |
| 组织点数转让边界 | 通过：`private_trade` 在平台层拒绝 `organization_point`，贡献兑换只能通过正式场景账本结算；缺少经济端口时贡献兑换 Toil 失败且资产不变 |
| 势力贸易 | 通过：`faction_trade` 账本可观察，势力灵石和粮食读写走 state；`FactionEntity.onPreTick` 以 state 为真相源并同步 inventory，避免交易结算被旧库存覆盖 |
| 任务奖励与短款债务 | 通过：悬赏阁付款不足时任务交付仍完成，经济底座生成 active 债务 |
| 抽象拍卖成交、流拍与财富曝光 | 通过：`auction_sale` 账本、成交资产变化和财富曝光事件可观察 |
| 长程模拟异常 | 通过：seed 1001 / 1002 / 1003 各 300 天真实模拟无运行时异常 |
