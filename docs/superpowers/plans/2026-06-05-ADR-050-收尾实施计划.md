# ADR-050 收尾实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 完成 ADR-050 的实施收尾：确认 Job/Toil 分层已真实接入，并修复动态事件 JobAction 串线、Job 配置引用不存在物品、文档状态滞后三类问题。

**Architecture:** 保留既有五层 AI 架构 `Reaction -> Utility / Intent -> GOAP -> Job / Toil -> Execution`。本次只做窄口径收尾：GOAP 仍只规划高层 Action；Job/Toil 仍只负责复杂 Action 内部流程；新增的事件类型派生状态只用于选择正确 JobAction，不参与 Utility 评分。

**Tech Stack:** 原生 ES Module JavaScript、JSON 数据配置、PowerShell、Node.js 工具脚本、真实多种子长程模拟验证。

## 规划口径

本计划分两段执行：

1. 默认启用前必须完成：配置引用校验、事件类型绑定、ADR/spec 状态同步、真实多种子验证。
2. 默认启用后或另开计划推进：动态事件/人生内容库扩展、PvP 真实扣血和反击触发。

内容库和 PvP 不阻塞 ADR-050 的基础设施收尾；它们作为后续阶段进入独立规格或实施计划。

const itemRefKeys = new Set(['itemId', 'priceItemId', 'currencyItemId']);
function collectItemRefs(value, refs = []) {
  if (Array.isArray(value)) {
    for (const item of value) collectItemRefs(item, refs);
    return refs;
  }
  if (!value || typeof value !== 'object') return refs;
  for (const [key, child] of Object.entries(value)) {
    if (itemRefKeys.has(key) && typeof child === 'string') refs.push(child);
    collectItemRefs(child, refs);
  }
  return refs;
}

  assert(itemIds.has(itemId), `${job.id} references existing item ${itemId}`);
}
