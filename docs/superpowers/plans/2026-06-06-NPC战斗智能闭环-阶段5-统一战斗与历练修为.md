# NPC 战斗智能闭环阶段 5：统一战斗与历练修为 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
> 最后更新：2026-06-06

**Goal:** 补齐统一战斗公式的剩余场景，并把 `opportunity`、`pvp`、`social_travel` 接入历练修为。

**Architecture:** `resolveCombatEncounter()` 返回所有战斗/风险场景的统一结果结构；实际扣血仍交给 `applyDamage()`。历练修为统一走 `applyCultivationExperience()`，避免散落在 Action 或 Toil 中各自加数值。

**Tech Stack:** Combat Encounter、Combat Pipeline、NPC Toils、Relationship/Opportunity Actions、Cultivation Experience、Node 工具测试。

## Task 2: 统一战斗场景

- [ ] **Step 1: `combat-encounter.js` 补齐结果结构**

所有 scene 都返回：

```js
{
  scene,
  hit: true,
  damage,
  winChance,
  died: false,
  injuryGain,
  retreatSuggested,
  deathInfo: null,
  experienceContext: { sourceKind, value, riskScore, outcome }
}
```

- [ ] **Step 2: PvP Toil 使用统一胜率**

`NPCKillEnemyToilExecutor` 用 `resolveCombatEncounter(scene:'pvp')` 取 `winChance`。

- [ ] **Step 3: 普通风险用 `quest_risk`**

`npc-action-utils.js` 的风险扣血先走 `resolveCombatEncounter(scene:'quest_risk')`，再交给 `applyDamage()`。

- [ ] **Step 4: NPC 反击妖兽用 `monster_counter`**

`MonsterEntity._attack()` 中 NPC 反击伤害改由 `resolveCombatEncounter(scene:'monster_counter')` 生成。

## Task 3: 历练修为来源

- [ ] **Step 1: 配置补齐**

`cultivation.json.experience.baseBySource` 包含：

```json
"opportunity": 5,
"pvp": 8,
"social_travel": 1
```

- [ ] **Step 2: PvP 存活获得历练**

复仇、击杀仇人或 PvP 存活路径调用 `applyCultivationExperience(sourceKind:'pvp')`。

- [ ] **Step 3: 机会点获得历练**

`NPCGotoOpportunityExecutor` 成功领取机会点后调用 `applyCultivationExperience(sourceKind:'opportunity')`。

- [ ] **Step 4: 外出社交获得历练**

`NPCTeachDiscipleToilExecutor`、`NPCVisitMasterToilExecutor` 以及仍保留的关系 SimpleAction 成功后调用 `applyCultivationExperience(sourceKind:'social_travel')`。
