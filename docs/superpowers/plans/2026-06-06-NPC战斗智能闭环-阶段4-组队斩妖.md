# NPC 战斗智能闭环阶段 4：组队斩妖 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
> 最后更新：2026-06-06

**Goal:** 让高价值高风险斩妖任务能触发邀请同伴、等待同伴、共同战力评估和共同结算，组队失败时不阻塞任务链。

**Architecture:** 阶段 2 已产出 `needsCompanion`。本阶段补完 `job_npc_request_hunt_companion` 的 Toil 链和 `settleMonsterHunt()` 的 party 结算。组队只是斩妖任务的加成路径，不改变普通单人斩妖主链。

**Tech Stack:** Social Toils、Combat Toils、Monster Resources、Quest Service、Node 工具测试。

## Task 1: 失败测试

- [ ] **Step 1: 扩展战斗智能测试**

在 `test-combat-intelligence-jobs.mjs` 增加：

- 请求成功后写 `huntCompanionId`、`huntPartyIds`、`hasHuntCompanion=true`。
- 有同伴后再次评估风险，`combatRiskScore` 下降。

- [ ] **Step 2: 扩展斩妖结算测试**

在 `test-monster-hunt-job.mjs` 增加：

- `settleMonsterHunt(entity, monster, worldContext, randomFn, { party })` 使用共同战力。
- 妖兽死亡信息含 `assistNpcIds`。
- 同伴获得部分 `experienceCultivation`。

## Task 2: 请求同伴 Job

- [ ] **Step 1: 扩展 Job 配置**

`job_npc_request_hunt_companion` 改为：

```json
"toils": [
  { "id": "select", "type": "toil_select_companion", "params": { "sameFactionPreferred": true, "maxDistance": 60, "minPowerRatio": 0.4 } },
  { "id": "request", "type": "toil_request_companion" },
  { "id": "wait", "type": "toil_wait_for_hunt_companion", "params": { "maxDays": 3, "targetResolver": "quest_target" } }
]
```

- [ ] **Step 2: 新增等待 Toil 配置**

```json
{ "id": "toil_wait_for_hunt_companion", "name": "等待斩妖同伴", "category": "combat" }
```

## Task 3: 同伴选择与等待

- [ ] **Step 1: 同伴评分**

`NPCSelectCompanionToilExecutor` 按这些因素评分：

- 同势力加分。
- 战力越高越好。
- 距离越近越好。
- 死亡或非 alive 排除。

- [ ] **Step 2: 请求写状态**

`NPCRequestCompanionToilExecutor` 成功写：

```text
lastCompanionId
huntCompanionId
huntPartyIds
huntCompanionRequested=true
hasHuntCompanion=true
needsCompanion=false
```

- [ ] **Step 3: 等待同伴**

`toil_wait_for_hunt_companion` 将同伴目的地设为任务目标。超过 `maxDays` 未抵达时返回 `REPLAN`，reason 为 `hunt_companion_timeout`，并清 `hasHuntCompanion`。

## Task 4: 共同结算

- [ ] **Step 1: `settleMonsterHunt()` 支持 party**

新增可选参数：

```js
settleMonsterHunt(entity, monster, worldContext, randomFn, { party })
```

主猎人 100% 战力，同伴 70% 战力计入总战力。

- [ ] **Step 2: 共同经验与死亡信息**

成功击杀时：

- `_deathInfo.assistNpcIds = party.slice(1).map(n => n.id)`。
- 主猎人获得 `monster_hunt_success`。
- 同伴获得 `monster_hunt_success` 的部分收益，`outcome:'partial'`。
