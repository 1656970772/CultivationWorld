# 门派架构与运行模拟 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立第一版可真实运行的门派组织、宗门背包、月俸库存压力、统一任务列表、个人悬赏托管、弟子离宗和宗门倒闭闭环。

**Architecture:** 仍以现有 `FactionEntity` 作为门派主体，不新增独立门派实体；新增 `engine/sect/` 作为门派运行子域。子域采用配置注册表 + 策略规则链：`SectConfigRegistry` 统一读取组织、运行数值和初始化模板，`SectOperationService` 只编排规则，月俸、库存压力、离宗、倒闭分别由可注册 `SectOperationRule` 执行。宗门灵石仍以 `FactionState.low_spirit_stone` 为单一真相源，丹药、材料、法器等物品保存在 faction inventory，个人悬赏奖励进入独立的悬赏托管仓，避免被势力 state/inventory 同步覆盖。

**Tech Stack:** JavaScript ES modules、JSON 数据配置、现有 WorldEngine / TickManager / EconomicSystem / QuestService / Job-Toil、Node.js 工具验证。

**最后更新:** 2026-06-08

### 架构与配置化审查结论（2026-06-08）

本计划经多 agent 并行审查后，执行时必须遵守以下硬约束：

1. **禁止把门派运行做成单个上帝服务。** `SectOperationService` 只负责读取上下文、按配置顺序调用规则、汇总结果；月俸、库存压力、离宗、倒闭、后续任命/内部政治都实现为可注册策略规则。
2. **禁止用代码硬编码堂口、任务类型、门派 ID、NPC ID、初始库存、初始道具或阈值。** 这些内容必须来自 JSON 配置：组织定义、运行数值、初始化模板、实体覆盖或任务模板元数据。
3. **初始资源和初始道具全部走配置。** 势力宏观资源使用 `factions.json.resources` 或 `sect-seed-profiles.json.resourceProfiles`；势力实物库存使用 `sect-seed-profiles.json.inventoryProfiles` + `factions.json.inventoryOverrides`；NPC 初始道具使用 `npcs.json.items` 或 `sect-seed-profiles.json.npcStarterKits`；堂口归属使用 `sect-seed-profiles.json.hallAssignmentProfiles` + NPC 显式覆盖。
4. **实体类型必须显式配置。** 核心宗门在 `factions.json` 写 `isSect: true` 与 `sectTemplateId`；功能组织写 `isPublic: true` / `isSect: false`。代码不得再用 `id.startsWith('sect_')` 或“无 subtype 即宗门”推断运行资格。
5. **缺配置是校验失败，不是旧流程回退。** 最新门派运行默认进入正式流程；不得新增“关闭新机制后旧月俸继续跑”的运行时 fallback。测试 fixture 可以构造最小配置，但正式 runtime 不走旧月俸双轨。
6. **配置引用必须校验。** `sect-organization.json`、`sect-operation.json`、`sect-seed-profiles.json` 中引用的 hall、role、rank、item、questTemplate、seed profile 都要在加载验证脚本中校验存在。
7. **任务筛选也走配置。** 任务板与 QuestService 不得用 `qt_slay_monster` 这类硬编码集合判断任务类型，应使用 `quest-templates.json` 的 category/tag/`sectIssuerHints` 或 `sect-operation.json.questSelection`。

#!/usr/bin/env node
import { fileURLToPath, pathToFileURL } from 'node:url';
import { dirname, resolve } from 'node:path';

const __dirname = dirname(fileURLToPath(import.meta.url));
const GAME_ROOT = resolve(__dirname, '..');
const imp = (p) => import(pathToFileURL(resolve(GAME_ROOT, p)).href);

const { Inventory } = await imp('js/engine/abstract/inventory.js');
const { RuntimeState } = await imp('js/engine/abstract/runtime-state.js');

    id,

    name: staticData.name || id,

function clone(value) {
  return value == null ? value : JSON.parse(JSON.stringify(value));
}

export class QuestRepository {
  constructor() {
    this.quests = new Map();
    this.nextId = 1;
  }

  save(input = {}) {
    const id = input.id || `board_quest_${this.nextId++}`;
    const quest = { ...clone(input), id };
    this.quests.set(id, quest);
    return clone(quest);
  }

  byId(id) {
    const quest = this.quests.get(id);
    return quest ? clone(quest) : null;
  }

  update(id, patch = {}) {
    const quest = this.quests.get(id);
    if (!quest) return null;
    Object.assign(quest, clone(patch));
    return clone(quest);
  }

  all() {
    return [...this.quests.values()].map(clone);
  }
}

export class QuestStateMachine {
  constructor({ allowed = ['open', 'accepted', 'completed', 'cancelled'] } = {}) {
    this.allowed = new Set(allowed);
  }

  transition(quest, state, patch = {}) {
    if (!quest || !this.allowed.has(state)) return { success: false, reason: 'quest_state_invalid' };
    return { success: true, patch: { ...patch, state } };
  }
}

export class QuestVisibilityPolicy {
  constructor({ publicBoards = ['bounty'] } = {}) {
    this.publicBoards = new Set(publicBoards);
  }

  canSee(quest, { factionId = null, questBoard = null } = {}) {
    if (quest.state !== 'open') return false;
    if (questBoard && quest.questBoard !== questBoard) return false;
    return !factionId || quest.factionId === factionId || this.publicBoards.has(quest.questBoard);
  }
}

export class QuestDedupSpecification {
  hasOpen(repository, dedupeKey) {
    if (!dedupeKey) return false;
    return repository.all().some(q => q.state === 'open' && q.dedupeKey === dedupeKey);
  }
}

export class QuestBoard {
  constructor({ repository = new QuestRepository(), stateMachine = new QuestStateMachine(), visibilityPolicy = new QuestVisibilityPolicy(), dedupeSpec = new QuestDedupSpecification() } = {}) {
    this.repository = repository;
    this.stateMachine = stateMachine;
    this.visibilityPolicy = visibilityPolicy;
    this.dedupeSpec = dedupeSpec;
  }

  publish(input = {}) {
    const quest = {
      state: 'open',
      createdDay: input.day ?? 0,
      acceptedDay: null,
      completedDay: null,
      cancelledDay: null,
      factionId: input.factionId || null,
      issuerType: input.issuerType || 'faction',
      issuerId: input.issuerId || input.factionId || null,
      issuerName: input.issuerName || input.issuerId || input.factionId || '宗门',
      questBoard: input.questBoard || 'sect',
      questKind: input.questKind || 'sect_task',
      questTemplateId: input.questTemplateId,
      questTypeId: input.questTypeId || input.questTemplateId,
      difficulty: input.difficulty ?? 1,
      priority: input.priority ?? 0,
      requiredResourceId: input.requiredResourceId || null,
      dedupeKey: input.dedupeKey || null,
      rewardContribution: input.rewardContribution ?? 0,
      escrowId: input.escrowId || null,
      issuerNpcId: input.issuerNpcId || null,
      acceptedByNpcId: null,
      completedByNpcId: null,
      metadata: clone(input.metadata || {}),
    };
    if (this.dedupeSpec.hasOpen(this.repository, quest.dedupeKey)) {
      return { success: false, reason: 'quest_deduped', dedupeKey: quest.dedupeKey };
    }
    return this.repository.save(quest);
  }

  byId(id) {
    return this.repository.byId(id);
  }

  openFor({ factionId = null, questBoard = null } = {}) {
    return this.repository.all()
      .filter(q => this.visibilityPolicy.canSee(q, { factionId, questBoard }))
      .sort((a, b) => (b.priority || 0) - (a.priority || 0) || a.createdDay - b.createdDay)
      .map(clone);
  }

  hasOpenDemand(dedupeKey) {
    return this.dedupeSpec.hasOpen(this.repository, dedupeKey);
  }

  accept(questId, npc, day = 0) {
    const quest = this.repository.byId(questId);
    if (!quest || quest.state !== 'open') return { success: false, reason: 'quest_not_open' };
    const transition = this.stateMachine.transition(quest, 'accepted', { acceptedDay: day, acceptedByNpcId: npc?.id || null });
    if (!transition.success) return transition;
    return { success: true, quest: this.repository.update(questId, transition.patch) };
  }

  complete(questId, npc, day = 0) {
    const quest = this.repository.byId(questId);
    if (!quest || quest.state !== 'accepted') return { success: false, reason: 'quest_not_accepted' };
    const transition = this.stateMachine.transition(quest, 'completed', { completedDay: day, completedByNpcId: npc?.id || null });
    if (!transition.success) return transition;
    return { success: true, quest: this.repository.update(questId, transition.patch) };
  }

  cancel(questId, day = 0) {
    const quest = this.repository.byId(questId);
    if (!quest || quest.state === 'completed') return { success: false, reason: 'quest_not_cancellable' };
    const transition = this.stateMachine.transition(quest, 'cancelled', { cancelledDay: day });
    if (!transition.success) return transition;
    return { success: true, quest: this.repository.update(questId, transition.patch) };
  }

  snapshot() {
    return this.repository.all();
  }
}

        ? (npc.state.get('isHallChief') === true ? number(extra.chief, 0) : number(extra.member, 0))
        : 0;
      const stoneDue = number(roleStones[role], roleStones.disciple ?? 0) + hallExtra;
      totalStoneDue += stoneDue;
      const stone = treasury.transferFactionStonesToNpc({ day, faction, npc, quantity: stoneDue, source: { type: this.id, factionId: faction.id, npcId: npc.id } });
      if (stone.success) totalStonePaid += stone.record?.assets?.[0]?.asset?.quantity ?? stone.quantity ?? stoneDue;

      const pillRule = rankPills[memberRank(npc)];
      if (pillRule?.itemId && pillRule.quantity > 0) {
        totalPillDue += pillRule.quantity;
        const pill = treasury.transferFactionItemToNpc({ day, faction, npc, itemId: pillRule.itemId, quantity: pillRule.quantity, source: { type: 'sect_pill_stipend', factionId: faction.id, npcId: npc.id } });
        if (pill.success) totalPillPaid += pill.record?.assets?.[0]?.asset?.quantity ?? pillRule.quantity;
      }
    }

    const stoneShort = totalStonePaid < totalStoneDue;
    const pillShort = totalPillPaid < totalPillDue;
    faction.state.set('sectLastMonthlyDay', day);
    faction.state.set('sectLastStoneDue', totalStoneDue);
    faction.state.set('sectLastStonePaid', totalStonePaid);
    faction.state.set('sectLastPillDue', totalPillDue);
    faction.state.set('sectLastPillPaid', totalPillPaid);
    faction.state.set('sectSalaryShortfallStreak', stoneShort ? number(faction.state.get('sectSalaryShortfallStreak')) + 1 : 0);
    faction.state.set('sectPillShortfallStreak', pillShort ? number(faction.state.get('sectPillShortfallStreak')) + 1 : 0);
    if (stoneShort || pillShort) {
      faction.state.set('stability', Math.max(0, number(faction.state.get('stability'), 50) - number(config.decline?.stabilityPenaltyPerMonth, 8)));
    }
    return { totalStoneDue, totalStonePaid, totalPillDue, totalPillPaid, stoneShort, pillShort };
  }
}

export class StockPressureRule {
  id = 'stock_pressure';

  stockAmount(faction, rule) {
    if (rule.kind === 'faction_state_resource') return number(faction.state.get(rule.resourceId), 0);
    return number(faction.inventory?.getAmount?.(rule.resourceId), 0);
  }

  run(ctx) {
    const { day, faction, config, organization, questBoard } = ctx;
    const created = [];
    if (!questBoard) return { created };
    for (const rule of config.stockPressure || []) {
      const amount = this.stockAmount(faction, rule);
      if (amount >= number(rule.safeStock, 0)) continue;
      if (questBoard.hasOpenDemand(rule.dedupeKey)) continue;
      const severity = amount <= number(rule.criticalStock, 0) ? 'critical' : 'safe';
      const hall = hallForPressure(organization, rule.issuerHall);
      const quest = questBoard.publish({
        day,
        factionId: faction.id,
        issuerType: 'hall',
        issuerId: hall.id,
        issuerName: hall.issuerName || hall.name,
        questBoard: rule.questBoard || 'sect',
        questKind: rule.questKind,
        questTemplateId: rule.questTemplateId,
        difficulty: rule.difficultyBySeverity?.[severity] ?? 1,
        priority: rule.priority ?? 50,
        requiredResourceId: rule.resourceId,
        rewardContribution: rule.rewardContributionBySeverity?.[severity] ?? 0,
        dedupeKey: rule.dedupeKey,
        metadata: { severity, stockAmount: amount, safeStock: rule.safeStock, criticalStock: rule.criticalStock, settlement: rule.settlement || null },
      });
      if (quest?.success !== false) created.push(quest);
    }
    faction.state.set('sectOpenPressureQuestCount', questBoard.openFor({ factionId: faction.id }).length);
    return { created };
  }
}

export class DepartureRule {
  id = 'departure';

  run(ctx) {
    const { day, faction, members, config, rng } = ctx;
    const cfg = config.decline || {};
    const streak = Math.max(number(faction.state.get('sectSalaryShortfallStreak')), number(faction.state.get('sectPillShortfallStreak')));
    const lowStability = number(faction.state.get('stability'), 50) <= number(cfg.stabilityForDeparture, 35);
    const threshold = number(cfg.shortfallStreakForDeparture, 2);
    const leftNpcIds = [];
    if (streak < threshold && !lowStability) return { leftNpcIds };
    const exempt = new Set(cfg.exemptRoles || ['leader']);
    for (const npc of members) {
      if (exempt.has(memberRole(npc))) continue;
      const chance = Math.min(1, number(cfg.leaveChanceBase, 0.35) + Math.max(0, streak - threshold) * number(cfg.leaveChancePerShortfallStreak, 0.15));
      const roll = rng?.next?.() ?? Math.random();
      if (roll > chance) continue;
      npc.state.set('factionId', null);
      npc.state.set('hasFaction', false);
      npc.state.set('isWanderer', true);
      npc.state.set('hallId', null);
      npc.state.set('isHallChief', false);
      npc.state.set('sectLeftDay', day);
      npc.state.set('sectLeaveReason', lowStability ? 'low_stability' : 'stipend_shortfall');
      leftNpcIds.push(npc.id);
    }
    if (leftNpcIds.length > 0) addState(faction, 'sectDepartureCount', leftNpcIds.length);
    return { leftNpcIds };
  }
}

export class DestructionRule {
  id = 'destruction';

  run({ day, faction, members }) {
    const aliveMembers = members.filter(n => n.alive !== false && n.state?.get?.('alive') !== false);
    if (aliveMembers.length > 0) return { destroyed: false };
    faction.state.set('isDestroyed', true);
    faction.state.set('destroyedReason', 'no_members');
    faction.alive = false;
    return { destroyed: true, reason: 'no_members', day };
  }
}

      faction,

      rng,
      config: this.config,
      organization: this.organization,
      ranksData: this.ranksData,
      economicSystem: this.economicSystem,
      questBoard: this.questBoard,
      bountyService: this.bountyService,
      treasury: this.treasury,
      host: this.host,
    };
    const result = {};
    for (const rule of this.ruleRegistry.resolve(this.config.operationFlow || [])) {
      result[rule.id] = rule.run(ctx);
    }
    return result;
  }

  processAllMonthly({ day = 0, tickLog = null, rng = null } = {}) {
    if (!this.host?.entityRegistry) return [];

    if (day <= 0 || day % interval !== 0) return [];
    const results = [];
    for (const faction of this.host.entityRegistry.getAliveByType('faction')) {
      if (!isSectFaction(faction)) continue;
      const members = this.membersOf(faction);
      const result = this.processMonthly({ day, faction, members, rng });
      results.push({ factionId: faction.id, ...result });
    }
    if (tickLog) tickLog.sectOperations = results;
    return results;
  }
}

## 背景

门派运行需要同时处理组织身份、月俸、宗门库存、库存压力任务、个人悬赏托管、离宗和倒闭。如果直接在 `TickManager` 或单个服务中硬编码，会破坏数据驱动、开闭原则和后续扩展内部政治/宗门经营的边界。

## 决策

1. 门派仍复用 `FactionEntity`，不新增独立门派实体。
2. 门派运行子域放在 `apps/game/js/engine/sect/`。
3. 门派运行采用配置注册表 + 策略规则链；`SectOperationService` 只编排，具体机制由可注册规则处理。
4. 所有可配置内容进入 JSON：组织、运行数值、初始资源/库存、NPC starter kit、堂口编制、任务板可见性、库存压力回流、交易场景。
5. 宗门任务、堂口任务和个人悬赏进入统一任务板；任务来源、可见性、去重、完成处理通过策略/处理器扩展。
6. 宗门灵石等宏观资源以 `FactionState` 为单一真相源；丹药、材料、法器、符箓、功法等实物以 Inventory 为真相源。
7. 个人悬赏奖励使用统一经济托管，不进入宗门普通库存。
8. 最新门派运行默认进入正式流程，不保留旧月俸 fallback。

## 影响

- 后续新增堂口、宗门运行规则、任务来源或悬赏类型时，优先新增配置和策略类，不修改核心编排。
- 初始资产和道具可通过 profile 调参，避免逐宗门/逐 NPC 复制固定库存。
- 验证关注真实模拟中的资产变化、任务生成、托管释放、离宗和倒闭，不使用指纹自证。
```

# 门派运行系统

> 最后更新：2026-06-07

门派运行系统把现有 `FactionEntity` 扩展为可运转的宗门组织。第一版负责堂口身份、宗门背包、月俸、库存压力任务、个人悬赏托管、离宗和倒闭判定。

## 模块

| 模块 | 职责 |
|------|------|
| `SectConfigRegistry` | 校验和解析组织、运行数值、初始化 profile、实体覆盖和任务引用 |
| `SectOperationService` | 编排门派运行规则链，不直接承载具体机制 |
| `SectOperationRuleRegistry` | 注册月俸、库存压力、离宗、倒闭等规则策略 |
| `QuestBoard` | 统一任务列表门面，由 repository/state machine/visibility/dedup policy 组成 |
| `SectBountyService` | 个人悬赏发布、奖励托管、完成释放和取消返还 |
| `SectEscrowHolderRepository` | 管理个人悬赏托管仓快照与恢复 |
| `SectTreasury` | 通过 `EconomicSystem.settle` 发起门派资产转移命令 |
| `sect-organization.js` | 堂口、管理层、堂口资格等纯函数 |

## 真相源

宗门灵石、粮食、抽象弟子数仍以 `FactionState` 为单一真相源。宗门丹药、材料、法器、符箓和功法库存放在 faction inventory。个人悬赏托管奖励由 `SectEscrowHolderRepository` 管理并通过统一经济托管结算，不参与宗门普通月俸、兑换和维护消耗。

## 月度流程

每 `sect-operation.monthlyIntervalDays` 天按 `sect-operation.operationFlow` 执行一次规则链：

1. 按弟子身份发放灵石月俸。
2. 按境界发放丹药月俸。
3. 堂口成员和堂主领取额外灵石。
4. 扣除基础维护费。
5. 欠发会降低稳定度并累计欠发 streak。
6. 库存低于安全线时发布堂口任务。
7. 欠发或稳定度过低时，非豁免弟子可能离宗。
8. 成员全部离开或死亡时宗门倒闭。

## 验证

验证必须观察真实资产变化、任务列表、托管释放、离宗状态和长程模拟统计，不使用指纹或哈希作为正确性证据。
```

| `sectLeftDay` | number/null | 离宗时间 |
