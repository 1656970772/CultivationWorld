## Global Rules

- 原文 `.txt` 检索默认按 GBK/GB18030 或 PowerShell `-Encoding Default` 处理；不要按 UTF-8 直接断言“无命中”。
- `武破九荒.txt` 多繁体，检索必须繁简同搜，例如 `蕭葉/萧叶`、`真靈大陸/真灵大陆`。
- `武炼巅峰.txt` 可能含 `CRCRLF` 换行，执行时不要改写原文文件。
- 不粘贴长篇原文。每条事实使用“文件路径 + 行号/章节/关键词 + 事实摘要”作为证据锚点。
- “原作事实”和“我的判断/游戏化抽象”必须分栏；概率、占比、权重、AI 参数如果没有原文直接依据，必须标注“我的判断”或“待核验”。
- 已删除的临时文件 `docs/世界观参考/来源索引.md` 和 `docs/世界观参考/世界观参考资料补全计划.md` 不恢复。
- 不直接删除时间戳备份文件。先把独有内容归并到主文档，再在对应 `待做分析列表.md` 中记录“可删除候选”，由用户确认后处理。
- 世界观参考资料不等于项目已敲定设定。只有用户确认后的设定才同步到 `docs/worldbuilding/wiki/`。

## Target Template

每个主题 Markdown 最终至少包含以下结构；旧文档可保留更细的二级标题，但必须补齐这些字段。

```md
# 《作品名》主题名

> 最后更新：2026-06-05
> 状态：原文已复核 / 待复核 / 待补原文
> 资料来源：`docs/世界观参考/作品名/作品名.txt`
> 适用范围：说明本文覆盖和不覆盖的设定范围

## 原文证据锚点

| 锚点ID | 原文文件 | 行号/章节/关键词 | 事实摘要 | 支持的设定点 |
|--------|----------|------------------|----------|--------------|

## 设定摘要

## 原作事实

## 游戏化抽象（我的判断）

## 可转化为数据或事件的条目

## 与 Wiki / 数据 / ADR 的关系

## 待确认问题

## 变更记录
```
