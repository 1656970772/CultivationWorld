# NPC 战斗智能闭环阶段 3：路线与目标重定向 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
> 最后更新：2026-06-06

**Goal:** 让斩妖任务在移动前评估路线风险，避开高阶妖兽密集路径，并在目标失踪或目标过强时重定向到可解释的安全目标。

**Architecture:** 新增轻量 `combat-route-risk.js` 服务，只做路径附近威胁评分和安全目标选择，不重写移动系统。`quest-service.js` 绑定目标时读取排除列表和路线风险，找不到安全目标时返回失败，由阶段 2 的风险分支处理。

**Tech Stack:** WorldContext `computePath()`、EntityRegistry、Quest Service、Quest Toils、Node 工具测试。

## Task 2: 路线风险服务

- [ ] **Step 1: 创建服务**

Create `apps/game/js/engine/npc/services/combat-route-risk.js`，导出：

```js
export function scoreRouteRisk(entity, target, worldContext, opts = {}) {}
export function chooseSafeHuntTarget(entity, monsters, worldContext, opts = {}) {}
```

评分输入：

- NPC 当前坐标。
- 目标坐标。
- `worldContext.computePath()` 路径。
- 路径半径内活体妖兽。
- `worldContext.npcCombatPower()` 与妖兽 power。

- [ ] **Step 2: 斩妖移动前插入路线 Toil**

在 `job_npc_monster_hunt` 的 `move` 前加入：

```json
{ "id": "plan_route", "type": "toil_plan_safe_hunt_route" }
```

新增 Toil 配置：

```json
{ "id": "toil_plan_safe_hunt_route", "name": "规划安全斩妖路线", "category": "combat" }
```

- [ ] **Step 3: 实现路线 Toil**

路线风险可接受时写：

```text
safeHuntRouteReady=true
combatRouteRiskScore=<number>
```

路线风险过高时写：

```text
safeHuntRouteReady=false
needsEasierHuntTarget=true
```

并返回 `REPLAN`，reason 为 `hunt_route_too_dangerous`。

## Task 3: 目标绑定重定向

- [ ] **Step 1: 排除过强目标**

`resolveQuestTargetMonster()` 或新 helper 读取 `excludedHuntMonsterIds`，不再选择已排除妖兽。

- [ ] **Step 2: 安全目标选择**

候选妖兽按阶位、距离、路线风险、直接战力风险排序。优先同阶或低一阶；没有安全目标时返回 `safe_hunt_target_missing`。

- [ ] **Step 3: 目标失踪重定向**

锁定目标死亡或失踪时：

- 尝试同阶附近活体妖兽。
- 写回新 `questTargetMonsterId`、坐标、妖兽名、阶位。
- 找不到时任务失败，不能凭空完成。
