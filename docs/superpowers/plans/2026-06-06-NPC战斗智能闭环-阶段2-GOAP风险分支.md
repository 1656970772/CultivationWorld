# NPC 战斗智能闭环阶段 2：GOAP 风险分支 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
> 最后更新：2026-06-06

**Goal:** 让斩妖前置风险评估输出可规划状态，使 NPC 能进入准备、补给、恢复、请求同伴或换目标分支，而不是只 `REPLAN` 后继续硬杀。

**Architecture:** 即时战斗仍由阶段 1 的 Reaction 层处理。本阶段只处理斩妖任务出发前和战后恢复的 GOAP Job/Toil 分支：`needsCombatSupply`、`needsCombatRecovery`、`needsCompanion`、`needsEasierHuntTarget` 等状态由风险 Toil 写入，再由 JobAction 消费。

**Tech Stack:** GOAP Planner、JobAction、Job/Toil、Economy Toils、Node 工具测试。

## Task 2: JobAction 与默认行为池

- [ ] **Step 1: 新增 JobAction**

在 `npc-job-actions.json` 增加：

- `act_npc_prepare_combat`：前置 `needsCombatSupply=true`，效果 `combatReady=true`、`needsCombatSupply=false`。
- `act_npc_retreat_and_heal`：前置 `needsCombatRecovery=true`，效果 `needsCombatRecovery=false`、`shouldRetreat=false`。
- `act_npc_request_hunt_companion`：前置 `needsCompanion=true`、`huntCompanionRequested=false`，效果 `hasHuntCompanion=true`。

- [ ] **Step 2: 加入默认 JobAction 池**

在 `npc-action-sets.json.defaultNpcJobActionIds` 中加入三条 JobAction，放在经济补给 JobAction 之前。

- [ ] **Step 3: 收紧经济补给前置**

`act_npc_acquire_heal_item` 和 `act_npc_acquire_artifact` 加 `needsCombatSupply=true`，避免日常规划频繁购买战斗物资。

## Task 3: 风险 Toil 写入可规划状态

- [ ] **Step 1: 扩展 `assessCombatRisk()`**

写入：

```js
combatRiskScore
monsterTooDangerous
shouldRetreat

needsCombatSupply
needsCompanion
needsEasierHuntTarget
combatReady

```

- [ ] **Step 2: 风险 Toil 返回明确分支原因**

按优先级返回：

```text

combat_supply_required
hunt_companion_required
easier_hunt_target_required
monster_too_dangerous
```

- [ ] **Step 3: 补给准备不只检查回血丹**

`toil_prepare_combat_supply` 同时检查回血丹、法器和真气。缺回血丹或法器时返回 `REPLAN`，让经济 Job 接手。

- [ ] **Step 4: 放弃过强目标写排除列表**

`toil_abort_overdangerous_target` 写入 `excludedHuntMonsterIds`，清空当前 `questTargetMonsterId`，写 `needsEasierHuntTarget=true`。
