# 修士战斗属性体系 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
> 最后更新：2026-06-07

**Goal:** 将修士从旧的 `hp/maxHp + baseDef 比例减伤 + rank/qi 战力估值` 迁移到“境界裸表 × 小层倍率 + AttributeSet/GAS 修正 + 数值护甲伤害”的统一战斗属性体系。

**Architecture:** 新增三张运行时数据表作为单一真相源：境界参考基表、普通修士裸表、妖兽危险层级参考表；修士运行时通过 `cultivator-combat-attributes.js` 计算基础面板，并把 `maxHp/maxYuan/attack/defense/speed/soul` 写入 NPC state 作为 AttributeSet 基值。旧开关关闭时保留现有 HP 与比例减伤路径；新开关开启后统一战斗入口读取数值攻击、防御和有效属性，功法、法宝、体质、丹药和临时状态只通过 AttributeSet/GAS 叠加。

**Tech Stack:** JSON 数据配置、ESM 纯函数计算器、NPCEntity/WorldEngine 运行时接线、GameplayEffect/AttributeSet、Node 工具测试、真实多种子模拟、ADR/系统文档。

```js
export const COMBAT_ATTRIBUTE_KEYS = ['hp', 'yuan', 'attack', 'defense', 'speed', 'soul'];

export const RANK_STAGE_MULTIPLIERS = Object.freeze({
  early: 1.0,
  middle: 1.15,
  late: 1.45,
  perfection: 2.0,
});

export function normalizeRankStage(stage, rankId = 'mortal') {
  if (rankId === 'mortal') return null;
  return Object.prototype.hasOwnProperty.call(RANK_STAGE_MULTIPLIERS, stage) ? stage : 'early';
}

export function rankStageMultiplier(stage, rankId = 'mortal', tables = {}) {
  if (rankId === 'mortal') return 1;
  const configured = tables.combatBaseTable?.stageMultipliers || RANK_STAGE_MULTIPLIERS;
  const normalized = normalizeRankStage(stage, rankId);
  return configured[normalized] ?? RANK_STAGE_MULTIPLIERS[normalized] ?? 1;
}

export function calculateCultivatorCombatAttributes({ rankId = 'mortal', rankStage = null, tables = {} } = {}) {
  const table = tables.cultivatorCombat?.ranks || tables.ranks || {};
  const row = table[rankId] || table.mortal || {};
  const multiplier = rankStageMultiplier(rankStage, rankId, tables);
  const hp = Math.max(1, Math.round((row.hp ?? 1) * multiplier));
  const yuan = Math.max(0, Math.round((row.yuan ?? 0) * multiplier));
  const attack = Math.max(0, Math.round((row.attack ?? 0) * multiplier));
  const defense = Math.max(0, Math.round((row.defense ?? 0) * multiplier));
  const speed = Math.max(0, Math.round((row.speed ?? 0) * multiplier));
  const soul = Math.max(0, Math.round((row.soul ?? 0) * multiplier));
  return {
    rankId,
    rankStage: normalizeRankStage(rankStage, rankId),
    hp,
    maxHp: hp,
    yuan,
    maxYuan: yuan,
    attack,
    defense,
    speed,
    soul,
  };
}

export function readEffectiveCombatAttribute(entity, key, fallback = 0) {
  if (!entity) return fallback;
  if (entity.attributes?.getEffective) {
    const effective = Number(entity.attributes.getEffective(key));
    if (Number.isFinite(effective)) return effective;
  }
  const stateValue = Number(entity.state?.get?.(key));
  return Number.isFinite(stateValue) ? stateValue : fallback;
}

export function calculateNumericArmorDamage({
  attack = 0,
  defense = 0,
  skillMultiplier = 1,
  sceneMultiplier = 1,
  randomMultiplier = 1,
  extraReductionMultiplier = 1,
} = {}) {
  const atk = Math.max(0, Number(attack) || 0);
  const def = Math.max(0, Number(defense) || 0);
  const baseDamage = atk * Math.max(0, Number(skillMultiplier) || 0) * Math.max(0, Number(sceneMultiplier) || 0);
  const armorCoefficient = atk > 0 ? atk / (atk + def) : 0;
  const reduced = baseDamage * armorCoefficient * Math.max(0, Number(randomMultiplier) || 0) * Math.max(0, Number(extraReductionMultiplier) || 0);
  return Math.max(1, reduced);
}

# ADR-053：修士战斗属性体系

> 最后更新：2026-06-07
> 状态：已采纳
> 关联：ADR-041、ADR-042、ADR-051、ADR-052
> 来源：`docs/superpowers/specs/2026-06-06-修士战斗属性体系-design.md`

## 背景

旧 NPC 战斗只维护 `hp/maxHp`、按境界读取的比例减伤 `baseDef`，以及用于 AI 的综合战力估值。修士、妖兽、功法、法宝和 GAS 之间没有共同的基础属性语言，导致后续技能、装备、神魂和追逃机制难以统一接入。

## 决策

1. 修士与妖兽统一使用气血、真元、攻击、防御、速度、神魂六项基础属性。
2. `qi` 继续表示修炼与突破真气；`yuan/maxYuan` 表示战斗真元。
3. `combat-base-table.json` 是境界参考基表，`cultivator-combat.json` 是普通修士裸表，`monster-combat.json` 是妖兽危险层级参考表。
4. `rankStage` 表示小层，凡人为 `null`，修士境界默认为 `early`，突破到新境界后重置为 `early`。
5. NPC state 中的 `maxHp/maxYuan/attack/defense/speed/soul` 是 AttributeSet 基值；功法、法宝、体质、灵根、丹药、符箓、阵法和临时状态通过修正层叠加。
6. 伤害迁移为数值护甲公式：`damage = max(1, attack * skillMultiplier * sceneMultiplier * attack / (attack + defense) * random * extraReduction)`。
7. 迁移由 `combat.cultivatorAttributes.enabled` 和 `numericArmorDamage` 分阶段开启。

## 后果

- 普通无装备修士不会自动压制同阶妖兽；好功法和好法宝可以形成越阶空间。
- 神魂第一阶段不进入普通伤害公式，只进入感知、幻术、驯化、威压和 AI 估值。
- 旧 `baseDef` 在开关关闭时继续可用，避免一次性改变所有战斗结果。

## 验证要求

- 运行属性表、计算器、NPC runtime、数值护甲和既有战斗智能脚本。
- 开关关闭时观察旧路径不崩。
- 开关开启后用真实多种子模拟观察低阶危险、同阶妖兽压制、装备/功法带来的反杀空间和 `qi/yuan` 拆分后的突破节奏。
```

# 修士战斗属性体系

> 最后更新：2026-06-07

## 属性

六项基础属性为气血、真元、攻击、防御、速度、神魂。运行时字段为 `maxHp/hp`、`maxYuan/yuan`、`attack`、`defense`、`speed`、`soul`。

## 数据来源

- `apps/game/data/definitions/combat-base-table.json`：境界参考基表。
- `apps/game/data/definitions/cultivator-combat.json`：普通修士裸表。
- `apps/game/data/definitions/monster-combat.json`：妖兽危险层级参考表。

## 运行时

`NPCEntity` 在迁移开关开启时用 `calculateCultivatorCombatAttributes()` 生成 AttributeSet 基值。功法与法宝通过 `combatModifiers` 写入 AttributeSet 修正层。

## 伤害

数值护甲公式由 `resolveCombatEncounter()` 在 `combat.cultivatorAttributes.numericArmorDamage=true` 时启用。开关关闭时保留旧比例减伤。

## 验证

不得用单一摘要值自证。需要运行工具测试，并用真实多种子模拟观察战斗、追逃、斩妖、突破节奏。
```

# 战斗属性数据模型

> 最后更新：2026-06-07

| 字段 | 类型 | 说明 |
|------|------|------|
| `rankStage` | string/null | `early` / `middle` / `late` / `perfection`，凡人为 `null` |
| `hp` | number | 当前气血 |
| `maxHp` | number | 气血上限 |
| `yuan` | number | 当前战斗真元 |
| `maxYuan` | number | 战斗真元上限 |
| `attack` | number | 普攻、术法、法宝的基础攻击面板 |
| `defense` | number | 数值护甲 |
| `speed` | number | 战斗速度、追逃、反应标尺 |
| `soul` | number | 神魂、感知、幻术、驯化、威压和 AI 估值 |

`qi` 不是战斗真元，继续表示修炼、积累和突破用真气。
```
